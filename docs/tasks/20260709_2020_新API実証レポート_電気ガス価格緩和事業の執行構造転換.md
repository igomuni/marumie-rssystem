# 新API実証レポート：年度間比較で見つけた「電気・ガス価格激変緩和対策等事業」の執行構造転換

- 作成日: 2026-07-09
- 対象ブランチ: `feat/sankey-query-extensions`
- 対象API: `GET /api/sankey/query`（`compareYears` / `summary.recipients.topShare1・topShare3`）、`GET /api/search/projects?scope=details`

## 要旨

1. `compareYears=2024,2025` を1コール引くだけで、全5,664〜5,794事業の中から支出増減トップ・新規/消滅事業が自動抽出された。中でも「電気・ガス価格激変緩和対策等事業」は事業年度2024→2025（予算年度2023→2024実績）で支出が2兆6,812億円→9,766億円へ約64%減少していた。
2. 同事業を `topShare1/3` で見ると、上位1支出先シェアが**88.27%→21.91%**へ劇的に低下していた。深掘りすると、単一の執行代行事業者（株式会社博報堂）を通す一括執行から、電力・ガス各社への直接支払いへと**執行スキームそのものが2025年に切り替わった**ことが原因だとわかった（執行代行費もHakuhodoの約2.37兆円→デロイトトーマツの約43.6億円へ縮小）。
3. `scope=details` 全文検索で「値引き原資」という語を引くと、事業名に一致しない同事業が `matchedIn: "details"` として1コールで特定でき、事業名検索では拾えない背景情報の発掘が確認できた。

## 発見2: 電気・ガス価格激変緩和対策等事業 — topShare1が88%→22%に急落、原因は執行スキームの切替

`fnp=電気・ガス価格激変緩和対策等事業` で単年度クエリを引き、`summary.recipients.topShare1/3` を確認した。

| 年度（実質予算年度） | 支出先数 | topShare1 | topShare3 | 上位支出先 |
|---|---|---|---|---|
| 2024（2023実績） | 12 | **0.8827** | 0.9321 | 株式会社博報堂 2兆3,666.5億円 |
| 2025（2024実績） | 13 | **0.2191** | 0.5216 | 「その他」2,140.3億円／東京電力エナジーパートナー 1,914.3億円 |

`summary.projects.spendingTotal` は2024年が2兆6,812.4億円・2025年が9,766.5億円で、`compareYears` のdiff値（spendingBase/spendingCompare）と完全一致し、クロスチェックが取れた。

topShare1=88.27%だけを見ると「1社への極端な集中」に見えるが、`/api/subcontracts/7259` を引いて内訳を確認すると解釈が変わる。

- **2024年（予算2023実績）**: ブロックA「株式会社博報堂」は `isDirect: true, isTerminal: false` の**執行団体（ルート）ノード**で、役割は「需要家に対する電気・都市ガス料金の値引きを実施する小売事業者等の公募・選定、実施結果の報告・管理業務」。総額2兆3,666.5億円のうち、実際に電力・ガス各社へ再委託された値引き原資はブロックB（東京電力エナジーパートナー他）の1兆8,997.3億円で、博報堂自身の人件費は7.9億円、再委託費（審査業務・コールセンター・広報等）は135.2億円にとどまる。つまりtopShare1=88.27%の大半は**博報堂という一企業への集中ではなく、単一執行代行スキームを通した資金の通過分**である（プレイブック§2「ルートとベンダーの二重計上」の典型例）。
- **2025年（予算2024実績）**: `/api/subcontracts/7259?year=2025` では博報堂ブロックが消え、電力各社への支払いブロック（A・H）が `isDirect: true, isTerminal: true` として**直接**計上され、執行代行の役割はデロイトトーマツファイナンシャルアドバイザリー合同会社（申請受付・審査支援等）に移ったが、その総額はわずか43.6億円（2024年の博報堂ブロック2兆3,666.5億円の1/500程度）。この執行スキームの転換（単一代行事業者への一括計上 → 各電力・ガス会社への直接計上）が、topShare1が88.27%から21.91%へ急落した直接の理由である。

**旧API比較**: 支出先集中度（topShare）は旧来summaryのtop10件からは算出不可能で、全マッチの合計とシェアを出すには支出先明細を人力で合算する必要があった（`/api/quality-scores/recipients` 等で数KB〜のレスポンスを事業ごとに引いて自前集計）。新APIでは `summary.recipients.topShare1/3` が**フィルタクエリ1コールに同梱**され、集中度の一次スクリーニングが即時にできる。ただし今回のように「集中度の実態（ルート/ベンダーの別）」まで踏み込むには、従来通り `/api/subcontracts/[pid]` の明細確認（今回2コール）が引き続き必要——topShareは「スクリーニング」であり「解釈」まではしてくれない。

## 発見3: scope=details 全文検索が事業名検索では拾えない事業を特定した

「電気・ガス価格激変緩和対策等事業」の概要文には「値引き原資」という語が繰り返し登場するが、この語は事業名には含まれない。`GET /api/search/projects?q=値引き原資&scope=details&year=2025` を引くと：

```json
{
  "metadata": { "totalHits": 1, "scope": "details" },
  "items": [
    {
      "pid": "7259",
      "name": "電気・ガス価格激変緩和対策等事業",
      "ministry": "経済産業省",
      "matchedIn": "details"
    }
  ]
}
```

`matchedIn: "details"` が明示され、事業名検索（`scope=name`、既定）では引っかからない事業が概要・目的・現状課題のテキストから正しく特定できた。今回は事業を既に知っていたため確認的な実行だったが、事業名が抽象的（例:「〇〇推進事業」のような形式的名称）で内容から探すしかないケースでの威力を示す最小デモとなった。

**旧API比較**: 旧来は事業名検索のみのため、名称に手がかりがない場合は候補事業を大量に列挙して概要文を目視確認するしかなかった（数十〜数百件規模の走査）。`scope=details` は**1コール**でヒット事業と一致箇所種別（`matchedIn`）を返す。

## 限界・注意事項

- **年度ズレの罠**: `compareYears=2024,2025` は「事業年度2024 vs 2025」だが、事業年度Nのデータは**予算年度N-1の執行実績**を表す。したがって本レポートの比較は実質**予算年度2023実績 vs 2024実績**の比較である。本文中「2024年」「2025年」の表記はすべて事業年度（=API応答の年度パラメータ）を指し、括弧内に実質の予算年度実績を併記した。
- **ルートとベンダーの二重計上**: 発見2の博報堂ブロック（2024年）は執行団体（ルート）であり事業費全額を計上する。その内訳のベンダー（東京電力エナジーパートナー等）への再委託額と単純合算すると二重計上になるため、本レポートではルート額・ベンダー額を分けて記述した。
- **topShareは「スクリーニング指標」**: `topShare1/3` は支出先の名称・法人番号単位でのシェアであり、執行団体か最終ベンダーかの区別はしない。88.27%という高い値だけを見て「1社への異常な集中」と断定するのは誤りで、本レポートのように `/api/subcontracts/[pid]` で内訳を確認する必要がある。
- **中小企業生産性革命推進事業の類似ケース**: 発見1の増加事業には「中小企業生産性革命推進事業」（`fnp=中小企業生産性革命推進事業` で2事業がヒット、2024年計443.5億円→2025年計4,973.4億円）も含まれていたが、`topShare1=1.0`（両年度とも上位1件のみ）だった。ただし唯一の支出先は独立行政法人中小企業基盤整備機構であり、これも執行団体（プレイブック§2「執行団体の壁」）——その先の個々の補助対象中小企業への配分はデータに現れない。本レポートでは電気・ガス事業の方がスキーム転換という時系列の変化を示せるため主題としたが、topShare=1.0の事業すべてを「極端な集中」と読むのは同様に誤りうる。
- データ範囲は行政事業レビュー対象事業のみで、国の予算全体の約27%（国債費・地方交付税等は含まない）。「無駄」の断定はしていない（観測事実は「増減」「集中度の変化」「執行スキームの転換」のみ）。

### 絞り込みに使ったSankeyQuery（`fnp` 短縮形と等価）

発見2で使用した「電気・ガス価格激変緩和対策等事業」の単年度クエリを `SankeyQuery` JSON形式で示す（`q=` パラメータで直接指定した場合と等価）:

```json
{
  "year": "2024",
  "filter": {
    "projectName": { "query": "電気・ガス価格激変緩和対策等事業", "regex": false },
    "recipientName": null,
    "ministries": [],
    "budget": { "min": null, "max": null },
    "spending": { "min": null, "max": null },
    "accountCategories": ["general", "special", "both", "none"]
  }
}
```

`compareYears` 利用時は上記の `filter` をそのまま両年度に適用し、`compareYears=2024,2025` をURLパラメータとして追加する。
