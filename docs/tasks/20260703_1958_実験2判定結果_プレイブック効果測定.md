## 判定の前提: 数値の独立検証

両レポートの主要数値を判定者が API で独立再取得した結果、**両者とも完全一致**（94事業・予算5,319,994,379,000円・支出4,340,875,418,209円・支出先638・上位支出先の金額0.1億円単位まで一致）。**数値の正確性ではモデルの地力に問題がなく、差は「規律・様式・効率」に現れた**。

## 結論

**仮説は支持された。B（プレイブックあり）が罠回避・様式完成度・探索効率で明確に優位。** プロトコルの「結果の使い道」に従い、**プレイブックは恒久運用とし、公開フェーズ（F）の llms.txt 設計に反映する**。

特筆点:

1. **数値の正確性に差はなかった**（両者満点・独立クロス一致）。プレイブックが埋めたのは計算力ではなく「規律」——pids バッチの使用、断定回避の言い回し、自己検証、用語の厳密さ
2. **基準外の定性観察**（事前固定基準の外であることを明示した上で記録): A は top10 に絞らず 94 事業全体の統計・低スコア事業リストまで踏み込んでおり、**探索の幅では A が上回る**。プレイブックが「標準ルートへの収束」と引き換えに探索の多様性をやや狭めた可能性がある。プレイブックに「標準ルートは出発点であり、価値ある逸脱を妨げない」旨の一文を足す価値がある
3. 前回（省エネ・プレイブックなし）に「まずまず」だった Opus 4.8 が、条件Aでもかなり良いレポートを出している。api-guide の整備（URL パラメータ表・notes）自体が条件Aにも効いており、**ドキュメント整備の効果はプレイブック単体より広い**

## 副産物（レポート内容からの発見・改善候補）

1. **防衛省2024の品質スコアは全94事業が `aiSource=heuristic`**（両レポートが独立に発見）— AI再評価の未適用領域。スコア再生成の対象候補として要検討
2. **防衛データは「執行団体の壁」の最大級の事例**: 「その他」5,124.8億円（支出の11.8%・単独最大）、FMS 米政府 7,637億円（17.6%）、防衛局系 pass-through 4,854億円。実験①（再エネ）の構図の相似形で、探索レポートの汎用性を裏づける
3. **プロトコル改善**: API呼び出しの一覧・回数の記録をレポート必須項目にする（条件Aの P2 が判定不能だった）→ プレイブック §5 に追記候補
4. **プレイブック/api-guide 改善**: `pp` パラメータの正規形式（`project-spending-<pid>`）の明記
