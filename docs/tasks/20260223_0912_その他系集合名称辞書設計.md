## 3. 新規辞書ファイルの設計

### 3-1. ファイル

| 項目 | 内容 |
|------|------|
| ファイル名 | `public/data/dictionaries/aggregate_names.csv` |
| 生成スクリプト | `scripts/generate-aggregate-names.py` |
| 種別 | キュレーション型（スクリプト内にデータ埋め込み） |
| エントリ数 | 33件（canonical 5件 + exact alias 23件 + regex alias 5件）|

### 3-2. スキーマ

既存の `beneficiary_names.csv` に準拠する。

| 列名 | 型 | 役割 |
|------|----|------|
| `aggregate_id` | string | エンティティ一意 ID（スネークケース） |
| `category` | string | 分類カテゴリ（都道府県 / 市区町村 / 地方公共団体 / 行政機関 / 国際機関） |
| `canonical_name` | string | 代表表記（UI 表示・集計用）← このグループの「まとめ名」 |
| `name` | string | マッチング対象の文字列（完全一致 or 正規表現） |
| `name_type` | string | `canonical` / `alias` |
| `match_type` | string | `exact`（完全一致）/ `regex`（正規表現）※ 省略時は `exact` |

### 3-3. 正規表現エントリの設計方針

数字部分が変動する集合表現（`その他37県`、`その他11都道府県` 等）は、`match_type = regex` の単一エントリに汎化する。正規表現は Python `re.fullmatch()` で評価する。

| aggregate_id | regex パターン | 汎化対象（数字バリアント） |
|-------------|--------------|--------------------------|
| `pref_aggregate` | `その他[0-9]+(県\|都道府県\|道府県\|府県)` | その他37県, その他37都道府県, その他11都道府県, その他24都道府県, その他25府県 |
| `muni_aggregate` | `その他[0-9]+市町村` | その他31市町村 |
| `rodo_kyoku_agg` | `その他[0-9]+(都道府県)?労働局` | その他37都道府県労働局, その他37労働局, その他28労働局 |
| `rodo_kyoku_agg` | `その他労働局\([0-9]+局\)` | その他労働局(37局), その他労働局(25局) |
| `local_gov_mixed` | `その他自治体\([0-9]+\)` | その他自治体(54) |

**適用ルール:**
- `canonical` 行は常に `match_type = exact`（canonical_name はデータに存在しない代表名のため）
- `match_type = regex` は `alias` 行にのみ付与する
- 正規表現を使用しない大多数のエントリは `match_type = exact`（明記するか省略可）

---

### 3-4. canonical_name の設計方針

「その他X系」の名称は多数の表記ゆれを持つ集合表現のため、**1グループ = 1 canonical** でまとめる。各 canonical_name は集合であることを明示するために「（集合）」を末尾に付与する。

| aggregate_id | category | canonical_name |
|-------------|---------|---------------|
| `pref_aggregate` | 都道府県 | `都道府県（集合）` |
| `muni_aggregate` | 市区町村 | `市区町村（集合）` |
| `local_gov_mixed` | 地方公共団体 | `地方公共団体（都道府県・市区町村集合）` |
| `rodo_kyoku_agg` | 行政機関 | `労働局（集合）` |
| `intl_org_agg` | 国際機関 | `国際機関（集合）` |

## 6. 実装方針

### レイヤー設計上の位置付け

| 要素 | 該当レイヤー | 適合性 |
|------|------------|--------|
| `scripts/generate-aggregate-names.py` | Data Pipeline (`scripts/`) | ✅ CSV処理のみ |
| `public/data/dictionaries/aggregate_names.csv` | Data Pipeline（出力物） | ✅ Git 管理対象 |

### 生成スクリプトの方針

`beneficiary_names.csv` と同様の**キュレーション型**（スクリプト内にデータ埋め込み）とする。

- 外部データソース不要（エントリはすべて本設計書から手動確定）
- 出力先: `public/data/dictionaries/aggregate_names.csv`
- 実行: `python3 scripts/generate-aggregate-names.py`

### 既存辞書ファイルへの追記は行わない

`prefecture_names.csv` / `municipality_names.csv` / `ministry_supplement.csv` への追記は行わない。理由:

1. 既存辞書は「実在する個別エンティティ」を登録する設計だが、`その他都道府県` 等は「集合表現」であり性質が異なる
2. 別ファイルに分離することで「集合名称」の管理境界を明確化できる
3. 既存生成スクリプトの再実行で上書きされるリスクを回避できる
