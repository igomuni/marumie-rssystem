# /sankey-svg ノードラベル常時表示 設計

## 目的

ユーザーが「すべてのノードラベルを表示」トグルを操作しなくても常にラベルが読める状態にする。設定項目を減らしてUIをシンプルにする。

---

## 現状

### `showLabels` の役割

`showLabels`（URL param `sl`）は2つの箇所に影響している。

**① レイアウト計算（`page.tsx` ~L772）**

```
showLabels=false → computeLayout(... /* minNodeGap なし */)
showLabels=true  → 2パス計算:
                    1) noGap = computeLayout(... /* minNodeGap なし */) で fitZoom を算出
                    2) computeLayout(... , minNodeGap = 14 / fitZoom)
```

2パス目の `minNodeGap = 14 / fitZoom` により、小さいノードの後ろのギャップが最低 14px（視覚）まで拡張される。これはノード高さは変えず、**ギャップのみ** を広げる。

**② ラベル描画（`page.tsx` ~L1739, ~L1811）**

```
labelVisible = showLabels || (h + NODE_PAD) * zoom > 10 || isSelected
```

`showLabels=false` のとき、視覚高さが 10px 未満のノードはラベルを描画しない。

### `minNodeGap` の仕組み（`sankey-svg-filter.ts` L777〜）

```
effectivePad = max(NODE_PAD, minNodeGap)

各ノードのギャップ:
  effectiveH < effectivePad → gap = effectivePad  // 拡張
  effectiveH ≥ effectivePad → gap = NODE_PAD      // 通常
```

ノード高さ（`effectiveH`）は変更されない。ギャップだけ広がる。

## 設計

### 変更方針

`showLabels` を廃止し、常に 2パスレイアウト＋常時ラベル描画を採用する。

| 項目 | 現状 | 変更後 |
|---|---|---|
| `showLabels` state | あり | 削除 |
| URLパラメータ `sl` | あり | 削除 |
| UIチェックボックス | あり | 削除 |
| レイアウト計算 | 条件分岐 | 常に2パス |
| ラベル描画条件 | `showLabels \|\| 高さ>10px \|\| selected` | 常に描画（選択/グレーアウト条件のみ残す） |
| ページの縦方向 | fitZoomでビューポートに収める | 収まらなくても許容 |

### レイアウト変更の詳細

`showLabels` の条件分岐を削除し、常に2パスを実行する。

```
// 変更前
if (!showLabels) {
  const result = computeLayout(...)
  ...
}
const noGap = computeLayout(...)
const fitZoom = ...
const result = computeLayout(... , 14 / fitZoom)

// 変更後
const noGap = computeLayout(...)
const fitZoom = ...
const result = computeLayout(... , 14 / fitZoom)
```

`useMemo` の依存配列から `showLabels` を除去する。

### ラベル描画の変更

`showLabels` の参照を削除し、`labelVisible` を以下に変更する。

**通常ノード（~L1811）:**

```
// 変更前
const labelVisible = showLabels || (h + NODE_PAD) * zoom > 10 || isSelected;

// 変更後
const labelVisible = true;  // または isSelected など残すべき条件のみ
```

実際には `connectedNodeIds`（フォーカスモード）・`hoveredNodeIds` によるグレーアウトはラベルの `fill` 色で表現されており、`labelVisible` 自体は `true` 固定で問題ない。

**マージノード（~L1739）:** 同様に `showLabels` 参照を削除。

### `SankeyUrlState` 型の変更

`showLabels: boolean` フィールドを削除する。URL パース処理（`sl` パラメータの読み取り）も削除する。

## 影響範囲

| ファイル | 変更内容 |
|---|---|
| `app/sankey-svg/page.tsx` | `showLabels` state・URL param・UI・labelVisible条件・useMemo依存 |
| `app/lib/sankey-svg-filter.ts` | 変更なし |
| `types/` | 変更なし（`SankeyUrlState` は page.tsx 内で定義） |

---

## 注意事項

- `minNodeGap = 14 / fitZoom` は **現在の fitZoom（ビューポートに収めたときのズーム）** を基準に算出する。ギャップ拡張によりレイアウト高さが `svgHeight` を超える場合、初期ズームでは下部が切れる。これは許容とする（後続でオートズーム調整を検討）。
- 既存の `sl=0` URLを保持しているユーザーがいる場合、`sl` パラメータは無視されるだけで壊れない。
