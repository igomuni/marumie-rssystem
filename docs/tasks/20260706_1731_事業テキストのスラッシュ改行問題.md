# 事業テキストの「/」改行問題 — 調査と方針

- 日付: 2026-07-06
- 対象: 事業スコア詳細ダイアログ（`ScoreDetailDialog`）の事業テキスト表示（目的・現状・課題・概要）
- きっかけ: PID 3522 の「現状・課題」に含まれる URL が改行で分断されて表示される不具合

## 1. 事象

`/quality`・`/sankey-svg` の事業スコア詳細ダイアログで、事業テキスト内の URL が
複数行に分断されて表示されていた。

例（PID 3522「現状・課題」の一部・実データ）:

```text
…供給安定性等の観点で脆弱な状況にある。/(※)総務省2023年版情報通信白書第2部第5節1(2) https://www.soumu.go.jp/johotsusintokei/whitepaper/ja/r05/html/nd245120.html/世界のマクロセル基地局市場のシェア(2022年・出荷額)において、日本企業の合計シェアは2.3%
```

この URL `https://www.soumu.go.jp/.../nd245120.html` はスラッシュを多数含むため、
表示ロジックがすべての `/` を改行に置換した結果、URL が縦にバラバラになっていた。

## 2. 原因

表示コンポーネント `client/components/quality/ScoreDetailDialog.tsx` が、事業テキストを
次のように**全ての `/` を改行へ一律置換**していた。

```tsx
<div className="… whitespace-pre-wrap …">{text.replace(/\//g, '\n')}</div>
```

### 導入経緯（git 履歴）

- 初出コミット: `bc9eef2`（2026-06-19）「feat(quality): /quality を AI評価ベースの品質スコアに再設計（rs-visから取り込み）」
- **rs-vis から現行ダイアログを取り込んだ時点で一緒に入った実装**。プロジェクト当初からではなく、
  現行 `/quality` ダイアログの発足以降に存在。その後 `#234 → #245` で複製され現在に至る。

### 「/」はどこから来るのか — **元CSV由来**

- 生成パイプライン `scripts/generate-project-details.ts` の `normalizeString()` は `trim()` のみで、
  **改行を `/` に変換していない**。
- `/` は **RSシステムの元CSV（`1-2_RS_{YEAR}_基本情報_事業概要等.csv`）のセル内にそのまま含まれる**。
  1セルに複数の注記・箇条を `/` 区切りで詰め込む形式になっている。
- つまり `/` を改行にしていたのは「元は改行だったものを復元」ではなく、
  **元データの区切り文字 `/` を表示側が改行として解釈していた**にすぎない。

## 3. データ実態（定量）

`public/data/rs{2024,2025}-project-details.json` の `purpose` / `currentIssues` / `overview` を集計。

| 指標 | 2024（全5,664事業） | 2025（全5,794事業） |
|---|---|---|
| `/` を含む事業 | 3,766（66.5%） | 3,895（67.2%） |
| `/` の総出現数 | 16,862 | 18,240 |
| URL を含む事業 | 62 | 57 |
| └ うち改行で分断されうる | **62（100%）** | **56** |
| 数字/数字（日付・分数等）を含む事業 | 341 | 331 |

`/` の用法は一義的でなく、少なくとも次が混在する（実データ抜粋）:

```text
1/2上限10百        ← 補助率の分数（改行は明確に誤り）
定率補助2/3上限15百  ← 補助率の分数
研修)/2.新任管理     ← 箇条書きの区切り（改行が自然な箇所）
の統括・監理/デジタル庁が  ← 語間の区切り
https://…/index.html)  ← URL 内スラッシュ＋閉じ括弧
```

**結論**: `/` は「区切り」「補助率(分数)」「URL内」「日付」など多義的に使われており、
一律に改行へ置換すると URL・分数・日付を破壊する。実データで URL を含む事業は
100% 近くが分断されており、影響は無視できない。

## 4. 方針（決定）

**表示側で `/` を改行に変換するのを現時点で廃止する。**

- 事業テキストは**ソースの表記をそのまま保持**して表示する（`/` は文字として表示）。
- **URL のリンク化は現時点では行わない**（プレーンテキストのまま表示）。長い URL・語は
  `break-words` でコンテナ内に折り返す。改行分断が無くなるため、URL も1つの連続文字列として読める。

### トレードオフ

- 箇条書きの区切り（`/2. …`）も改行されなくなり、長文が連続表示になる箇所がある。
  → **正確性（URL・分数・日付を壊さない）を優先**し、可読性の作り込みは将来課題とする。
- URL は非クリックのプレーンテキスト。リンク化は必要になった時点で別途対応（下記「将来課題」）。

### 実装

`ScoreDetailDialog.tsx` の表示を `text.replace(/\//g, '\n')` から**素の `{text}`** に変更
（`whitespace-pre-wrap break-words` でソース改行・長語の折り返しは維持）。整形ヘルパーは設けない。

## 5. 将来課題（今回はやらない）

- **URL のリンク化**: 事業テキスト内の URL を検出してクリック可能なリンクにする（今回は見送り）。
  末尾の閉じ括弧・句読点・区切りスラッシュを href から除外する処理が必要。`/sankey-svg`
  サイドパネルの事業概要表示（現状プレーンテキスト）と方針を揃えて実装するのが望ましい。
- **より賢い区切り解釈**: 「補助率(分数)・日付・URL は保持」しつつ「箇条書き区切りだけ改行」する
  ヒューリスティック（例: `。/`・`）/`・`/\d+\.` のような境界のみ改行）。誤爆と可読性の綱引き。
- **パイプライン側での構造化**: 生成時に `/` 区切りを配列へ分解し、UI では箇条書き表示する案。
  ただし分数・URL・日付の `/` を巻き込まない分割ロジックが必要で、本質的な難しさは同じ。
- **他ページへの波及確認**: `/sankey-svg` サイドパネルの事業概要表示など、同種の `/` 改行や
  URL 表示があれば同じ方針で揃える。

## 6. 影響範囲

- 変更ファイル: `client/components/quality/ScoreDetailDialog.tsx`（`/quality` と `/sankey-svg` 共有）
- データ再生成: 不要（表示ロジックのみ）
- 検証: `npx tsc --noEmit` パス。PID 3522 で URL が改行分断されず1つの連続文字列として表示され、
  補助率(1/2 等)・日付・箇条書きの `/` も文字のまま壊れないことを確認。
