## Codex向けプロンプト案

私なら、単なる「デザイン変更」ではなく、設計思想から共有します。

> **目的**
>
> 再委託ビューを「別アプリ」ではなく、メインのSankeyビューの自然な延長として再設計する。
>
> **デザイン原則**
>
> * メイン画面をデザインシステムの唯一の基準とする。
> * フォント、余白、角丸、色、アイコン、サイドパネル構成を統一する。
> * カード型UIよりも情報密度の高いリスト・ラベル中心のレイアウトを採用する。
> * 白背景と広い余白を維持し、色は「事業・直接・再委託・別財源」の意味付けのみに使用する。
> * サイドパネルは詳細ページではなく、選択中ノードのインスペクターとして機能させる。
> * 再委託フローはメインSankeyと同じ操作感（ズーム、パン、選択、ハイライト）を維持する。
> * 画面遷移を減らし、可能な限りコンテキストを維持したまま表示モードだけを切り替える。
>
> **許容範囲**
>
> UI・レイアウト・操作体系は既存との互換性を重視せず、UX向上を優先して全面的に再構成してよい。

このレベルまで方針を明確にすると、Codexや他のAIが「見た目を揃える」だけでなく、**一つのプロダクトとして統一された体験**を目指した提案を出しやすくなると思います。
