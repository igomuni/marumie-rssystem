# 再委託CSVデータ構造再整理

調査日: 2026-05-09  
対象: `data/year_2024`, `data/year_2025` の 5-1 / 5-2 / 5-3 CSV と `scripts/generate-subcontracts.ts`

## 結論

再委託構造ビューは、現状の「ノードをクリックするとサイドパネルの主内容が切り替わる」構成よりも、次の2層を分けて扱うほうがデータ構造に合っている。

1. **資金の流れリスト**
   - ブロック同士の接続構造
   - 担当組織からの直接支出か、ブロック間支出か
   - 補足情報
   - 参考標記や間接経費の扱い
2. **ブロック内訳リスト**
   - ブロックごとの合計支出額
   - 支出先の数
   - 事業を行う上での役割
   - ブロック内の支出先、契約概要、費目・使途

「ブロック同士のつながり」と「ブロック内の支出先内訳」は別物として見るべき。支出先ノード間の関係は CSV から直接は分からないケースがある。

## CSVの役割

### 5-1 支出先_支出情報

ブロックと、そのブロック内の支出先・契約情報を持つ。

主要列:

- `予算事業ID`
- `支出先ブロック番号`
- `支出先ブロック名`
- `支出先の数`
- `事業を行う上での役割`
- `ブロックの合計支出額`
- `支出先名`
- `法人番号`
- `支出先の合計支出額`
- `契約概要`
- `金額`
- `契約方式等`

行の意味は大きく2種類ある。

- ブロックサマリ行: `支出先ブロック名` と `ブロックの合計支出額` があり、`支出先名` が空
- 支出先/契約行: `支出先名` があり、支出先合計や契約概要、契約金額を持つ

例: PID 1 / ブロック A

```text
ブロックサマリ:
  A / 株式会社ウルフスタイル
  役割: ウェブ会議システムを利用した研修の運営支援に関する業務
  ブロック合計: 3,025,000

支出先合計:
  株式会社ウルフスタイル / 3,025,000

契約行:
  2024年度ウェブ会議システム新任管理者マネジメント研修の運営支援業務の請負 / 1,980,000
  2024年度ウェブ会議システム幹部候補育成課程中央研修(係長級)業務の請負 / 1,045,000
```

### 5-2 支出先_支出ブロックのつながり

ブロック間の接続だけを持つ。ブロック内のどの支出先が下流に支出したかは分からない。

主要列:

- `支出元の支出先ブロック`
- `支出元の支出先ブロック名`
- `担当組織からの支出`
- `支出先の支出先ブロック`
- `支出先の支出先ブロック名`
- `資金の流れの補足情報`
- `国自らが支出する間接経費`
- `国自らが支出する間接経費の項目`
- `国自らが支出する間接経費の金額`

`支出元の支出先ブロック` が空で `担当組織からの支出=TRUE` の行は、担当組織からブロックへの直接支出を表す。

### 5-3 支出先_費目・使途

支出先ごとの費目・使途を持つ。ただし全支出先に存在するわけではない。

主要列:

- `支出先ブロック番号`
- `支出先名`
- `法人番号`
- `契約概要`
- `費目`
- `使途`
- `金額`

## 役割とブロックの関係

「事業があり、その事業を行うための細分化された役割がある」という見方はかなり合っている。

2025年のブロックサマリ行では:

| 項目 | 件数 |
|---|---:|
| 役割あり | 19,120 |
| 役割空欄 | 5 |
| 役割の種類 | 15,269 |

ただし「役割 = ブロック」とは限らない。

同一事業内で、同じ `事業を行う上での役割` が複数ブロックに分かれるケース:

| 項目 | 件数 |
|---|---:|
| 該当事業数 | 831 |
| 該当する 事業 x 役割 ペア | 1,212 |

例:

| PID | 役割 | ブロック |
|---:|---|---|
| 7 | 広報関係業務 | B, G, H |
| 7 | インターネットによるアンケート調査の実施業務 | D, I |
| 11 | 機器・ソフトウェア等の貸付 | A, E |
| 11 | システム運用・保守 | B, F |
| 12 | システム開発・設計 | A, F |

解釈:

- 役割は「事業を分解した業務カテゴリ」に近い
- ブロックは「その役割を担う支出先集合」または「その役割の支出段階」
- 同じ役割が直接支出ブロックと再委託ブロックに分かれることがある
- したがって、UIでは `役割` をブロックカードの補足に埋め込むだけでなく、一覧の主要列として扱うべき

## ブロック名と支出先名の関係

ブロック名は支出先名と一致する場合もあるが、一致しない場合も多い。

2025年の観察:

| パターン | 件数 |
|---|---:|
| 支出先0件のブロック | 1,022 |
| 支出先1件のブロック | 11,416 |
| 支出先1件でブロック名=支出先名 | 9,601 |
| 支出先1件だが名称表記が違う | 1,815 |
| 支出先複数のブロック | 8,059 |

支出先1件でも表記揺れがある。

例:

- `富士通エフネッツサービス株式会社` vs `富士通エフネットサービス株式会社`
- `(株)毎日企画サービス` vs `株式会社毎日企画サービス`
- `The Global Steering Group for Impact Investment` vs `The Global Steerig Group for Impact Investment`

解釈:

- ブロック名は必ずしも正規化済みの支出先名ではない
- ブロック名を支出先IDとして扱うと危険
- ブロックを主語にした表示が必要

## 費目・使途の有無

2025年:

| 項目 | 件数 |
|---|---:|
| 一意の支出先 | 74,291 |
| 費目・使途がある支出先 | 15,594 |
| 費目・使途があるブロック | 15,594 |
| 支出先を持つブロック | 19,475 |
| 費目・使途がない支出先ブロック | 3,881 |
| 支出先ベースの費目・使途カバー率 | 21.0% |
| ブロックベースの費目・使途カバー率 | 80.1% |

解釈:

- 費目・使途は「ある支出先にはある」補足情報で、全体の主構造にはできない
- ブロック単位では多くのブロックに存在するが、支出先単位では約2割
- UIでは常時表示ではなく、支出先詳細または契約/費目タブで扱うほうがよい

## 5-2 フローの注意点

2025年:

| 項目 | 件数 |
|---|---:|
| フロー行 | 23,381 |
| 担当組織からブロックへの行 | 15,887 |
| ブロック間フロー行 | 7,494 |
| 補足情報あり | 4,510 |
| 参考標記らしい補足情報 | 7 |
| 国自らが支出する間接経費行 | 2,432 |

`国自らが支出する間接経費` は `支出先の支出先ブロック` が空の行として出ることがあり、現行の `generate-subcontracts.ts` では `targetBlock` が空のためフローとしては捨てられる。

これはブロック構造とは別の「国自身が使った経費」として扱うべき可能性が高い。

### 府省庁以外が起点のように見えるケース

`支出元の支出先ブロック` が空で、かつ `支出先の支出先ブロック` がある明示的なルート行については、2024年・2025年ともに次の性質だった。

- `担当組織からの支出=TRUE`
- `支出元の支出先ブロック名` は `府省庁` 列と完全一致

つまり、CSVの明示的なルートは府省庁と見てよい。

## 支出先間エッジを作れない理由

2025年のブロック間フロー 7,494 件について、支出元ブロック内の支出先数を見ると:

| 支出元ブロック内の支出先数 | 接続数 | 割合 |
|---:|---:|---:|
| 0件 | 202 | 2.7% |
| 1件 | 5,435 | 72.5% |
| 2件以上 | 1,857 | 24.8% |

支出元ブロックに支出先が1件だけなら、その支出先から下流へ流れたと推定できる。  
ただし支出先が2件以上あるブロックでは、CSVに「どの支出先が下流ブロックへ支出したか」がない。

したがって、支出先同士のエッジを作ると約25%のブロック間フローで推測や捏造が混ざる。画面の主構造は支出先ではなくブロックに置くべき。

## 現在の画面構成が微妙に見える理由

現状のサイドパネルは「選択中ブロックの詳細」と「支出先一覧」を同じ文脈で出している。しかしCSV上は:

- 事業
- 資金の流れ
- 支出ブロック
- 事業を行う上での役割
- 支出先
- 契約概要
- 費目・使途
- 国自らが支出する間接経費

が別粒度で存在する。

ノードクリックでサイドパネル全体の主語を切り替えると、「構造を見たい」のか「支出先内訳を見たい」のかが混ざる。

## UI再設計の方向

ページの主語は常に「事業」に固定する。クリックは主語変更ではなく、ハイライトやリスト内のフォーカスに留める。

推奨構成:

```text
ヘッダー
  事業名 / PID / 府省庁 / 予算 / 執行 / ブロック数 / 支出先数

左または上: フロー図
  事業 -> ブロック -> ブロック
  構造把握とハイライトに限定

右または下: 資金の流れリスト
  from / to / 直接支出 / 補足情報 / 参考標記 / 間接経費

下または右: ブロック一覧
  ブロックID
  ブロック名
  合計支出額
  支出先の数
  事業を行う上での役割
  depth
  直接/再委託

詳細領域:
  選択ブロックの支出先内訳
  契約概要
  費目・使途
```

タブにするなら:

- `資金の流れ`
- `ブロック`
- `支出先`
- `契約・費目`
- `間接経費`

重要なのは、最初に `資金の流れリスト` と `ブロック一覧` を並べて見せること。ノードクリックでいきなり支出先詳細に飛ばすより、事業構造を理解しやすい。
