# 予算0円・支出あり事業の調査

> 「支出0円事業の表示オプション化」実装中に偶然発見。

## 発見の経緯

「支出が0円の事業を対象にする」オプション OFF 時、`事業(予算)` 集約ノードのサイドパネル「流出先」に **0円の項目** が表示された。しかし実際の `事業(支出)` は 0円ではなかった。

## 実態

**224件**の事業が「予算額 = 0円、支出額 > 0円」という状態にある。

## 金額の由来（データ生成ロジック）

`scripts/generate-sankey3-data.ts:251` の処理:

```typescript
// 予算年度=2023 のデータのみ budgetMap に集計
const budgetAmount = budget?.totalBudget || 0;
```

2-1 CSV（予算・執行サマリ）は**予算年度 = 2023** のみフィルタリングして集計する。事業が 1-1 CSV（組織情報）に存在していても、2-1 CSV の2023年度行に対応エントリがなければ `budgetAmount = 0` になる。

## 推定原因

デジタル庁の「情報通信技術調達等適正・効率化推進費」のような**一括計上予算科目**は、複数府省共通のプラットフォーム費用として個別事業ではなく別の予算事業IDで計上されていると考えられる。そのため：

- 支出データ（5-1 CSV）には個別事業名・支出額が記録されている
- 予算データ（2-1 CSV）には対応する個別事業IDの2023年度行が存在しない
- → `budgetAmount = 0`、`spendingAmount > 0` となる

## 現状の扱い

- `generate-sankey3-data.ts:286` で `flowValue = Math.min(budgetAmount, spendingAmount) = 0`
- `事業(予算) → 事業(支出)` のエッジが生成されない（`flowValue > 0` の条件をパスできない）
- `事業(予算)` ノードが `value = 0` のまま存在し、支出ノードとの接続がない

## 未対応事項・次のステップ候補

- 2-1 CSV の予算年度フィルタを調整して対応できるか調査
- 「情報通信技術調達等適正・効率化推進費」のような一括計上予算が実際に別事業IDで存在するか確認
- 支出>0・予算=0 の事業に対して、表示上の扱い（非表示・警告・別表示）を検討
