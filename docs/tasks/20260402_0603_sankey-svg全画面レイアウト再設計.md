# /sankey-svg 全画面レイアウト再設計

## 目的

ユーザーが Sankey 図を閲覧する際、ヘッダー・フッターや外枠のパディングで描画領域が圧迫されている状態を解消し、画面全体をキャンバスとして活用できるようにする。

---

## 現状の問題点

| 要素 | 現状 | 問題 |
|------|------|------|
| ルートレイアウト | `padding: 20` + `minHeight: 100vh` の div | 全方向に余白が生まれ描画領域が小さい |
| ヘッダー | `<h1>` + `<p>` がキャンバス上部を占有 | 閲覧時に不要な情報が常に表示 |
| フッター（凡例） | キャンバス外の下部に配置 | 描画エリアをさらに圧迫 |
| コントロール | 1行に全スライダーを並べた横長パネル | 画面幅が狭いと折り返し・使いにくい |
| Zoom UI | 数値入力 + +/- ボタン | 細かい数値調整には不向き |
| TopN スライダー | 常時表示 | 支出先オフセット以外は変更頻度が低い |
| SVG コンテナ | `border: 1px solid #ddd` の白ボックス | 「埋め込み UI」感。地図的な没入感がない |

---

## 設計方針

**Google Maps スタイル**: キャンバスが画面全体を占め、UI コントロールは透過フローティングパネルとして重ねる。

---

## 新しいレイアウト構造

```text
┌─────────────────────────────────────────────────────────┐
│  [タイトルバッジ]               [支出先オフセット] [⚙]  │ ← top overlay
│                                                         │
│                                                         │
│                  SVG キャンバス（全画面）                 │
│                                                         │
│                                                         │
│  [ミニマップ]           [凡例]          [ズームスライダー]│ ← bottom overlay
└─────────────────────────────────────────────────────────┘
```

### ルートコンテナ

- `position: fixed; inset: 0; overflow: hidden`
- `/sankey2/page.tsx` と同じ `100vw / 100vh` 全画面レイアウト
- ヘッダー・フッター廃止（`<h1>`, `<p>`, 凡例 div を削除）
- 白背景（`background: '#fff'`）は維持（SVG の見やすさのため）
- `containerRef` をこのルート div に直接付ける（現在は内側の白ボックスに付いている）

### SVG

- `width={svgWidth}`, `height={svgHeight}` は `containerRef.current.clientWidth / clientHeight` から取得（変更なし、ResizeObserver 対応済み）
- `style={{ position: 'absolute', inset: 0 }}` でコンテナを完全に埋める
- ボーダーなし・`borderRadius` なし

---

## フローティングパネル配置

### 上部右パネル（`top: 12, right: 12`）

**含む要素:**
- 支出先オフセット（`recipientOffset`）スライダー一式
  - 数値入力 + `〜N位` ラベル + range スライダー + `/件` カウント
- ⚙ ボタン（設定ダイアログを開く）

**スタイル:** 現行の `rgba(255,255,255,0.92)` 半透明パネルを踏襲

### 上部左バッジ（`top: 12, left: 12`）

- 小さなテキストバッジとして「直接支出サンキー図」程度のタイトルを表示
- クリック不要・存在感を最小限に
- スタイル: `rgba(255,255,255,0.7)`, `fontSize: 11`, `padding: 4 8`

### 下部右パネル（`bottom: 12, right: 12`）

**ズームスライダー（現行の +/-/数値入力 を置き換え）:**

| 項目 | 仕様 |
|------|------|
| `type="range"` | `min=20, max=500, step=5` (%) |
| 表示ラベル | `{Math.round(zoom * 100)}%` |
| Reset ボタン | 現行の `resetView()` を呼ぶ |
| 向き | 水平スライダー（vertical は HTML 標準外のため避ける） |
| 廃止 | `+` / `-` ボタン、数値テキスト入力 |

> ズームスライダーの値変化時は現行の `setZoom(value / 100)` を呼ぶ。スライダーはキャンバス中心基準でズームする（`applyZoom` 相当の計算が必要）。

### 下部左（変更なし）

- ミニマップ（canvas）は現行の `left: 8, bottom: 8` を維持

### 凡例（下部中央付近）

- フッターから **フローティングパネルに移動**
- `bottom: 12, left: 8` のミニマップの右隣に配置（または `bottom: 12` 中央）
- 内容は現行と同じ（予算側 / 支出側 / 集約ノード）

## 設定ダイアログ

### トリガー

上部右パネルの ⚙ ボタン → `showSettings: boolean` state で開閉。

### 表示位置

⚙ ボタン直下（`position: absolute` on 上部右パネル内）または画面中央モーダル。  
**推奨**: ボタン直下のドロップダウン型（モーダルオーバーレイは不要な重さ）。

### 含む要素

| スライダー | 現行 state | range |
|-----------|-----------|-------|
| 省庁 TopN | `topMinistry` / `setTopMinistry` | 1〜37 |
| 事業 TopN | `topProject` / `setTopProject` | 1〜50 |
| 支出先 TopN | `topRecipient` / `setTopRecipient` | 1〜100 |

各スライダーは現行と同じ「数値入力 + range スライダー」を縦並び（flex-direction: column）で配置。

### 閉じ方

- ⚙ ボタン再クリック
- ダイアログ外クリック（`onMouseDown` on overlay 背景）
- `Escape` キー（optional）

---

## 変更が不要な要素

| 要素 | 理由 |
|------|------|
| ズーム・パン ハンドラ (`handleWheel`, `handleMouseDown` など) | ロジック変更なし |
| ツールチップ DOM | `position: absolute` で containerRef 基準のまま |
| `filterTopN()` 関数 | 変更なし |
| `computeLayout()` 関数 | 変更なし |
| SVG 描画コンテンツ（ノード・リンク・ラベル） | 変更なし |
| ミニマップ描画ロジック | 変更なし |
| ResizeObserver | `containerRef` が変わるが同じ使い方で動作する |

---

## state の変更

| state | 変更 |
|-------|------|
| `showSettings: boolean` | 新規追加（設定ダイアログ開閉） |
| 既存 state すべて | 変更なし |

## 実装上の注意

- **ズームスライダーの値変化** は `setZoom(value / 100)` を直接呼ぶと画面左上基準でズームする。キャンバス中心基準にするには `applyZoom(newZoom / zoom)` 相当の pan 補正が必要。既存 `applyZoom(factor)` はビューポート中心基準なので、スライダー変化時も同様に処理すること。
- **`containerRef` の移動**: 現在は白ボックス div に付いているが、ルート div に移す。`svgWidth`/`svgHeight` の計算元がルートコンテナになり全画面サイズになる。
- **`position: fixed` vs `position: relative`**: `/sankey2` は `position: relative` + `100vw/100vh`。どちらでも可だが、スクロールバーが出ないよう `overflow: hidden` を確実に付ける。
