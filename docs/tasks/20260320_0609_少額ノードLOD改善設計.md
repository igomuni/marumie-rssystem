# 少額ノードLOD改善設計

## 目的

高倍率ズーム時に少額ノードのラベルが重なり、gapがノードサイズより大きくなる問題を解消し、ズームレベルに応じて適切な粒度のノードが表示されるようにする。

## 前回の対策と残課題

前回（`MIN_NODE_SIZE=0.01` 保証）ではgap後のノードが最低0.01pxになるようにしたが、以下の問題が残っている:

1. **ラベル重なり**: ノード矩形が極小のため、ラベルが隣接ノードと重なる
2. **gap > ノードサイズ**: NODE_GAP=1pxに対し、50%のrecipientノード（~1280万円以下）がgap後の寸法0.01px。ズーム時にgapが巨大な空白として見える
3. **ノード過多**: k=100（10000%）で `MIN_SCREEN_AREA=1` だと37,258ノード全表示 → 描画負荷+視覚的ノイズ

## 解決方針

### 1. gapを比例制にする（レイアウト側）

`applyGap` で固定1pxではなく、ノード寸法に比例したgapを適用:

```
effectiveGap = min(NODE_GAP, min(width, height) * GAP_RATIO)
```

- `GAP_RATIO = 0.2` なら、ノード寸法の20%以上はgapに取られない
- 大きなノードは従来通り1pxギャップ
- 小さなノードはギャップが縮小し、面積が保全される
- `MIN_NODE_SIZE` 保証は不要になる（gap自体が縮小するため）

### 2. MIN_SCREEN_AREAを引き上げる（ビュー側）

現在 `MIN_SCREEN_AREA = 1`（1px²）だが、ラベル表示や視認性を考えると小さすぎる。

**推奨値: `MIN_SCREEN_AREA = 25`**（≈5×5px相当）

- k=100で22,056ノード表示（現在37,258から41%削減）
- ラベルは別途 `LABEL_SCREEN_AREA=400` で制御されるため、ノード矩形だけが表示される領域が生まれ、ラベル重なりが軽減
- これ以上大きくすると中程度のノードが消えるリスクあり

### 3. ラベルoverflow修正（ビュー側）

`foreignObject` の `overflow` 属性をSVG属性として設定し、ブラウザ互換性を確保:

- SVG属性: `overflow="hidden"` （CSSの `style={{ overflow: 'hidden' }}` に加えて）
- ラベルdiv: `whiteSpace: nowrap` + `overflow: hidden` + `textOverflow: ellipsis`
