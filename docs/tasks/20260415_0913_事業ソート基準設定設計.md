# 事業ソート基準設定設計

## 目的

ユーザーが事業ノードの並び順を「予算額」か「支出額」かで切り替えられるようにするため。
予算列と支出列が異なる基準でソートされると視覚的に矛盾が生じるため、両列を統一した基準で並べられるようにする。

---

## 現状の問題

| 列 | 現在のソート基準 |
|----|----------------|
| 予算列（project-budget） | `budgetNode.value`（予算額） |
| 支出列（project-spending） | `n.value`（支出額） |

- 予算額と支出額が大きく異なる事業があると、両列の並び順が一致しない
- `projectOffsetMode` では直前の修正で予算列を支出額基準に強制している（`layoutValue = projectOffsetMode ? n.value : adjBv`）が、これはあくまでオフセット整合のための暫定対応
- 通常モードでも両列の順序不一致は起きている（ただしオフセット影響がないため今まで顕在化していなかった）

---

## 新機能スペック

### 新規設定

| 設定 | 型 | デフォルト | 意味 |
|------|----|-----------|------|
| `projectSortBy` | `'budget' \| 'spending'` | `'budget'` | 事業ノードの並び順基準 |

デフォルトを `'budget'` にする理由：
- 予算額は事業の規模感をあらわす公式な指標であり、初期表示として自然
- 予算列のノード高さと並び順が一致するため、初見ユーザーに分かりやすい

---

## 実装設計

### 課題：`value` フィールドの二重使用

`computeLayout` のソート（[sankey-svg-filter.ts:648](app/lib/sankey-svg-filter.ts#L648)）：

```typescript
colNodes.sort((a, b) => {
  const ap = sortPriority(a); const bp = sortPriority(b);
  if (ap !== bp) return ap - bp;
  return b.value - a.value;  // ← ここが視覚的な上下順を決める
});
```

`value` は「ノードの視覚的な高さ（= 金額のスケール）」と「ソートキー」を兼ねており、予算ノードの高さ（予算額）を変えずにソート順だけ変えることができない。

### 解決策：`layoutSortValue` フィールドを追加

`RawNode` に `layoutSortValue?: number` を追加。`computeLayout` のソート時に `layoutSortValue ?? value` を使用する。

- ソートキーと視覚的な高さが分離され、どちらか一方だけ上書きできる
- 既存コードへの影響は最小限（デフォルトは `undefined` = 既存動作と同じ）

#### `types/sankey-svg.ts` 変更

`RawNode` に追加：

```
/** If set, layout engine uses this value as the column sort key instead of value */
layoutSortValue?: number;
```

#### `app/lib/sankey-svg-filter.ts` 変更

**1. シグネチャ追加**

```
projectSortBy: 'budget' | 'spending' = 'budget'
```

`offsetTarget` の直後に追加。

**2. `projectOffsetMode` プレ計算のランキング基準変更**

```typescript
// 現在
.sort((a, b) => b.value - a.value)

// 変更後
.sort((a, b) => {
  if (projectSortBy === 'budget') {
    const ba = nodeById.get(`project-budget-${a.projectId}`)?.value ?? 0;
    const bb = nodeById.get(`project-budget-${b.projectId}`)?.value ?? 0;
    return bb - ba;
  }
  return b.value - a.value;
})
```

**3. `topProjectNodes` 構築ループ（Step 4）のソート基準変更**

非 `projectOffsetMode` で `topMinistryAllProjects` を取り出すときも基準を揃える：

```typescript
topMinistryAllProjects.sort((a, b) => {
  if (projectSortBy === 'budget') {
    const ba = projectAdjustedBudget.get(`project-budget-${a.projectId}`) 
              ?? nodeById.get(`project-budget-${a.projectId}`)?.value ?? 0;
    const bb = projectAdjustedBudget.get(`project-budget-${b.projectId}`) 
              ?? nodeById.get(`project-budget-${b.projectId}`)?.value ?? 0;
    return bb - ba;
  }
  return (projectWindowValue.get(b.id) || 0) - (projectWindowValue.get(a.id) || 0);
});
```

**4. ノード構築ループ（`for (const n of topProjectNodes)`）**

`projectSortBy` に応じて `layoutSortValue` をセット：

```typescript
// 予算ノード
if (budgetNode) {
  const adjBv = projectAdjustedBudget.get(budgetNode.id) ?? budgetNode.value;
  const layoutValue = projectOffsetMode ? n.value : adjBv;          // 削除予定（後述）
  nodes.push({ ...budgetNode, value: adjBv, rawValue: budgetNode.value,
    isScaled: adjBv < budgetNode.value,
    layoutSortValue: projectSortBy === 'spending' ? n.value : undefined,
    skipLinkOverride: true });
}

// 支出ノード
nodes.push({ ...n, value: spendingValue, rawValue: ..., isScaled: ...,
  layoutSortValue: projectSortBy === 'budget'
    ? (nodeById.get(`project-budget-${n.projectId}`)?.value ?? undefined)
    : undefined,
  skipLinkOverride: true });
```

この変更で `layoutValue = projectOffsetMode ? n.value : adjBv` のハック行は不要になり削除できる。

**5. `computeLayout` のソート変更**

```typescript
colNodes.sort((a, b) => {
  const ap = sortPriority(a); const bp = sortPriority(b);
  if (ap !== bp) return ap - bp;
  return (b.layoutSortValue ?? b.value) - (a.layoutSortValue ?? a.value);
});
```

#### `app/sankey-svg/page.tsx` 変更

**状態追加**

```typescript
const [projectSortBy, setProjectSortBy] = useState<'budget' | 'spending'>('budget');
```

**URL パラメータ**

| param | 値 | 省略条件 |
|-------|----|---------|
| `ps` | `s`（spending）| デフォルト `budget` のため `spending` のときのみ保存 |

**`filterTopN` 呼び出しへの引数追加**

`projectOffset` の後に `projectSortBy` を追加。

**設定パネル UI**

「事業の集約ノードを表示」の下に追加：

```
事業ノードの並び順:
  ○ 支出額  ● 予算額
```

（ラジオボタン or セレクトボックス。他のトグルと揃えるならセレクト）

**オフセットリセット**

`projectSortBy` 変更時に `projectOffset` を 0 にリセット（ランキング基準が変わるため）。

## 既存機能への影響

| 機能 | 影響 |
|------|------|
| `projectOffsetMode` プレ計算ランキング | 基準が `projectSortBy` に従う。`layoutValue` ハック削除 |
| `ministryFocusMode` のプロジェクトソート | `topMinistryAllProjects` ソートに `projectSortBy` を適用 |
| `recipientFocusMode` | 変更なし（プロジェクトは流入額基準でソートされており、分離されている） |
| `aggNodeMembers` のソート | 変更なし（集約ノード内の並びは独立した `value` ソート） |
| `scaleBudgetToVisible` | 変更なし（`layoutSortValue` は元の `budgetNode.value` を参照するため、スケール後の `adjBv` の影響を受けない） |
