# サイドパネル再設計

## 目的

ユーザーが選択したノードに関連する府省庁・事業・支出先を、ノードの種類にかかわらず直感的に把握できるようにするため。  
現行の「流入元 / 流出先」タブはエッジの方向を意識する必要があり、探索的な利用には向いていない。

---

## 現状の課題

| 課題 | 内容 |
|------|------|
| タブ概念が直感に反する | 「流入元」「流出先」はネットワーク知識がないと意味が分かりにくい |
| 集約ノードの中身が見えない | `__agg-recipient` クリック時に内包する支出先リストが直接見えない |
| ノード種別ごとに表示内容が大きく変わる | project-budget / project-spending で微妙に異なる表示ロジックが混在している |

---

## 新設計: 3タブ型パネル

「流入元 / 流出先」の2タブを **府省庁 / 事業 / 支出先** の3タブに置き換える。  
選択ノードの種類によって、各タブの表示内容が変わる。  
どのノードを選択しても3タブは常にコンテンツを持つため、グレーアウトは行わない。

---

## セクション定義

### 府省庁タブ

| 選択ノード | 表示内容 |
|-----------|---------|
| `total` | 全省庁リスト（予算額降順） |
| `ministry` | **自身の省庁（1件）**＋配下の事業件数・予算総額 |
| `__agg-ministry` | 集約されている省庁のメンバーリスト |
| `project-budget` / `project-spending` | 所属省庁（1件） |
| `recipient` / `__agg-recipient` | この支出先に支出している省庁（事業経由で集計） |
| `__agg-project-*` | 集約事業の所属省庁リスト（重複省庁は合算） |

### 事業タブ

| 選択ノード | 表示内容 |
|-----------|---------|
| `total` | 全事業リスト（省庁グルーピング付き） |
| `ministry` | この省庁の事業リスト（支出額降順） |
| `__agg-ministry` | 集約省庁配下の事業リスト（省庁グルーピング付き） |
| `project-budget` / `project-spending` | **自身の事業（1件）**（予算額・支出額・支出先件数を表示） |
| `__agg-project-budget` / `__agg-project-spending` | 集約されている事業のメンバーリスト（省庁グルーピング付き） |
| `recipient` | この支出先に支出している事業リスト（支出額降順） |
| `__agg-recipient` | この集約支出先に支出している事業リスト |

### 支出先タブ

| 選択ノード | 表示内容 |
|-----------|---------|
| `total` | 全支出先リスト（上位N件） |
| `ministry` | この省庁が支出する支出先リスト（支出額降順） |
| `project-budget` / `project-spending` | この事業の支出先リスト（全件、支出額降順） |
| `__agg-project-budget` / `__agg-project-spending` | 集約事業が支出する支出先リスト（重複支出先は合算） |
| `recipient` | **自身の支出先（1件）**（名称・金額を表示） |
| `__agg-recipient` | 集約されている支出先のメンバーリスト |

## タブUI仕様

```
┌──────────────────────────────┐
│  府省庁(3) │ 事業(142) │支出先(40) │  ← タブバー（件数バッジ付き）
├──────────────────────────────┤
│ • 財務省              3.2兆円 │  ← クリックで選択
│ • 厚生労働省          2.1兆円 │
│ • その他              1.5兆円 │
│         さらに表示           │
└──────────────────────────────┘
```

- タブバーに件数バッジ（括弧内）を表示
- ノード選択時にデフォルトタブを自動選択する
- 省庁グルーピングは `renderGrouped` の既存実装を転用（事業タブ内）
- 各アイテムはクリックで `handleConnectionClick` を呼ぶ

### タブ自動選択ルール

| 選択ノード | デフォルトタブ |
|-----------|--------------|
| `total` | 府省庁 |
| `ministry` | 事業 |
| `__agg-ministry` | 府省庁 |
| `project-budget` / `project-spending` | 支出先 |
| `__agg-project-*` | 事業 |
| `recipient` | 事業 |
| `__agg-recipient` | 支出先 |

## データソース

セクションのデータは `selectedNodeAllConnections` を再設計し、以下の構造に変える。

```typescript
type PanelSections = {
  ministries: { id: string; name: string; value: number }[];
  projects:   { id: string; name: string; value: number; ministry?: string; aggregated?: boolean }[];
  recipients: { id: string; name: string; value: number; aggregated?: boolean }[];
};
```

計算ロジックは `useMemo` として `page.tsx` 内に実装。  
集約ノードのメンバーは `filtered.aggNodeMembers` から取得。

### 集約ノードのメンバー表示について

`__agg-recipient` の支出先セクションは `filtered.aggNodeMembers.get('__agg-recipient')` をそのままリスト表示する。  
現行で「流入元タブ」に混在していた集約展開ロジックをここで自然に表現できる。

## 既存実装との対応

| 現行 | 新設計 |
|------|--------|
| `connectionTab: 'in' | 'out'` state | `panelTab: 'ministry' | 'project' | 'recipient'` に置き換え |
| `inDisplayCount` / `outDisplayCount` | 各セクション個別の `displayCount` に置き換え（セクションごとに state） |
| `collapsedMinistries` | そのまま転用（事業セクション内の省庁グルーピング用）|
| `ministryDisplayCounts` | そのまま転用 |
| `selectedNodeAllConnections` useMemo | `panelSections` useMemo に置き換え |
| 事業統計（total/ministryノードの件数表示） | 事業セクションのヘッダーに統合 |

## 事業ノード統合設計との関係

[事業ノード統合設計](./20260414_0752_事業ノード統合設計.md) と本設計は独立して実装可能。  
ただし事業ノード統合後は「事業セクション」に `project-budget` / `project-spending` の区別が不要になり、`project` 型として統一できる。  
事業ノード統合を先に実施する場合、本設計の「事業セクション」は `project` 型のみを扱えばよい。
