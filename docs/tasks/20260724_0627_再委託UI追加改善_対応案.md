# 再委託UI 追加改善 対応案

再委託UI刷新 Phase 1–4 完了後の追加改善要望（2026-07-24）への対応案。
「一気にやる必要はない」との方針のため、**小さな独立PRに分割**して優先度・依存を整理する。
既存の設計方針は [20260720_1011_再委託UI刷新_設計方針.md](20260720_1011_再委託UI刷新_設計方針.md)、
今後の課題は [20260722_0818](20260722_0818_間接経費のサンキー非表示問題とブロック連結可視化.md)。

## 要望一覧と現状・対応方針

### サイドパネル

#### S1. ヘッダー・事業概要・品質スコアをメイン画面と一致させる

- **現状**: 再委託パネルのヘッダは 事業名／予算／執行／タグ／主要経費＋RSリンク。
  メイン画面（/sankey-svg）は 予算額／支出額（1円単位併記）＋**事業概要アコーディオン**＋**品質スコアブロック**を持つが、
  再委託パネルには**事業概要・品質スコアが無い**。
- **データ**: `projectDetail`（`/api/project-details`）は再委託ページで既に取得済み → `overview` はfetch追加なしで使える。
  品質スコアは未取得（`/api/quality-scores/{pid}` の追加fetchが必要）。共有部品 `ScoreDetailDialog` は既存。
- **対応方針**:
  1. メイン画面の「品質スコアブロック」を `client/components/quality/` に**共有コンポーネント抽出**し、両ページで使う
     （メインは現状 page.tsx 内インライン。抽出すれば一致が保たれる ＝ 設計方針の共有プリミティブ路線）。
  2. 「事業概要」アコーディオンも同様に共有化 or 再委託側に同型実装（overview は取得済み）。
  3. ヘッダの数値表記（予算額／支出額・1円単位サブ表記）をメインの `StatRow` 相当に揃える。
- **規模**: 中。品質スコアの抽出＋fetch追加が主。**S1a=品質スコア**、**S1b=事業概要**、**S1c=ヘッダ数値表記** に分割可。
- **メモ**: 品質スコアブロック抽出はメイン画面にも影響する（インライン→コンポーネント差し替え）。回帰に注意。

#### S2. リストタブの「流れ」を「予算・執行」の右へ移動

- **現状**: タブ順は `予算・執行 / ブロック / 支出先 / 流れ / 間接経費`（既定タブ blocks、URL: bg/bl/rc/fl/ic）。
- **対応方針**: タブ配列を `予算・執行 / 流れ / ブロック / 支出先 / 間接経費` に並べ替え。
  URL短縮コード（fl 等）は不変なので**後方互換は保たれる**。既定タブ（blocks）は維持。
- **規模**: 小（tabs 配列の並べ替え1箇所）。**単独PRで即実施可**。

#### S3. 「流れ」に金額表示 ＋ 合流時は上流側を複数行に

- **現状**: `FlowListRow` は 起点タグ＋合流バッジ＋`起点ブロック → 対象ブロック` 名のみで**金額を出していない**。
- **データ**: `BlockEdge` に辺ごとの金額フィールドは無い。フローの金額は実質**対象ブロックの `totalAmount`**
  （そのブロックへ流入する額）で代表するのが妥当。合流（`targetIncomingBlockCount >= 2`）は
  1つの対象に複数の起点が流入する。
- **対応方針**:
  1. 各行に対象ブロックの金額（`formatYen(targetBlock.totalAmount)`）を表示。
  2. **合流のある対象は行を対象ごとにまとめ**、カード内で上流（矢印の左）を**複数行リスト**にする
     （例: `[B地方整備局 …] ← A本省 / ← J水資源機構 …`）。矢印右（対象）は1行、左（起点群）を縦積み。
  3. 非合流の辺は現行どおり1対1の1行。
- **規模**: 中。流れリストを「辺の列挙」から「対象ブロックでグルーピング（合流は起点を集約）」に組み替える。
  金額の意味づけ（辺の額ではなく対象総額）を UI 上で明確に（「流入額」等のラベル）。

### フロー図

#### F1. 事業ノードの左に「予算・執行」ノード列を追加

- **現状**: リボンフロー図の最左は事業（root）ノード。予算・執行の列は無い。
  （当初モック画面②では「一般会計 → 事業」を描いていたが本実装には未反映。）
- **データ**: `graph.budgetSummary`（totalBudget/executedAmount）・`budgetBreakdown` は API 取得済み（Phase 3で追加）。
- **対応方針**: リボンレイアウトに最左列「予算・執行」を追加し、緑ノード（予算総額／執行）から事業ノードへ1本のリボン。
  メイン画面の `総計→省庁→事業→支出先` と方向感を揃える（再委託図は `予算・執行→事業→ブロック→再委託ブロック`）。
- **規模**: 中。`subcontract-ribbon-layout` に列を1つ足し、ノード・リボン描画とズーム範囲（fit計算）に反映。
- **メモ**: 会計区分（一般/特別）で分けるか総額1本かは要判断。まず総額1本で足場を作るのが安全。

#### F2. ズームのインタラクション・ラベル構成をメイン画面に合わせる

- **現状**: 再委託図は独自のズーム/パン（`transform`/`baseZoom`/`scrollMode`、wheelハンドラ）。
  メインは `ZOOM_MIN/MAX_*`、ズーム連動フォント拡大（`ZOOM_FONT_MAX_RATIO`）、ミニマップ（`MinimapOverlay`）を持つ。
  ラベル構成（切り詰め規則・金額tspan・列見出し）も細部が異なる。
- **対応方針**: 段階的に寄せる。
  - **F2a**: ズーム上下限・ホイール挙動・スクロールモードの定数/作法をメインと統一（`ZOOM_*` 相当を共有定数化）。
  - **F2b**: ズーム連動フォント拡大をメインの `zoomFontRatio` 相当に合わせる。
  - **F2c**: ラベル構成（列見出し・名前切り詰め・金額tspan）をメインの流儀に合わせる。
  - **F2d（任意）**: ミニマップ導入。
- **規模**: 中〜大。ズーム機構は両者で作り込みがあり、完全一致は工数大。共有化できる定数/ユーティリティは
  `app/lib` に切り出す価値あり。**F2a→F2b→F2c** の順で小さく。

#### F3. hover ポップアップのレイアウト・スタイルをメイン画面に合わせる

- **現状**: 再委託図は独自のホバーツールチップ（マウス追従 div、淡色背景）。
  メインは `hoveredLinkStable` ベースのツールチップで独自レイアウト/スタイル。
- **対応方針**: メインのツールチップの見た目（角丸・影・行構成・タイポ・意味色の出し方）に再委託側を寄せる。
  可能なら**ツールチップ表示部を共有コンポーネント化**して両者で使う。
- **規模**: 小〜中（既存ツールチップの再スタイル）。共有化まで踏むと中。

## 推奨実施順（小さいPRから）

依存が薄く効果が見えやすい順。各行 = 1PR目安。

| # | 項目 | 規模 | 依存 | 効果 |
|---|------|------|------|------|
| 1 | **S2** タブ並べ替え（流れを2番目へ） | 小 | なし | 即・低リスク |
| 2 | **F3** hoverツールチップをメインに合わせる | 小〜中 | なし | 見た目の一致感 |
| 3 | **S3** 流れの金額表示＋合流の上流複数行 | 中 | なし | 情報量・視認性 |
| 4 | **F1** 予算・執行ノード列を追加 | 中 | なし（データ済） | 金の全体像が図で閉じる |
| 5 | **S1a** 品質スコアブロック共有化＋再委託パネルに追加 | 中 | メイン抽出 | メインとの一致 |
| 6 | **S1b/S1c** 事業概要・ヘッダ数値表記の一致 | 中 | S1a と同系 | メインとの一致 |
| 7 | **F2a→F2b→F2c** ズーム/ラベルの段階的一致 | 中〜大 | なし | 操作感の一致（仕上げ） |

- **まず 1–2 の軽量2件**で体感を上げ、**3–4 で情報設計**、**5–6 でメイン一致（共有部品化）**、**7 で操作感の仕上げ**。
- 5–7 は共有コンポーネント/共有ズームユーティリティの抽出を伴うため、Phase 1残の「共有プリミティブ路線」の延長として設計すると一貫する。

## データ・技術メモ（実装時の要点）

- 品質スコア: 再委託ページに `/api/quality-scores/{pid}` の遅延fetchを追加（メインの `useProjectPidCache` 相当）。
- 事業概要: `projectDetail.overview` は取得済み。fetch追加不要。
- 流れの金額: 辺ごとの額はデータに無い → 対象ブロック `totalAmount` を「流入額」として表示（意味を明記）。
- 予算・執行列: `graph.budgetSummary` を使用。会計区分分割は将来拡張、初手は総額1本。
- ズーム/ツールチップ共有化: `ZOOM_*` 定数・`zoomFontRatio`・ツールチップ表示部を `app/lib`／`client/components` へ抽出すると、
  以後メイン・再委託の乖離が再発しにくい（Phase 1 で semantic-colors / TagChip を共有化したのと同じ方針）。
