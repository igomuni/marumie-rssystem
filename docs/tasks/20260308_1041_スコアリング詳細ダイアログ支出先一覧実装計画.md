# スコアリング詳細ダイアログ 支出先一覧実装計画

## 目的

分析者がスコアリング詳細ダイアログで、事業に紐づく支出先の一覧とそれぞれのスコアリング根拠（辞書判定・CN記入・不透明フラグ・金額・ブロック位置）を確認できるようにする。

---

## 現状と課題

`project-quality-scores.json` が持つのは**事業単位の集計値**のみ。
支出先ごとのスコアリング根拠（どの行がvalid/invalid/不透明か）はダイアログで表示できない。

スコアリングスクリプト `score-project-quality.py` では 5-1 CSVを行単位で処理しているため、
各行のスコアリング結果をそのまま収集して出力すれば、追加の計算なしに実現できる。

---

## 設計

### 新規出力ファイル: `public/data/project-quality-recipients.json`

スコアリングスクリプトの 5-1 CSV処理ループ内で、各支出先行のスコアリング結果を収集する。

```
{
  "1001": [
    { "name": "株式会社ABC", "blockNo": "A", "status": "valid",   "cnFilled": true,  "opaque": false, "amount": 5000000, "isRoot": true  },
    { "name": "日田市",      "blockNo": "A", "status": "invalid", "cnFilled": false, "opaque": false, "amount": 200000,  "isRoot": true  },
    { "name": "その他",      "blockNo": "B", "status": "unknown", "cnFilled": false, "opaque": true,  "amount": 100000,  "isRoot": false }
  ],
  "1002": [ ... ]
}
```

#### `status` の値定義

| 値 | 条件 | 軸1への影響 |
|----|------|------------|
| `"valid"` | 厳密辞書に存在し `valid=True` | valid_count++ |
| `"gov"` | 厳密辞書で invalid、行政機関辞書にヒット | gov_agency_count++ |
| `"supp"` | 厳密辞書で invalid、補助辞書にヒット | supp_valid_count++ |
| `"invalid"` | 厳密辞書で invalid、どの辞書にもヒットしない | invalid_count++ |
| `"unknown"` | 厳密辞書自体に存在しない（カウント対象外） | ± 0 |

#### ファイルサイズ試算

- 対象: 5-1 CSV の全支出先行（`支出先名` が空のブロックヘッダー行は除外）
- 総行数: 5,664事業 × 平均 rowCount ≈ 数万〜十数万行
- 行あたり: 約 100〜150 バイト
- 推定: **5〜10 MB**（現行 `project-quality-scores.json` 4.2MB と同規模）

---

### 新規 API: `app/api/quality-scores/recipients/route.ts`

`GET /api/quality-scores/recipients?pid=1001`

- `project-quality-recipients.json` をメモリキャッシュ（既存 API と同パターン）
- クエリパラメータ `pid` で1事業分のみ返す
- `pid` なしの場合は 400 エラー

```typescript
export interface RecipientRow {
  name: string;
  blockNo: string;
  status: 'valid' | 'gov' | 'supp' | 'invalid' | 'unknown';
  cnFilled: boolean;
  opaque: boolean;
  amount: number;
  isRoot: boolean;
}
```

---

### UI: `ScoreDetailDialog` への追加（`app/quality/page.tsx`）

ダイアログの既存軸スコアセクションの下に「支出先一覧」セクションを追加する。

#### 動作フロー

1. ダイアログが開いた時（`useEffect([item.pid])`）に `/api/quality-scores/recipients?pid=XXX` をフェッチ
2. ローディング中はスピナー表示
3. 取得後、支出先テーブルを表示

#### テーブル列構成

| 列 | 内容 |
|----|------|
| 支出先名 | テキスト（長い場合は省略） |
| ブロック | blockNo（ルートブロックは強調） |
| 軸1判定 | status バッジ（色分け） |
| CN | 記入あり / なし |
| 透明性 | 不透明フラグ（マッチした場合のみ表示） |
| 金額 | formatAmount 形式 |

#### status バッジの色

| status | ラベル | 色 |
|--------|--------|----|
| `valid` | 厳密OK | 緑 |
| `gov` | 行政機関 | 緑（薄） |
| `supp` | 補助辞書 | 青 |
| `invalid` | 不一致 | 赤 |
| `unknown` | 未登録 | グレー |

#### 表示制御

- デフォルトはブロック番号順、同ブロック内は金額降順でソート
- 件数が多い場合（> 50件）はページネーション or スクロール（ダイアログ内スクロール）
