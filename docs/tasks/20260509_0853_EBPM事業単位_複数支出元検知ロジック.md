# EBPM事業単位 複数支出元検知ロジック

調査日: 2026-05-09  
実装: `scripts/analyze-multiple-project-funding-sources.ts`

## 目的

EBPMで事業単位に予算、執行、支出、再委託構造を確認するとき、事業の支出元が単一の府省庁予算だけではないケースを CSV から検知する。

典型例:

- JICA有償資金協力: `一般会計出資金` に加えて `財政融資資金借入金`、`回収金等` が合流
- 防災科研運営費交付金: `運営費交付金` に加えて `自己収入等` が合流
- 水資源開発事業: 国交省予算に加えて `利水者等` からの負担金・借入金等が合流

## 前提

5-2 の明示ルートは、次の行である。

```text
支出元の支出先ブロック = 空
担当組織からの支出 = TRUE
支出先の支出先ブロック = 非空
```

これは通常、府省庁から直接支出されたブロックを表す。

一方で、5-2 には次のようなブロックもある。

```text
支出元として登場する
しかし、どの支出先ブロックとしても登場しない
かつ、担当組織からの直接支出ブロックでもない
```

このブロックはグラフ上の「別起点」と見なせる。  
さらに 5-1 に金額がある場合、sankey-svg などでは通常の支出先として集計されてしまう可能性がある。

## 検知ロジック

事業ごとに 5-2 をグラフとして見る。

1. `targetBlocks` を作る
   - 5-2 の `支出先の支出先ブロック`
2. `directTargetBlocks` を作る
   - `支出元の支出先ブロック` が空
   - `担当組織からの支出=TRUE`
   - `支出先の支出先ブロック` が非空
3. `sourceBlocks` を作る
   - 5-2 の `支出元の支出先ブロック`
   - 空は除外
4. `sourceBlocks` のうち、以下を満たすものを `separate-origin` とする
   - `targetBlocks` に含まれない
   - `directTargetBlocks` に含まれない
5. 5-1 と突合する
   - ブロックサマリ行から `ブロック名`, `役割`, `ブロックの合計支出額`
   - 支出先行から `支出先名`, `支出先の合計支出額`
6. 金額があるものを候補として出力する

加えて、より強い別起点候補として「合流支出元でもある」ことを確認する。

```text
候補ブロックが支出元として流れる先に、
別の支出元ブロックも同時に流入している
```

これは `sourceFeedsMerge=true` として出力する。  
単に上流ルートが欠落しているだけの可能性があるケースは、この条件では落ちやすい。たとえば `2776 障害者自立支援給付` は `都道府県 -> 市区町村` の単独ルートであり、合流支出元ではない。一方、JICA有償資金協力の `財政融資資金借入金`、`回収金等` は、`一般会計出資金` とともに `独立行政法人国際協力機構` ブロックへ合流するため、この条件を満たす。

疑似コード:

```ts
for each project:
  targetBlocks = set(5-2.targetBlock)
  directTargetBlocks = set(5-2 rows where sourceBlock is empty and direct=true).targetBlock
  sourceBlocks = set(5-2.sourceBlock excluding empty)

  for sourceBlock of sourceBlocks:
    if !targetBlocks.has(sourceBlock) && !directTargetBlocks.has(sourceBlock):
      if 5-1 has amount for sourceBlock:
        emit separate-origin candidate
```

## 分類

候補のうち、ブロック名・役割・支出先名・補足情報に以下を含むものは、財源・収入・借入・回収・移替系として扱う。

| 分類 | キーワード |
|---|---|
| 借入・財投・保証 | `財政`, `財投`, `融資資金`, `借入`, `政府保証` |
| 回収・償還 | `回収金`, `償還` |
| 自己収入・その他収入 | `自己収入`, `収入等`, `受講料収入`, `手数料収入`, `その他収入` |
| 負担金 | `負担金`, `利水者` |
| 移替 | `移替` |
| 繰入・債務 | `繰入`, `交付金債務` |

該当しない候補は `経由主体・実施主体など` として残す。  
これは都道府県、管区警察局、地方公共団体など、事業実施上の経由主体も検知対象に残すため。

### 構造条件に当てはまる全候補

以下はキーワード分類で絞らず、構造条件だけで検知した全候補。`階層=0` は、5-2上で担当組織からの明示ルートが見つからないことを表す。そのため `2776 障害者自立支援給付` のようなケースは、真の別起点ではなく、ルートレコード欠落または記述省略の可能性がある。`分岐元数` は事業内で2つ以上の下流ブロックを持つ支出元ブロック数、`最大分岐` は1ブロックから出る下流ブロック数の最大値、`候補下流数` は検知された候補ブロック自身の下流ブロック数。

合流も同時に集計する。ここでの合流は、5-2上で「複数の支出元ブロックから同じ支出先ブロックへ入る」構造を指す。出力CSVには `projectMergingBlockCount`、`projectMaxMergeWidth`、`sourceIncomingBlockCount` を追加した。たとえば `1406 独立行政法人国際協力機構有償資金協力部門への出資` は `projectMergingBlockCount=1`、`projectMaxMergeWidth=3` で、複数の起点が `独立行政法人国際協力機構` ブロックへ合流している。

## UI・データ生成への含意

`sankey-svg` の通常支出先列にこれらを混ぜると、次が区別できなくなる。

- 本当の支出先
- 事業内の別財源
- 自己収入
- 借入金・回収金
- 予算移替
- 経由主体・実施主体

EBPM向けには、少なくとも `separate-origin` 候補を「複数支出元あり」として事業にフラグ付けし、詳細画面で `通常支出先` と分離して表示するのがよい。
