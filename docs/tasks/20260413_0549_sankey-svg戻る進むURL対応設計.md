# /sankey-svg 戻る・進む・URL対応 設計

## 目的

ユーザーがノードをクリックして探索した経路（事業 → 支出先 → 省庁など）をブラウザの戻る・進むボタンで辿り直せるようにするため。また、URL を共有することで同じ表示状態を他者に伝えられるようにするため。

---

## 現状

`app/sankey-svg/page.tsx` は全ステートを `useState` で管理しており、URL との連携はない。
ブラウザの戻る・進むボタンは機能せず、URL を共有しても初期状態のページが開く。

---

## URL パラメータ設計

### 対象ステート

| パラメータ名 | ステート | デフォルト値 | 備考 |
|-------------|---------|------------|------|
| `sel` | `selectedNodeId` | `null` | 選択中ノードID |
| `pp` | `pinnedProjectId` | `null` | ピン留め事業ID |
| `pr` | `pinnedRecipientId` | `null` | ピン留め支出先ID |
| `pm` | `pinnedMinistryName` | `null` | ピン留め省庁名 |
| `ro` | `recipientOffset` | `0` | 支出先オフセット |
| `tm` | `topMinistry` | `37` | 省庁TopN |
| `tp` | `topProject` | `40` | 事業TopN |
| `tr` | `topRecipient` | `40` | 支出先TopN |
| `bz` | `baseZoom` | `1` | 基準ズーム倍率 |
| `sl` | `showLabels` | `true` | ラベル表示 |
| `iz` | `includeZeroSpending` | `false` | 支出0円事業表示 |
| `ar` | `showAggRecipient` | `true` | 集約支出先表示 |
| `sb` | `scaleBudgetToVisible` | `true` | 予算をウィンドウ内支出にスケール |
| `fr` | `focusRelated` | `true` | 関連ノードのみ表示 |

**シリアライズ方針**: デフォルト値と同じ場合はパラメータを省略する。

### 対象外ステート（URL に含めない）

| ステート | 理由 |
|---------|------|
| `zoom`, `pan` | ビューポート位置は共有に不向き。ズームは `baseZoom` のみ対象 |
| `showSettings` | 一時的なUI状態 |
| `hoveredNode`, `hoveredLink` | ホバー状態はセッション固有 |
| `searchQuery` | 一時的な検索状態 |
| `isPanelCollapsed` | UI表示状態 |
| `inDisplayCount`, `outDisplayCount` 等 | サイドパネル内の一時状態 |

## 履歴エントリの設計

### `pushState` を使うケース（「ナビゲーション」として記録）

ユーザーが意図的に探索の方向を変えた操作：

- ノードをクリックして `selectedNodeId` / `pinnedProjectId` / `pinnedRecipientId` / `pinnedMinistryName` が変化したとき

### `replaceState` を使うケース（現在のエントリを上書き）

頻繁に変化する操作や、バック操作の対象にしたくない操作：

- スライダー操作（`topMinistry`, `topProject`, `topRecipient`, `recipientOffset`, `baseZoom`）
- トグルオプション（`showLabels`, `includeZeroSpending`, `showAggRecipient`, `scaleBudgetToVisible`, `focusRelated`）
- ノード選択解除（`selectedNodeId = null` への変化）

## 実装方針

### 初期化（URLからの読み込み）

`useEffect`（deps: `[]`）で `window.location.search` をパースし、各ステートの `useState` 初期値を上書きする。

`useState` の初期値をそのまま URL から読み込む形にはせず、**まず `useState` のデフォルト値で初期化し、その後 `useEffect` で URL パラメータを適用する**。これにより SSR/ハイドレーション上の不整合を回避する。

### URL 更新

URL 更新は `syncUrl(push: boolean)` のような内部関数にまとめる。

```
syncUrl(push)
  → 現在のすべてのURL対象ステートをシリアライズ
  → デフォルト値と同値のパラメータは削除
  → push=true なら history.pushState、false なら history.replaceState
```

`syncUrl` は `useCallback` でメモ化し、URL対象ステートすべてを deps に含める。

### 戻る・進む対応（popstate）

`useEffect` で `window.addEventListener('popstate', handler)` を登録する。
`handler` は `window.location.search` をパースして各ステートを一括で復元する。

パース → ステート適用ロジックは初期化と共通化する（`parseSearchParams(search: string): Partial<SankeyUrlState>` のような純粋関数として分離）。

### ステート変更時の URL 同期

各ステートを変更する既存のセッター呼び出し箇所に `syncUrl` 呼び出しを追加する。

ノードクリック（`handleNodeClick`）では `syncUrl(true)`、
スライダー・トグル変更では `syncUrl(false)` を呼ぶ。

## 変更スコープ

### 変更ファイル: `app/sankey-svg/page.tsx`

- `parseSearchParams` 純粋関数の追加（ファイル先頭付近）
- `useState` 初期値はデフォルト値のまま維持
- 初期化 `useEffect` の追加（URL → ステート読み込み）
- `syncUrl` コールバックの追加
- `popstate` 購読 `useEffect` の追加
- `handleNodeClick` に `syncUrl(true)` 追加
- スライダー・トグルのセッター箇所に `syncUrl(false)` 追加

### 変更なしのファイル

- `app/lib/sankey-svg-filter.ts`
- `app/lib/sankey-svg-constants.ts`
- `types/sankey-svg.ts`
