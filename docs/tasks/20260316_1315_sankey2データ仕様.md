## 5,664件 vs 5,003件の差分

### 原因

| PID集合 | 件数 | 説明 |
|---------|------|------|
| 全事業（1-1 CSV） | 5,664 | 組織情報に登録された全事業 |
| 予算年度2023あり | 5,003 | 2-1 CSVに予算年度2023のレコードがある事業 |
| **差分** | **661** | 予算年度2023のデータがない事業 |

661件は2-1 CSVに予算年度2023の行が存在しない事業。2021/2022/2024年度のデータのみ保有。

### 設計上の意味

既存パイプライン（5,003件）では661件の事業に紐づく**1,571支出先・1,469億円が孤児データ**となっていた。`/sankey2` では1-1 CSVの全5,664事業を起点にすることで、支出先の紐づけが完全になる。

## CSV間のPID整合性

| クロスリファレンス | 件数 |
|-------------------|------|
| 1-1にあり 2-1(全年度)にない | 0 |
| 1-1にあり 5-1にない | 0 |
| 5-1にあり 1-1にない（孤児支出） | 0 |
| 2-1にあり 1-1にない（孤児予算） | 0 |

**3つのCSVのPID集合は完全一致**（5,664件）。データの整合性に問題なし。

## 支出先の一意キー

現行パイプラインでは `支出先名 + 法人番号` を一意キーとしていたが、`/sankey2` では **`支出先名`のみ**を一意キーとしている。

| キー方式 | ユニーク数 | 備考 |
|---------|-----------|------|
| 支出先名のみ | 25,892 | sankey2（現行） |
| 支出先名 + 法人番号 | 26,823 | 既存パイプライン |
| 基準値（no_total_stats.txt） | 26,192 | 合計支出額なし行のみ |

差分の931件（26,823 - 25,892）は、同じ支出先名で異なる法人番号が付与されているケース（例: 「その他」が複数の法人番号と紐づく）。
