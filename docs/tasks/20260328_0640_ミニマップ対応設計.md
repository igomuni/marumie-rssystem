# ミニマップ対応設計

## 目的

ユーザーが `/sankey-lab/real` でズーム操作時に、全体の中で現在どこを見ているかを把握し、素早く別の領域にナビゲートできるようにするため。

現状はズームインすると全体像を見失い、見たい領域への移動がドラッグ操作頼りになっている。

## 現状

| 項目 | 値 |
|------|-----|
| SVG描画領域 | `svgWidth`（コンテナ幅、動的） × `SVG_H`（800px固定） |
| zoom範囲 | 0.2x 〜 50x |
| pan | 制限なし（自由ドラッグ） |
| zoom/panの管理 | `useState` で `zoom`, `pan: {x, y}` を管理 |
| SVG transform | `translate(${pan.x},${pan.y}) scale(${zoom})` |

## 設計

### 配置

ミニマップは描画エリアの**左下**に `position: absolute` で配置する。コントロールバー（右上）と干渉しない位置。

### サイズ

全体図のアスペクト比を維持した固定サイズ:
- 幅: 200px
- 高さ: `200 * (SVG_H / svgWidth)` px（アスペクト比維持、概ね100〜140px程度）

### 描画内容

Canvas要素で軽量に描画する。ノードのみ（リンクは省略）。

| 要素 | 描画 |
|------|------|
| ノード（通常） | ノードカラーで塗りつぶした矩形 |
| ノード（集約） | グレーで塗りつぶした矩形 |
| リンク | 描画しない（ミニマップでは視認性が低く、描画コストが高い） |
| ビューポート矩形 | 現在の表示領域を半透明の青枠で表示 |

### ビューポート矩形の計算

現在の表示領域をミニマップ上の矩形として描画する:

```text
ミニマップのスケール = minimapWidth / svgWidth

ビューポート左上（ミニマップ座標）:
  x = -pan.x / zoom * scale
  y = -pan.y / zoom * scale

ビューポートサイズ（ミニマップ座標）:
  w = svgWidth / zoom * scale
  h = SVG_H / zoom * scale
```

### インタラクション

| 操作 | 動作 |
|------|------|
| ミニマップ上でクリック | クリック位置が表示領域の中心になるようにpanを更新 |
| ミニマップ上でドラッグ | ビューポート矩形をドラッグしてpanをリアルタイム更新 |
| メインエリアでzoom/pan | ミニマップのビューポート矩形が連動更新 |

クリック/ドラッグ時のpan計算:

```text
クリック位置（ミニマップ座標）: mx, my
対応するSVG座標: sx = mx / scale, sy = my / scale
新しいpan: { x: svgWidth/2 - sx * zoom, y: SVG_H/2 - sy * zoom }
```

### 表示/非表示

- zoom === 1 かつ pan === {0, 0}（初期状態）のときは非表示
- zoom > 1 またはpanが初期値から変わったときに表示
- フェードイン/アウトで切り替え

### 実装方針

`<canvas>` 要素を使用。理由:
- ノード数が最大数十件（filterTopN後）なので描画は軽量
- SVGネストより実装がシンプル
- ビューポート矩形の描画が容易

レイアウト計算済みの `layout.nodes` を使ってミニマップ用の座標を算出する。メインのレイアウトエンジンとは別にミニマップ用の再計算は不要（スケーリングのみ）。
