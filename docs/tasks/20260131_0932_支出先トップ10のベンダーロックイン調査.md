# 支出先トップ10のベンダーロックイン調査報告

**調査日**: 2026年1月31日
**調査対象**: 令和6年度（2024年度）行政事業レビューデータにおける民間受託額上位企業
**目的**: 策定した調査フローに基づき、特定ベンダーへの依存（ベンダーロックイン）が疑われる事例を抽出・分析する。

---

## 1. 調査結果サマリ

データ分析の結果、民間受託額上位の企業において、以下の3つのパターンで顕著なベンダーロックイン（またはそれに準ずる独占的状態）が確認されました。

1.  **ITシステム・インフラ型**: 既存システムとの互換性や技術的難易度を理由に、特定のITベンダーが継続受注。
2.  **防衛・戦略装備型**: 装備品の開発・製造において、特定の重工・電機メーカーが100%近い占有率を保持。
3.  **事務局・プラットフォーム型**: 大規模な補助金事業において、広告代理店や印刷大手が事務局・システム運営を一括受託。

---

## 2. 主要な調査事例

### 2.1 富士通株式会社（ITシステム・インフラ型）
最も典型的なITベンダーロックインの事例。

- **対象事業**: ハローワークシステム運営費（ID: 2519）
- **占有率**: **80.7%**（支出額 641億円 / 事業総額 795億円）
- **契約の特徴**: 1者応札。
- **ロックイン要因**: 
  - 「既存システムの改修であり規模が大きく、難易度が高く確実な履行ができない」という他者の入札辞退。
  - 既存の「互換性」を維持する必要性。
- **課題**: 大規模システムの一括発注が、他社の参入障壁となっている。

### 2.2 三菱重工業株式会社（防衛・戦略装備型）
国家的戦略に基づく長期的な独占状態。

- **対象事業**: 地対艦誘導弾の取得（ID: 5172）、島嶼防衛用高速滑空弾の取得（ID: 7539）
- **占有率**: **99.9% 〜 100%**
- **契約の特徴**: 事実上の独占。
- **ロックイン要因**: 装備品固有の技術、開発からの継続性。
- **考察**: 安全保障上の理由もあり、代替ベンダーの確保が極めて困難な構造。

### 2.3 株式会社博報堂（事務局・プラットフォーム型）
大規模な緊急対策事業における運営依存。

- **対象事業**: 電気・ガス価格激変緩和対策等事業（ID: 7259）
- **受託額**: **2.37兆円**（※補助金原資を含む事務局受託）
- **占有率**: **46.9%**
- **契約の特徴**: 入札者2社。
- **考察**: 数兆円規模の資金執行を短期間で構築・運営できる能力が、特定の大手広告代理店に限定されていることによるロックイン。

### 2.4 大日本印刷株式会社（事務局・プラットフォーム型）
住宅支援事業等の大規模事務局を継続的に受託。

- **対象事業**: こどもエコすまい支援事業（ID: 7266）
- **占有率**: **50.0%**（支出額 1,500億円 / 事業総額 3,000億円）
- **考察**: 「住宅・建築物カーボンニュートラル総合推進事業」等、類似の住宅支援スキームにおいて継続的にプラットフォームを運営。

### 2.5 アクセンチュア株式会社（コンサル・IT統合型）
特定技術領域での高度な依存。

- **対象事業**: 情報提供等記録開示システム（ID: 6546）
- **占有率**: **90.4%**
- **契約の特徴**: 1者応札。
- **ロックイン要因**: 「役務規模が大きく、支援対象の情報システム数が多すぎる」ことによる他社の敬遠。

---

## 3. 分析と提言

### 3.1 ロックインの共通要因
1.  **規模の障壁**: 案件が巨大すぎて、リスクや人員確保の面から他社が参入できない（博報堂、アクセンチュア）。
2.  **技術のブラックボックス化**: 既存システム（Legacy）の仕様が現受託者にしか詳細不明となっている（富士通、NTTデータ、日立）。
3.  **資産の固定化**: 国庫債務負担行為（複数年度契約）や、既に導入済みのハードウェア・ライセンスへの依存。

### 3.2 調査フローに基づく評価と今後のアクション

| ベンダー | 評価 | 今後のアクション |
| :--- | :--- | :--- |
| **富士通** | **高 (Critical)** | モジュール化による分割発注の徹底調査 |
| **博報堂** | **高 (Critical)** | 事務局経費の内訳と再委託先の透明化調査 |
| **三菱重工** | **特例 (Strategic)** | 装備品ライフサイクルコストの妥当性検証 |
| **アクセンチュア** | **中 (Warning)** | 業務領域の整理と内製化・分離可能性の検討 |
