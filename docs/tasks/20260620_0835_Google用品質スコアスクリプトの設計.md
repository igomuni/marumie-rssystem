# Google AI Studio 用 品質スコア検証スクリプトの設計

**日時**: 2026-06-20 08:35 (JST)
**対象**: `/quality` の AIスコアリングを **Google AI Studio（無料枠の実 Gemini）**で深掘り検証するための、独立スクリプトの設計
**前提**: 既存 `scripts/score-project-quality-ai.py`（OpenRouter版）は**一切変更しない**。Google向けは別ファイルとして新設する。

---

## 1. 目的（Why）

開発者が、**$0 の Google AI Studio 無料枠で「本物の Gemini」を使った2024スコアの深掘り検証**（プレビュー生成・モデル世代比較）を、既存の本番スクリプトを壊さずに行えるようにするため。

背景（既出の検証で判明）:

- OpenRouter の無料モデルは中央寄せが強く、`gemini-3.5-flash`(2025本番)の代替にならない（`20260620_0650` §5）。
- 一方 **Google無料枠の実 `gemini-2.5-flash` は無料勢で最も 3.5-flash に近い**（MAE 1.28）。
- ただし既存スクリプトを `--base-url` でGoogleに向けても、**`reasoning` フィールド非互換で 400 → 全件 heuristic に落ちる**（`20260620_0650` §5-4 で確認）。`aiSource` ラベルも `openrouter:` 固定で実態と食い違う。

→ Google で実 Gemini を回すには、既存スクリプトに数点の差分が要る。本番スクリプトの安定を守るため、**変更ではなく別スクリプトで分離**する。

---

## 2. 設計方針

### 2-1. 配置と命名

- 新規: `scripts/score-project-quality-ai-google.py`（既存 `...-ai.py` の隣）。
- 既存 OpenRouter 版は**そのまま温存**（本番・2025生成の再現性を保持）。

### 2-2. 採点ロジックは本番と同一に保つ

検証の目的は「Google無料Geminiの判定品質を本番と比較すること」。よって**プロンプト・レベル定義・金額加重集計・軸C/D機械計算・総合重み（A28/B22/C15/E35）・出力スキーマは既存と完全一致**させる。Google対応のための差分は **API接続まわりだけ**に限定する。

### 2-3. コード重複の扱い（トレードオフ）

既存スクリプトはモジュール先頭で `argparse` を実行するため `import` 再利用ができない。「既存は不変」という制約下では選択肢は次の2つ:

| 案 | 内容 | 評価 |
|----|------|------|
| **A. 派生コピー**（推奨） | 既存をコピーし §3 の差分のみ適用した独立スクリプト | 単純・低リスク。採点ロジックの将来分岐に注意（コメントで「本番と同期」を明記） |
| B. 共通コア抽出 | 採点コアを `scripts/lib/quality_core.py` に切り出し、新スクリプトのみ import | 重複は消えるが、既存スクリプトを「触らない」制約と両立しづらく、リスク増 |

→ 検証用途・短期の深掘りという性格から、**A（派生コピー）**を採る。差分は §3 の6点に閉じるため、レビューで同期確認が容易。

## 3. 既存スクリプトからの差分（6点）

既存 `score-project-quality-ai.py` の該当行に対する Google 版の変更点。

| # | 箇所（既存の行） | 既存（OpenRouter） | Google版 |
|---|------------------|--------------------|----------|
| 1 | APIキー env（L214, L525） | `OPENROUTER_API_KEY` | **`GOOGLE_API_KEY`**（未設定時は従来どおり heuristic 完走） |
| 2 | `--base-url` 既定（L54） | `https://openrouter.ai/api/v1` | **`https://generativelanguage.googleapis.com/v1beta/openai/`** |
| 3 | `--model` 既定（L52） | `google/gemini-3.5-flash` | **`gemini-2.5-flash`**（Googleネイティブ表記・`google/`接頭辞なし） |
| 4 | thinking制御（L231） | `extra_body={'reasoning':{'effort':'low'}}` | **送らない**（400回避）。コスト制御が要るなら Google 流の thinking_config を使用（§3-1） |
| 5 | `aiSource` ラベル（L534, L674, docstring L26） | `openrouter:<model>` | **`google:<model>`**（実態反映） |
| 6 | キャッシュ名（L83-84） | モデル末尾セグメント（`gemini-2.5-flash`） | **`google-` 接頭辞で分離**（例 `ai-quality-cache-google-gemini-2.5-flash.json`）。OpenRouter版の壊れた gemini キャッシュ（gate のみ）との混在を防ぐ |

### 3-1. thinking（特殊対応のメモ）

Google の OpenAI互換層は OpenRouter の `reasoning` フィールドを知らない（`400 Unknown name "reasoning"`）。対応は2択:

- **既定: 何も送らない**。`gemini-2.5-flash` は動的 thinking で判定（無料枠なのでコスト問題なし）。本検証の head-to-head はこの方式で実 Gemini 判定を取得済み。
- **任意: thinking を抑制したい場合**のみ、Google 流のネスト指定を使う（OpenAIの`reasoning`ではない）:

```python
# 例: thinking を最小化したいときだけ
extra_body={'extra_body': {'google': {'thinking_config': {'thinking_budget': 0}}}}
```

検証目的では既定（送らない）で十分。

### 3-2. レート制限（無料枠運用）への配慮

Google 無料枠は分間（RPM/TPM）＋**1日あたり（RPD）**制限がある。既存の「指数バックオフ4回リトライ＋永続キャッシュ＋逐次フラッシュ」をそのまま活かしつつ、Google版では:

- 既定 `--concurrency` を低め（例 2〜3）に。
- 全件（軸A/B 約4,383req＋軸E 約706req）は**RPDを跨ぐ前提**。キャッシュにより**日跨ぎで分割再開**可能（成功分のみ永続化、heuristic補完は非永続なので翌日実Geminiで穴埋めされる）。
- 429 が続く場合は concurrency をさらに下げる運用を docstring に明記。

## 4. このスクリプトで可能になる深掘り検証

本スクリプトが入ると、$0 で以下が回せる（いずれも既出の比較手法の延長）:

- **2024 のプレビュー全件生成**（実 `gemini-2.5-flash`、日跨ぎ・再開可）。本番 3.5-flash 採用前に、実Gemini品質の分布を2024で確認。
- **モデル世代比較の拡張**: `gemini-2.5-flash` vs `gemini-2.5-flash-lite`（ともに無料枠候補）を同一サンプルで比較し、コスト／品質のトレードオフを把握。
- **head-to-head の標本拡大**: 軸E の n を増やし、`20260620_0650` の MAE/バイアスを安定推定。

> ⚠️ これらは検証であり、**確定スコア（2025と基準一致）は引き続き有料 `gemini-3.5-flash`**（`20260620_0636`）。Google無料 2.5-flash は「最も近い無料近似」の位置づけ。

## 5. 留意点

- **採点ロジックの同期**: 派生コピーのため、本番スクリプトの採点ロジックを変えたら Google 版にも反映する（先頭コメントで明示）。差分は §3 の API 周りに限定し、それ以外は触らない。
- **`aiSource` の区別**: 出力に `google:<model>` を付け、本番（`openrouter:google/gemini-3.5-flash`）や heuristic と取り違えないようにする。Google無料 2.5-flash 生成物を**確定データとして混入させない**運用上の歯止めになる。
- **データ利用ポリシー**: Google無料枠は入力が製品改善に使われうる規約。対象は公開済み行政データのため実害小だが認識する。
- **キャッシュ分離**: §3-6 のとおり接頭辞で分け、OpenRouter版の不完全 gemini キャッシュと混ぜない。
