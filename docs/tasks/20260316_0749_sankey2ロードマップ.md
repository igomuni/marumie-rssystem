# /sankey2 実装ロードマップ

設計ドキュメント: [20260316_0711_sankey2設計.md](20260316_0711_sankey2設計.md)

## Phase 1: データパイプライン

元CSV `data/year_2024/` から `/sankey2` 用のグラフデータを生成する。

| タスク | 内容 |
|--------|------|
| 1-1 | `scripts/generate-sankey2-data.ts` 新規作成 |
| 1-2 | `1-1` CSV → 府省庁ノード生成（33府省庁） |
| 1-3 | `2-1` CSV → 事業ノード生成（予算額・執行額付き、全事業対象） |
| 1-4 | `5-1` CSV → 支出先ノード生成（合計行除外、ユニーク支出先名） |
| 1-5 | 府省庁→事業、事業→支出先のエッジ生成 |
| 1-6 | `public/data/sankey2-graph.json.gz` 出力 + npm script 追加 |

**完了条件**: グラフJSONが生成され、ノード数・エッジ数・金額合計が `no_total_stats.txt` の基準値と整合すること。

## Phase 2: レイアウト計算

グラフデータから事前計算済みレイアウト（座標 + Bezierパス）を生成する。

| タスク | 内容 |
|--------|------|
| 2-1 | `app/lib/sankey2-layout.ts` 新規作成 |
| 2-2 | ノードのX座標計算（type別のベースライン配置） |
| 2-3 | ノードのY座標計算（府省庁セクション分割、金額比例高さ） |
| 2-4 | エッジのBezierパス事前計算 |
| 2-5 | レイアウト結果を `sankey2-layout.json.gz` に出力 |

**完了条件**: 全ノードに座標が割り当てられ、エッジパスが計算済みであること。

## Phase 3: SVG描画 + Zoom/Pan

基本的なSVG描画とGoogle Maps風のZoom/Pan操作を実装する。

| タスク | 内容 |
|--------|------|
| 3-1 | `app/sankey2/page.tsx` 新規作成（ページ枠） |
| 3-2 | `client/components/Sankey2/` ディレクトリ新規作成 |
| 3-3 | SVGノード描画（矩形 + typeごとの色分け） |
| 3-4 | SVGエッジ描画（Bezierパス、金額比例の線幅） |
| 3-5 | Zoom（ホイール）+ Pan（ドラッグ）実装 |
| 3-6 | Zoom閾値によるラベル段階表示 |

**完了条件**: 全ノード・エッジがSVGで描画され、Zoom/Panで自由に探索できること。

## Phase 4: 表示数スライダー

パーセンテージスライダーでノードの表示・非表示を制御する。

| タスク | 内容 |
|--------|------|
| 4-1 | 事業スライダー（0〜100%、予算額降順で上位N%を表示） |
| 4-2 | 支出先スライダー（0〜100%、支出額降順で上位N%を表示） |
| 4-3 | 非表示分の「その他」ノード集約表示 |
| 4-4 | スライダー変更時のレイアウト再計算（座標の再配置） |

**完了条件**: スライダー操作でノード数が増減し、「その他」ノードに正しく集約されること。

## Phase 5: インタラクション

ホバー・クリック・ハイライトなどのユーザー操作を実装する。

| タスク | 内容 |
|--------|------|
| 5-1 | ノードホバー → 接続ノード・エッジのBFSハイライト |
| 5-2 | ノードクリック → 選択状態 + サイドパネル詳細表示 |
| 5-3 | 非接続ノードのフェードアウト |
| 5-4 | Minimap（全体俯瞰 + 現在ビューポート表示） |

**完了条件**: ノードの選択・ハイライトが動作し、関連フローが視覚的にたどれること。
