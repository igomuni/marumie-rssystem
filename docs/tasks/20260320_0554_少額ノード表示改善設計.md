# 少額ノード表示改善設計

## 目的

ユーザーがZoom 10000%まで拡大しても1000万円以下のノードが表示されない問題を解消し、高倍率ズーム時に少額ノードも探索できるようにする。

## 調査結果

### 根本原因: `NODE_GAP`によるゼロ面積化

`compute-sankey2-layout.ts` の `applyGap()` 関数が原因。

```
NODE_GAP = 1  // treemap内ノード間のギャップ(px)
```

`applyGap` は矩形の幅・高さからそれぞれ `NODE_GAP`(1px) を減算する:
```
width  = max(0, rect.width  - gap)   // 1px未満 → 0
height = max(0, rect.height - gap)   // 1px未満 → 0
```

少額ノードはtreemapで1px未満の幅・高さに配置されるため、gap適用後に **面積 = 0** となり、どれだけズームしても `screenArea = 0 * k² = 0 < MIN_SCREEN_AREA(1)` で非表示になる。

### 影響範囲

| 指標 | 値 |
|------|-----|
| amount > 0 かつ area = 0 のノード | **12,889 / 37,258** (34.6%) |
| area = 0 の最大金額 | 3,300万円 |
| area > 0 の最小金額 | 約971万円 |
| type別内訳 | recipient: 11,944 / project-spending: 526 / project-budget: 419 |

### ノードサイズの具体例

クラスタサイズ 4000×4000 = 16,000,000 px²、総予算 ~151兆円の場合:

| 金額 | treemap面積(raw) | gap後面積 | k=100での screenArea |
|------|-----------------|----------|---------------------|
| 1000万円 | ~1.06 px² (≈1.07×1.0) | **0** (h→0) | 0 |
| 1億円 | ~10.6 px² (≈3.3×3.2) | ~5.3 px² | 53,000 |
| 10億円 | ~106 px² (≈10×10) | ~81 px² | 810,000 |

1px未満の辺がgapで0になるのが閾値（約1000万〜3300万円付近）。

## 解決方針

### 方針A: gap適用に最小サイズ保証を追加（推奨）

`applyGap` でgap適用後にwidth/heightの最小値を保証する。

- メリット: レイアウト段階で修正するため、ビュー側の変更不要
- 注意: ノード同士の重なりが発生するが、少額ノードは高倍率ズーム時のみ表示されるため実用上問題なし
- 最小サイズの目安: `0.01px`（k=100で1px、k=316で10px相当）

### 方針B: ビュー側でarea=0ノードに最小面積を付与

`Sankey2View.tsx` のLOD計算時に `node.area || MIN_FALLBACK_AREA` とする。

- メリット: レイアウト再生成不要
- デメリット: ノードの描画サイズが0のままなので、見えても矩形として描画されない（クリック不可）

### 方針C: レイアウト全体のスケールを拡大

`CLUSTER_WIDTH/HEIGHT` を4000→40000に拡大し、全ノードが1px以上になるようにする。

- メリット: gap問題が本質的に解消
- デメリット: 全座標系・エッジパスに影響し、既存のズーム倍率やビューポートカリングの再調整が必要

### 推奨: 方針A

`applyGap` に最小サイズ保証を追加するのが最もシンプル。変更箇所は `compute-sankey2-layout.ts` の `applyGap` 関数1箇所のみ。レイアウトデータ再生成が必要だが、パイプラインが整備済みのため問題なし。
