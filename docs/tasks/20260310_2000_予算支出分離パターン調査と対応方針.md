# 予算・支出分離パターン調査と対応方針

## 目的

RS Systemの可視化において、「予算/執行額は記録されているが支出先がない事業」と「支出先は記録されているが予算がない事業」が混在しており、Sankeyの見た目・品質スコアの信頼性に影響している。

このドキュメントでは、発生パターンを分類・定量化し、対応の優先度と方法を整理する。

---

## データ調査結果（2026-03-10時点）

### 予算額の定義

本調査では 2-1 CSV の `計(歳出予算現額合計)` (予算年度2023) を予算額として使用する。

> `当初予算(合計)` は補正予算・繰越を含まないため、補正予算のみで措置された事業（子育てエコホーム支援2,100億・ワクチン支援1,008億等）が「予算なし」に誤判定される。`計(歳出予算現額合計)` = 当初予算 + 補正予算 + 前年度繰越 + 予備費等 が正しい比較対象。

なお、`当初予算(合計)` を使った場合との違いは以下の通り：

| 基準 | 予算あり事業数 | 予算あり・支出なし | 支出あり・予算なし |
|------|--------------|-------------------|-------------------|
| 当初予算(合計) | 4,577件 | 109件 | 169件 |
| **計(歳出予算現額合計)** ← 採用 | **4,715件** | **247件** | **169件** |

新たに「予算あり」と判定された138件は主に補正予算事業（子育てエコホーム、ワクチン支援、省力化投資補助等）。

### 全体統計

| 状態 | 件数 |
|------|------|
| 総事業数（1-2 CSV） | 5,664件 |
| 2023年度 歳出予算現額あり（2-1 CSV） | 4,715件 |
| 支出レコードあり（5-1 CSV） | 4,637件 |
| **歳出予算現額あり・支出なし** | **247件** |
| **支出あり・歳出予算現額なし** | **169件** |

なお、1-2 CSV と 5-1 CSV の「事業名」列（列4）はすべての事業で一致しており、事業名の表記ゆれは存在しない。

### 情報源となるCSV列

調査を通じて、以下の列が「予算・支出分離」の原因を特定するために有効であることが判明した。

| CSV | 列名 | 内容 |
|-----|------|------|
| 1-2 | `備考` | 「当該シートは関連事業の合計額を整理するために作成」「詳細は統合先のシートを参照」等の明示的な注記 |
| 1-2 | `事業の概要` | 参照先事業IDが記載されるケースあり（例: `予算事業ID 001937001953参照`） |
| 1-5 | `関連事業の事業ID` + `関連性` | 親子・統合関係を明示するマシンリーダブルなデータ（最重要） |
| 2-1 | `主な増減理由` | 「重要政策推進枠」「2023年度補正予算事業」「2024年度から開始」等 |

1-5 CSV の `関連性` 列には以下の値が存在する：

| 値 | 件数 | 意味 |
|----|------|------|
| 子事業 / 親事業 | 各95件 | 親バケット型（予算は親、支出は子） |
| 統合先 / 統合元 | 各81件 | 事業統合（旧事業は廃止、支出は統合先で記録） |
| 分割先 / 分割元 | 各27件 | 事業分割 |
| その他関連先 / 元 | 各299/295件 | 緩やかな関連 |
| 運用交付先セグメントシート等 | 各多数件 | 基金・交付金の運用構造 |

## Sankeyへの現在の影響

### 影響1：パターンAの親事業（3件）

- 現状：Sankeyに事業ノードが表示されるが支出先リンクが存在せず、**クリックしても何も表示されない**
- PID=20032は「533億の予算があるはずなのに支出先が見えない」状態
- 子事業（合計31件）は別の事業ノードとして正常に表示されており、視覚的に親子関係が分断されている

### 影響2：パターンAの子事業（31件）

- 支出先リンクは正常に存在する
- 品質スコア（予算比較軸）において、子事業には2-1の予算データがないためスコアが不正確になる可能性がある

### 影響3：パターンBの親事業（デジタル庁PID=4）

- 「情報システムの整備」が事業ノードとして表示されるが支出なし
- 子事業98件はデジタル庁の事業ノードとして正常に表示
- 影響：2,358億の予算がSankeyで「消えている」ように見える

### 影響4：パターンCの大規模補正予算事業

- 子育てエコホーム(2,100億)・ワクチン支援(1,008億)等：予算は記録されているが支出が見えない
- 逆に「中小企業生産性革命推進事業」(4,247億)は支出はあるが予算がゼロに見える

## 対応案

### 案0：現状維持 + データ品質注記ドキュメント

**内容:** 対応しない。別途「データ品質上の制限事項」としてサイト内FAQや説明文に記載する。

**メリット:**
- 実装コストゼロ
- データの正確な再現（原データの構造をそのまま反映）

**デメリット:**
- ユーザーが「533億の事業が支出なし」に気づいたとき、バグと誤認する

**適用範囲:** 全パターン

---

### 案1：Sankeyノードへの注記バッジ

**内容:** 「予算あり・支出なし」の事業ノードに「⚠支出記録なし」バッジを表示。ホバーで説明文を表示する。

**実装変更:**
- `app/lib/sankey-generator.ts` でノード生成時にフラグ付与
- `client/components/` のノードレンダリングに条件分岐追加

**メリット:**
- 実装規模が小さい
- 原データを変更しない

**デメリット:**
- 根本的な「予算と支出の分離」の問題は解決しない

**適用範囲:** 全パターン（汎用）

---

### 案2：事業詳細ダイアログに「関連事業」リンク追加

**内容:** 既存の品質スコアダイアログ（または事業詳細）に「関連事業」セクションを追加する。

- 親事業を開くと、1-5 CSVの`子事業`リンクをもとに子事業一覧へのリンクが表示される
- 子事業を開くと、「予算は親事業（XXX）を参照」という注記が表示される

**データソース:** 1-5 CSV の `関連事業の事業ID` + `関連性=子事業` を structured JSON に組み込む（または generate-structured.ts で参照できるように追加）。

**実装変更:**
- `scripts/generate-structured-json.ts` で 1-5 CSV を読み込み `parentProjectId`・`childProjectIds` フィールドを付与
- ダイアログコンポーネントに「関連事業」セクション追加（Sankey本体は無変更）

**メリット:**
- Sankeyの基本構造を変えない
- 1-5 CSVが正式なデータソースなので整合性が保たれる
- パターンA（31件）は完全にカバーできる

**デメリット:**
- Sankey図の見た目は変わらず、初見のユーザーには伝わらない
- パターンB・C には対応できない（1-5にリンクがないため）

**適用範囲:** パターンA

---

### 案3：仮想グループノードの追加（Sankey構造変更）

**内容:** 親事業を「グループノード」として扱い、クリックすると子事業一覧が表示される新しい中間ビューを追加する。

```
省庁 → [生活困窮者就労準備支援事業費等補助金（グループ）] → 子事業11件
                                                              ↓
                                                        子事業 → 支出先
```

**実装変更:**
- `app/lib/sankey-generator.ts` に「グループ事業」の概念追加
- `app/sankey/page.tsx` に「グループビュー」の新しいView State追加
- URLパラメータ拡張

**メリット:**
- 予算と支出の関係が視覚的に明確になる
- パターンA（1-5リンクあり）は正確にカバーできる

**デメリット:**
- 実装規模が大きい（新しいView Stateとレイヤー追加）
- パターンB（デジタル庁）は1-5にリンクがないためカバー困難

**適用範囲:** パターンA

## 推奨方針

| パターン | 規模 | 推奨案 |
|----------|------|--------|
| パターンA（3親事業・31子事業） | 小 | 案2（ダイアログリンク）+ 将来的に案3 |
| パターンB（デジタル庁系・98件） | 大 | 案1（バッジ注記）のみ（1-5リンクなし） |
| パターンC（その他・212件） | 多様 | 案0（現状維持）または案1 |

パターンAは1-5 CSVに機械可読な親子リンクが存在するため、案2の実装コストは低い。1-5 CSV を structured JSON に統合する処理を追加するだけで対応できる。
