# 事業オフセットスライダー対応設計

## 目的

支出先数が多い場合に限られていたオフセット絞り込みを事業にも拡張し、ユーザーが全事業（最大数百件）をページングしながら閲覧できるようにするため。

---

## 現状

- `recipientOffset` で支出先ウィンドウをスクロールできる
- 事業はオフセットなし（TopN 件が常に固定表示）
- オフセット対象は支出先のみ

## 新機能スペック

### UIコンポーネント

オフセットスライダーバーの左端に **オフセット対象コンボボックス** を追加:

```text
[支出先 ▼] Top  [start]〜[end]  [slider]  /N件  [▲▼]  [⇤]  | 事業: [N]  支出先: [N]
```

選択肢:
- `支出先`（デフォルト、現在の挙動）
- `事業`

---

### 「事業」選択時の挙動

#### 事業ウィンドウの定義

- 事業をウィンドウスコープ外支出額（`projectWindowValue`）降順でランク付け
- `[projectOffset, projectOffset + topProject)` のランク範囲を **ウィンドウ事業**（個別表示）
- `[0, projectOffset)` のランク範囲は **非表示**（ノード・エッジとも除外。集約にも含めない）
- `[projectOffset + topProject, ...)` のランク範囲は **集約ノード**（現行の「その他事業」と同じ扱い）

#### 支出先の扱い

- **全体ランキングではなく、ウィンドウ事業からの流入額**で支出先をランク付け
- `topRecipient` 件を個別表示、それ以降は集約ノード
- 支出先 Top の件数はオフセットを持たない（`recipientOffset = 0` 固定）
- 現在の `projectRecipientsMode`（特定事業ピン留め時）に近い動作だが、対象が「複数事業のウィンドウ」になる点が異なる

#### 具体例（offset=10, topProject=40, topRecipient=40）

| 事業ランク | 扱い |
|-----------|------|
| 0〜9（Top1〜10） | **非表示**（ノード・エッジ除外） |
| 10〜49（Top11〜50） | **個別表示**（ウィンドウ） |
| 50〜（Top51〜） | **集約ノード** |

- 支出先: ウィンドウ事業（ランク10〜49）からの流入額合計でランク付け → Top40を個別表示、残りは集約

## 実装設計

### 新規状態・型

| 状態 | 型 | デフォルト | URL param |
|------|----|-----------|-----------|
| `offsetTarget` | `'recipient' \| 'project'` | `'recipient'` | `ot` (`p` = project のみ保存) |
| `projectOffset` | `number` | `0` | `po` |

- `recipientOffset` は `offsetTarget === 'recipient'` のときのみ使用
- `projectOffset` は `offsetTarget === 'project'` のときのみ使用

### `filterTopN` シグネチャ変更（`app/lib/sankey-svg-filter.ts`）

追加引数:

```typescript
offsetTarget: 'recipient' | 'project'  // デフォルト 'recipient'
projectOffset: number                   // デフォルト 0
```

戻り値に追加:

```typescript
totalProjectCount: number  // offsetTarget === 'project' 時のウィンドウ母集団件数
```

### `filterTopN` 内部ロジック変更

#### `offsetTarget === 'project'` のとき

**Step 1: 全事業ランキング確定**

- トップ府省庁の全事業を既存のウィンドウスコープ外支出額でソート（現行ロジックを流用）
- ただし、ランク付け基準に使う「window spending」は事業オフセット確定前の全支出先への流入で計算する必要がある

設計上の注意: 「事業オフセット」と「支出先ウィンドウ」は相互依存するため、以下の順で計算:

1. 事業を**全支出先への流入額合計**で事前ランク付け（事業オフセットの決定基準）
2. `projectOffset` 〜 `projectOffset + topProject - 1` をウィンドウ事業に確定
3. ウィンドウ事業からの流入額で支出先をランク付け → `windowRecipients` を確定
4. `windowRecipients` を使って既存の `projectWindowValue` / `projectTailValue` を再計算（事業ノードの高さ、エッジ幅に使用）

**Step 2: 非表示事業の除外**

- `aboveWindowProjectIds` = ランク `[0, projectOffset)` の事業 ID セット
- これらの `project-spending` / `project-budget` ノード、およびそれらへのエッジをすべて除外
- `effectivelyHiddenIds` とは別に `aboveWindowProjectIds` を管理

**Step 3: 集約ノード**

- ランク `[projectOffset + topProject, ...)` は既存の `otherProjects` と同じ集約処理

**Step 4: 支出先ウィンドウ**

- ウィンドウ事業（ランク10〜49）からの流入のみ集計した支出先ランキングで TopN を決定
- `recipientOffset` は無視（0 固定）
- `tailRecipients`（集約支出先）も、ウィンドウ事業からの流入があるもののみ

### `page.tsx` 変更

#### 状態追加

```typescript
offsetTarget: 'recipient' | 'project'
projectOffset: number
```

#### URL シリアライズ

- `ot=p`（project のときのみ保存、recipient はデフォルトなので省略）
- `po=<number>`（0 のときは省略）

#### オフセットバー表示分岐

`offsetTarget === 'project'` のとき:
- スライダーの `max` = `totalProjectCount - topProject`
- 表示テキスト = `事業 Top{projectOffset+1}〜{projectOffset+topProject}`
- 件数テキスト = `/{totalProjectCount}件`

`offsetTarget === 'recipient'` のとき: 現行どおり

#### `jumpToRecipientRank` の変更

`offsetTarget === 'project'` のとき、支出先クリックでのジャンプ機能は無効（プロジェクトオフセットと支出先ランクが独立のため）

### オフセットリセットのタイミング

- `offsetTarget` を切り替えたとき: 両オフセットを 0 にリセット
- `topProject` 変更時: `projectOffset` を 0 にリセット（`offsetTarget === 'project'` のみ）

---

## 既存機能との干渉

| 機能 | `offsetTarget === 'project'` 時の挙動 |
|------|------------------------------------|
| `pinnedProjectId`（フォーカスピン） | ピン留め事業はウィンドウ外でも強制表示（現行と同じ） |
| `focusRelated` モード | `offsetTarget` にかかわらず `focusRelated` が ON のときは既存ロジック優先 |
| `ministryFocusMode` / `recipientFocusMode` | `focusRelated` ON 時のみ発動。`offsetTarget` と競合しない |
| 年度切り替え | 両オフセットをリセット（現行の `year` 変更時リセットと同様） |
