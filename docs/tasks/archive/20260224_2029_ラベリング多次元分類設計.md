# ラベリング多次元分類設計（特殊法人・社会機能別・設置主体・専門職）

作成日: 2026-02-24
関連: [法人番号ラベリング設計（Step 3）](20260223_1009_法人番号ラベリング設計.md) ／ [完全一致ラベリングシステム総括](20260222_1700_完全一致ラベリングシステム総括.md) ／ [格による支出先法人格分類設計](20260223_0807_格による支出先法人格分類設計.md)

---

## 1. 目的

現在の支出先ラベリングは「法人格（EntityType）」という1次元で支出先を分類しているが、これだけではユーザーが「NHKはどこに分類されているのか」「国立大学への支出はどれか」「弁護士法人への支出はいくらか」といった問いに直感的に答えられない。

**社会機能・設置主体・特殊法人種別・専門職格** という追加の分類軸を設けることで、支出先を多次元的に理解できるようにする。

---

## 2. 現状の分類体系

### 2-1. 既存の EntityType（法人格分類）

| EntityType | 対象 |
|-----------|------|
| 民間企業 | 株式会社・有限会社・合同会社等 |
| 地方公共団体 | 都道府県・市区町村 |
| 国の機関 | 府省庁・外局・地方支分部局 |
| 独立行政法人 | 国立大学法人・国立研究開発法人含む |
| 公益法人・NPO | 公益社団・公益財団・一般社団・一般財団・NPO |
| 外国法人 | 外国会社等 |
| その他 | 上記に該当しないもの |

### 2-2. 既存の SpendingTags

- `primaryCategory`: government / private / public-interest / individual-other
- `secondaryCategory`: 地方公共団体・株式会社・独立行政法人等（文字列）
- `primaryIndustryTag` / `industryTags`: ITシステム・防衛・建設・教育・研究・医療・保険等（14種）

### 2-3. Step 3a（未実装）との関係

Step 3a は 法人種別コード 399（その他設立登記法人）を「設立登記法人（特殊法人等）」として一括ラベルする。本設計はその「特殊法人等」をさらに細分化し、かつ他の EntityType に属する支出先にも社会機能・設置主体・専門職格を付与する。

### 2-4. 既存 IndustryTag との違い

`IndustryTag` は「何にお金を払ったか（用途・産業）」を表す。本設計で追加する `FunctionalCategory` は「支出先が社会的にどういう機能を果たしているか（受領者の属性）」を表す。一致することもあるが（医療費 → 医療機関）、異なることもある（大学への事務委託 → IndustryTag=事務局・BPR、FunctionalCategory=教育機関）。

## 3. 新設する分類軸の定義

4つの軸を追加する。これらはすべてオプション（付与できない場合は付与しない）。

### 3-1. 軸A: 特殊法人サブタイプ（SpecialCorporationSubtype）

EntityType や Step 3a で「設立登記法人（特殊法人等）」とされた支出先のうち、具体的な組織が特定できるものにサブタイプを付与する。

| SpecialCorporationSubtype | 対象例 |
|--------------------------|-------|
| 特殊法人 | 全国健康保険協会、日本年金機構、日本放送協会（NHK）、日本私立学校振興・共済事業団 |
| 特殊会社 | 日本たばこ産業（JT）、NTT東日本・西日本、成田国際空港、首都高速道路 |
| 特殊会社（JR） | JR東日本・JR西日本・JR東海・JR北海道・JR四国・JR九州 |
| 特殊会社（NEXCO） | NEXCO東日本・NEXCO中日本・NEXCO西日本 |
| 沖縄振興開発金融公庫 | 個別設置法に基づく政策金融機関 |

> **注意**: JT・JR系は法的には株式会社（法人種別 301）であり、EntityType='民間企業' のまま。SpecialCorporationSubtype を付与することで「政府関係の特殊会社」として識別可能にする。EntityType は変更しない。

### 3-2. 軸B: 社会機能分類（FunctionalCategory）

支出先が社会的に果たしている主な機能を表す。EntityType と直交する分類軸。

| FunctionalCategory | 対象例 | 主な根拠法 |
|-------------------|-------|---------|
| 教育機関 | 大学・高校・小中学校・幼稚園・専修学校 | 学校教育法 |
| 医療機関 | 病院・診療所・助産所 | 医療法 |
| 福祉機関 | 保育所・特別養護老人ホーム・障害福祉サービス事業所 | 社会福祉法等 |
| 文化機関 | 美術館・博物館・図書館・劇場 | 博物館法・図書館法等 |
| 研究機関 | 研究所・試験場（大学附属含む） | — |
| 放送・報道機関 | NHK・民間放送局 | 放送法 |
| 宗教機関 | 寺院・神社・教会・その他宗教法人 | 宗教法人法 |
| 金融政策機関 | 日本政策金融公庫・国際協力銀行 | 個別設置法 |

> **複合機能の扱い**: 大学附属病院は「教育機関」かつ「医療機関」。`primaryFunctionalCategory`（主機能）と `functionalCategories`（全機能）の2フィールドで対応する。

### 3-3. 軸C: 設置主体（EstablishmentAuthority）

社会機能分類（FunctionalCategory）が付与された支出先に限り、設置主体を付与する。社会機能のない法人（民間企業単体など）には付与しない。

| EstablishmentAuthority | 対象例 |
|-----------------------|-------|
| 国立 | 国立大学法人、独立行政法人 国立病院機構、国立科学博物館 |
| 都道府県立 | 都道府県立高校、都立病院、府立大学 |
| 市区町村立 | 市立小学校、区立図書館、町立病院 |
| 公立 | 設置主体が地方公共団体だが国立/都道府県立/市区町村立が特定できない場合 |
| 私立 | 学校法人・医療法人・社会福祉法人等が設置する教育・医療・福祉機関 |

**EstablishmentAuthority の推定ルール**（優先順位順）:

1. 名称接頭辞で直接判定: 「国立」「都立」「府立」「道立」「○○県立」「○○市立」「○○区立」「○○町立」「○○村立」→ 各 Authority
2. 法人格から推定:
   - EntityType='国の機関' かつ FunctionalCategory あり → `国立`
   - EntityType='独立行政法人' かつ FunctionalCategory あり → `国立`（国立大学法人含む）
   - EntityType='地方公共団体' かつ FunctionalCategory あり → `公立`
   - EntityType='公益法人・NPO' or '民間企業' かつ FunctionalCategory あり → `私立`
3. 上記いずれにも該当しない場合: 付与しない

### 3-4. 軸D: 専門職格（ProfessionalType）

士業法人・士業個人・専門サービス事業者を識別する。

#### D-1. 法人格をもつ専門職法人（名称パターン：接頭辞・接尾辞）

| ProfessionalType | 名称パターン例 |
|----------------|-------------|
| 弁護士法人 | `弁護士法人○○` |
| 税理士法人 | `税理士法人○○` |
| 監査法人 | `監査法人○○`、`○○監査法人` |
| 司法書士法人 | `司法書士法人○○` |
| 社会保険労務士法人 | `社会保険労務士法人○○`、`社労士法人○○` |
| 弁理士法人 | `弁理士法人○○` |
| 行政書士法人 | `行政書士法人○○` |

#### D-2. 個人資格者・個人事務所（名称パターン）

| ProfessionalType | 名称パターン例 |
|----------------|-------------|
| 個人弁護士 | `弁護士○○`、`○○法律事務所`（法人格なし） |
| 個人税理士 | `税理士○○`、`○○税理士事務所` |
| 個人公認会計士 | `公認会計士○○`、`○○会計事務所` |
| 個人司法書士 | `司法書士○○`、`○○司法書士事務所` |
| 個人社会保険労務士 | `社会保険労務士○○`、`社労士○○` |
| 介護・福祉専門職 | `介護福祉士○○`、`社会福祉士○○` |

#### D-3. 「会計管理者」名義の支出先について

RS CSV に `○○会計管理者`（例: `大阪府会計管理者`、`A市会計管理者`）として出現する支出先は、地方自治法第168条に基づく各自治体の必置の役職名義。独立した法人ではない。

**調査結果**（2026-02-25 時点）:
- ユニーク支出先名: 11件、総額 約0.001兆円（= 約1,000万円）
- 全件が都道府県または市区町村への支払い（交付金・光熱水料按分・賃金等）

**設計上の扱い**: ProfessionalType の新設対象外。`displayName` 正規化で吸収する。

```
spendingName:  "大阪府会計管理者"
displayName:   "大阪府"           ← 会計管理者 suffix を除去
EntityType:    地方公共団体
```

正規化ルールは `municipality_supplement.csv`（または同等の名称補完辞書）に、パターン `(自治体名)会計管理者 → displayName=(自治体名)` として数行追加するだけで対応可。`spendingName` には原票名が保持されるため情報損失なし。

### 4-1. 新規辞書ファイル: `public/data/dictionaries/special_corporation_names.csv`

**スキーマ案**:

**収録対象**（参考情報より）:

- NEXCO東日本・NEXCO中日本・NEXCO西日本 → 特殊会社（NEXCO）
- 成田国際空港株式会社、関西国際空港・大阪国際空港（KANSAI AIRPORTS）→ 特殊会社
- 首都高速道路株式会社、阪神高速道路株式会社、本州四国連絡高速道路株式会社 → 特殊会社
- 東京地下鉄株式会社（東京メトロ）→ 特殊会社
- NTT東日本・NTT西日本・NTTコミュニケーションズ → 特殊会社
- 全国健康保険協会 → 特殊法人
- 日本年金機構 → 特殊法人
- 日本私立学校振興・共済事業団 → 特殊法人
- 日本中央競馬会（JRA）→ 特殊法人
- 沖縄振興開発金融公庫 → 特殊法人、FunctionalCategory=金融政策機関
- 日本政策金融公庫 → 特殊法人（準拠型）、FunctionalCategory=金融政策機関

> **注意**: 辞書へのエントリは既存の Step 1（完全一致辞書）の処理パイプラインと整合が必要。`special_corporation_names.csv` は既存辞書とは独立した新規ファイルとして追加する。

### 4-2. 社会機能・設置主体パターン

`scripts/tag-classifier.ts` へのパターン追加。

**FunctionalCategory 判定ルール（優先順位順）**:

1. 法人格接頭辞から推定:
   - `医療法人` → 医療機関
   - `学校法人` → 教育機関
   - `社会福祉法人` → 福祉機関
   - `宗教法人` → 宗教機関
2. 名称中の機能語から推定:
   - 大学・短大・高専・高校・小学校・中学校・幼稚園・保育所 → 教育機関
   - 病院・診療所・医院・クリニック・助産所 → 医療機関
   - 特別養護老人ホーム・老人保健施設・障害福祉・デイサービス → 福祉機関
   - 美術館・博物館・資料館・図書館・劇場 → 文化機関
   - 研究所・研究院・試験場・研究センター（大学附属含む） → 研究機関
   - 放送局・テレビ・ラジオ → 放送・報道機関
3. EntityType から補完:
   - EntityType='独立行政法人' かつ「大学」を含む → 教育機関
   - EntityType='独立行政法人' かつ「研究」を含む → 研究機関
   - EntityType='独立行政法人' かつ「病院」を含む → 医療機関

**EstablishmentAuthority 判定ルール**（Section 3-3 参照）

### 4-3. 専門職格パターン

`scripts/tag-classifier.ts` へのパターン追加。

- 接頭辞マッチ（高精度）: `弁護士法人`、`税理士法人`、`監査法人`、`司法書士法人`等
- 個人称号マッチ（低精度）: `弁護士`、`税理士`、`公認会計士`、`司法書士`、`社会保険労務士`、`介護福祉士`、`社会福祉士`
  - 接頭辞として出現するもの（「弁護士○○」）と事務所名中のもの（「○○弁護士事務所」）を区別する

> **実装上の注意**: 個人称号は一般名詞にも出現する（「税理士法人」内の「税理士」等）ため、法人格が先にマッチした場合は個人称号マッチを適用しない。

## 5. データ構造への影響

### 5-1. `types/structured.ts` への追加

既存の `EntityType` は変更しない。以下の型を新規追加する：

```
SpecialCorporationSubtype: 特殊法人 | 特殊会社 | 特殊会社（JR）| 特殊会社（NEXCO）| ...
FunctionalCategory: 教育機関 | 医療機関 | 福祉機関 | 文化機関 | 研究機関 | 放送・報道機関 | 宗教機関 | 金融政策機関
EstablishmentAuthority: 国立 | 都道府県立 | 市区町村立 | 公立 | 私立
ProfessionalType: 弁護士法人 | 税理士法人 | 監査法人 | 司法書士法人 | 社会保険労務士法人 | 弁理士法人 | 行政書士法人 | 個人弁護士 | 個人税理士 | 個人公認会計士 | 個人司法書士 | 個人社会保険労務士 | 介護・福祉専門職
```

**`SpendingTags` への追加フィールド**:

```
specialCorporationSubtype?: SpecialCorporationSubtype
primaryFunctionalCategory?: FunctionalCategory
functionalCategories?: FunctionalCategory[]
establishmentAuthority?: EstablishmentAuthority
professionalType?: ProfessionalType
```

### 5-2. `SpendingRecord` への影響

`SpendingRecord.tags` は現在 `SpendingTags | undefined`。上記フィールドを `SpendingTags` に追加するため、`SpendingRecord` 自体の変更は不要。

### 5-3. ファイル構成の変更

| ファイル | 変更内容 |
|---------|---------|
| `types/structured.ts` | 新型追加（SpecialCorporationSubtype, FunctionalCategory, EstablishmentAuthority, ProfessionalType）、SpendingTags に新フィールド追加 |
| `public/data/dictionaries/special_corporation_names.csv` | 新規作成（特殊法人・特殊会社辞書） |
| `scripts/tag-classifier.ts` | 社会機能・設置主体・専門職格のパターン追加、特殊法人辞書の読み込み追加 |
| `scripts/generate-entity-dict.ts` | 特殊法人辞書の読み込みとエントリ生成（SpecialCorporationSubtype の付与） |

## 8. 未確定事項（実装前に確認が必要）

1. **辞書優先度**: `special_corporation_names.csv` は Step 1 辞書と同等に扱うか（最優先）、Step 3a 後の上書き処理として扱うか
2. **FunctionalCategory の多重付与**: `functionalCategories` に複数値を持たせる場合、JSON サイズへの影響を測定する（特に多重機能を持つ大学・病院系）
3. **UI 側の表示**: 新分類軸をサンキー図のフィルタ・ツールチップに反映するかは UI 設計で別途決定
4. **`○○会計管理者` 正規化の網羅性**: RS CSV 全件での出現パターンをデータ更新時に再確認し、`municipality_supplement.csv` への追記を随時行う
