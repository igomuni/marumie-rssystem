# 再委託チェーン表示改善

## 目的

/sankey2 のユーザーが、再委託チェーンの構造（誰が誰に再委託し、実質支出がいくらか）を直感的に把握できるようにするため。

## 現状の課題

### 具体ケース: 政府共通ネットワーク（PID=18, デジタル庁）

- 事業(予算): 0円 / 執行額: 28.01億円
- 直接の支出先: **一般社団法人行政情報システム研究所のみ（28.01億円）**
- 同法人からの再委託フロー: 富士通 9.82億 + 支出委任16社 2.53億 + 民間企業9社 1.18億 = **13.53億円**
- しかし現在のグラフでは `project-spending-18 → 富士通 (9.82億)` の直接エッジが存在し、構造が不正確

### 根本問題: 間接支出先にも直接エッジが生成されている

`generate-sankey2-data.ts` が `isDirectFromGov` フラグを無視して全支出先に `project-spending → recipient` エッジを生成。

**現在のグラフ（不正確）**:

```text
project-spending-18 → 行政情報システム研究所 (28.01億)  ← 正しい
project-spending-18 → 富士通 (9.82億)                  ← 不正（再委託経由）
行政情報システム研究所 → 富士通 (10.20億) [subcontract]  ← 正しい
```

**あるべきグラフ**:

```text
project-spending-18 → 行政情報システム研究所 (28.01億)
行政情報システム研究所 → 富士通 (10.20億) [subcontract]
```

### ノード金額の不整合

現在の支出先ノードの `amount` は全事業・直接/間接を問わず合算したもの。再委託チェーンを正しく表示するには、ノード金額を **直接支出のみ** にするべき。

| 支出先 | 現在のamount | 直接支出のみ | 間接支出分 | 再委託流出 | 実質支出 |
|--------|-------------|------------|-----------|-----------|---------|
| 行政情報システム研究所 | 44.10億 | 44.10億 | 0 | 13.53億 | 30.57億 |
| 富士通 | 2,187億 | 1,818億 | 370億 | 41億 | 1,777億 |

### 全体への影響

| 指標 | 金額 |
|------|------|
| 全支出先 受入総額（現在） | 155.03兆 |
| うち直接支出 | 129.73兆 |
| うち間接支出（除外対象） | 25.31兆 |
| 再委託ありの支出先数 | 1,086件 |

### 執行額の非表示

現在のグラフデータに `executedAmount` フィールドがない。全体の予算 151.12兆 に対し執行額は 129.52兆だが、事業(予算)ノードでは予算額のみが表示されている。

## 改善案

### 0. グラフ生成の修正（必須・前提）

**対象**: `scripts/generate-sankey2-data.ts`, `client/components/Sankey2/types.ts`

3つの変更:

**a. 直接エッジの生成条件追加**

`project-spending → recipient` エッジ生成時に、structured.json の `isDirectFromGov` を参照。`isDirectFromGov === false` のエントリにはエッジを生成しない。

**b. 支出先ノードの金額を直接支出のみに変更**

`recipientMap` の金額集計時に `isDirectFromGov === false` のプロジェクト金額を除外。これにより:
- ノード金額 = 政府から直接受け取った合計
- `project-spending → recipient` エッジの value 合計 = ノード金額 で整合
- 再委託チェーンで受け取った分は subcontract エッジの value で表現

**c. 執行額フィールドの追加**

`Sankey2Node` と `LayoutNode` に `executedAmount?: number` を追加。以下のノードに付与:

| ノードtype | executedAmount の意味 | 算出方法 |
|-----------|---------------------|---------|
| `total` | 全事業の執行額合計 | `budgets` の `executedAmount` 合計 |
| `ministry` | 府省庁内の執行額合計 | 府省庁に属する事業の `executedAmount` 合計 |
| `project-budget` | 事業の執行額 | `budgets` の `executedAmount` |
| `project-spending` | 付与しない | — |
| `recipient` | 付与しない | — |

**影響**: レイアウト再生成が必要

### A. サイドパネルの表示拡張

**対象**: `DetailPanel` コンポーネント

**a. 執行額の表示**

`total`, `ministry`, `project-budget` ノードのヘッダーに:
- `執行額: {executedAmount}` を追加表示

**b. 実質支出の表示**

支出先（recipient）ノードで再委託流出がある場合:
- **再委託流出合計**: outEdges のうち `edgeType === 'subcontract'` の合計
- **実質支出**: ノード金額 - 再委託流出合計

### B. ツールチップの表示拡張

**対象**: ノードホバー時のツールチップ

- `project-budget` ノード: `執行額: {executedAmount}` を追加
- `total`, `ministry` ノード: `執行額: {executedAmount}` を追加
- 再委託流出がある `recipient` ノード: `実質: {実質支出額}` を追加

### C. 再委託チェーン専用BFSモード

**対象**: `bfsHighlight` 関数 / ハイライトロジック

現在のBFSに加えて、再委託エッジのみを辿るBFSモードを追加:
- トリガー: サイドパネルの「再委託」バッジクリック、またはキーボードショートカット
- 再委託エッジ（`edgeType === 'subcontract'`）のみを辿ってハイライト
- 通常エッジはdim表示（opacity低下）

### D. 再委託チェーン深さのバッジ表示

**対象**: `DetailPanel` ヘッダー / ノードラベル

再委託チェーンの深さ（subcontractエッジのみでBFS）を計算し表示:
- 深さ0: 再委託なし
- 深さ1: 直接の再委託元/先がある
- 深さ2+: 多段再委託

計算はビュー側で `edgeIndex` から動的に算出
