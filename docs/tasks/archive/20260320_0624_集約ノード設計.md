# LOD集約ノード設計

## 目的

ズーム不足で非表示になっているノードの存在と金額規模をユーザーが把握でき、見落としなく探索できるようにする。

## 現状の課題

`MIN_SCREEN_AREA=25` でLOD除外されたノードは画面上に表示されない。ユーザーは「そこにノードがあるのに見えない」ことを認知できず、ズームすべき場所がわからない。

### ズームレベル別の非表示ノード数

| k (zoom%) | 非表示ノード | うちrecipient | 非表示金額 |
|-----------|------------|--------------|----------|
| 1 (100%) | 27,416 | 22,531 | 8,327億 |
| 5 (500%) | 14,774 | 13,565 | 542億 |
| 10 (1000%) | 8,503 | 8,159 | 98億 |
| 50 (5000%) | 1,479 | 1,464 | 1億 |
| 100 (10000%) | 501 | 500 | 0.1億 |

主にrecipientクラスタで多数のノードが非表示になる。金額規模は総予算(151兆)に対し微小だが、個別の支出先探索時には重要。

## 設計

### 集約ノードの生成

`visibleNodes` 計算の `useMemo` 内で、LOD除外されたノードをクラスタtype別に集計し、仮想的な集約ノードを生成する。

- **集約単位**: クラスタtype別（`project-budget`, `project-spending`, `recipient`）
  - `total`/`ministry` は非表示ノードが少ないため対象外
- **集約ノードのプロパティ**:
  - `id`: `__lod-aggregate-{type}`
  - `label`: `他 {count}件`
  - `amount`: 非表示ノードの合計金額
  - `type`: 元のクラスタtype（色の統一のため）
- **非表示ノードが0件のtypeには集約ノードを生成しない**

### 配置

集約ノードの位置は、非表示ノード群の重心（centroid）を使用:

```
x = 非表示ノードのx座標の平均
y = 非表示ノードのy座標の平均
```

サイズは固定（例: 画面上で常に一定サイズに見えるよう `width = 40/k, height = 20/k`）。ズームしても画面上のサイズが変わらないアイコン的表示。

### 描画

- 通常のノード矩形とは区別し、破線枠 + 半透明背景で描画
- ラベル: `他 {count}件 / {金額}` を常時表示
- クリック時: そのエリアにズームイン（非表示ノードのbounding boxにfit）

### エッジの扱い

集約ノードへのエッジは描画しない。集約ノードは情報表示のみで、Sankeyフローの一部としては扱わない。ズームインすれば個別ノードとそのエッジが自然に表示される。
