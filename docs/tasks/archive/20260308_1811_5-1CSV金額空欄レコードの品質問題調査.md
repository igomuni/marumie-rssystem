# 5-1 CSV 金額空欄レコードの品質問題調査

## 目的

品質スコアリングで「支出実績が記録されていない事業・ブロック」を炙り出すため、
5-1 CSV（支出先・支出情報）における金額空欄レコードの実態を把握する。

## 発見した問題

### 前提: CSVの基本構造（2行1セット）

5-1 CSV は支出先1件につき原則2行で構成される：

| 行 | 記載内容 |
|----|---------|
| 1行目 | `ブロックの合計支出額`・`支出先の合計支出額` |
| 2行目 | `金額`・`契約概要`・`契約方式等` 等の個別契約詳細 |

### 金額空欄グループの規模

`(予算事業ID, 支出先ブロック番号, 支出先名)` をキーとしてグループ化した場合：

| 区分 | グループ数 | 割合 |
|------|-----------|------|
| 契約1件（正常） | 65,694 | 69.3% |
| 契約2件以上（複数契約） | 9,033 | 9.5% |
| **金額なし（問題対象）** | **20,135** | **21.2%** |

金額なしグループの内訳：

| `ブロックの合計支出額` | グループ数 |
|----------------------|-----------|
| あり（>0） | 18,741 (93%) |
| 空欄 | 1,394 (7%) |

### パターン分類

#### パターン1: ブロックヘッダー行（18,741件）

- **特徴**: 支出先名が空欄、`ブロックの合計支出額` のみ記載
- **実態**: ブロック集計行（CSVフォーマット上の区切り行）
- **判断**: 支出先ではないため、スコアリング対象から除外すべきダミー行

```
PID=1  Block=A 支出先=[] ブロック合計=10,070,000  事業=内閣人事局経費
PID=7  Block=A 支出先=[] ブロック合計=200,362,000 事業=マイナンバー制度の推進
PID=7  Block=B 支出先=[] ブロック合計=600,879,000 事業=マイナンバー制度の推進
```

#### パターン2: ブロック定義行（一部1,394件に含まれる）

- **特徴**: 支出先名も空欄、`ブロックの合計支出額` も空欄。`支出先ブロック名` と `事業を行う上での役割` だけが記載
- **実態**: ブロックの役割説明のみで金額情報なし
- **判断**: 記入漏れまたは構造説明専用の行。支出実績ゼロを意味する

```
PID=129 Block=A 支出先=[] ブロック名=各省庁     役割=総合特区に関する計画の実現を支援するため...
PID=129 Block=B 支出先=[] ブロック名=民間事業者等 役割=総合特区に関する計画の実施
PID=182 Block=B 支出先=[] ブロック名=出融資先
PID=183 Block=C 支出先=[] ブロック名=中小・小規模事業者等
```

#### パターン3: 「民間企業等」金額なし

- **特徴**: 支出先名=「民間企業等」または類似の総称、金額系列がすべて空欄
- **実態**: 記入漏れと思われる。`その他支出先=FALSE`・所在地のみ記載
- **判断**: データ不備。支出先は特定されているが支出金額の根拠がない

```
PID=4 Block=A 支出先=民間企業等 ブロック合計=空 支出先合計=空 金額=空
PID=4 Block=B 支出先=民間企業等 ブロック合計=空 支出先合計=空 金額=空
```

## 品質スコアへの影響

現状のスコアリングでは、上記の空欄レコードも `a=0, a2=0` として
`recipient_rows` に取り込まれており、以下の問題がある：

1. **軸1（支出先名品質）の分母汚染**: 支出先名空欄のレコードが `invalid` カウントに
   混入し、スコアを不当に下げる可能性がある
2. **支出先リスト表示のノイズ**: ScoreDetailDialog の支出先一覧に
   実態のない行が表示される
3. **軸5（透明性）への影響**: 金額不明な支出先が `opaque` 判定される可能性

## 今後の対応候補

### 短期（フィルタリング）
- `score-project-quality.py` のレコード取込時に
  `支出先の合計支出額` と `金額` が両方空欄のレコードを除外する

### 中期（新軸の追加・強化）
- **「ブロック記入完全性」スコア**: ブロックが定義されているのに
  支出先・金額が一切記載されていない割合を軸として評価する
- パターン2（ブロック定義行のみで実支出なし）は
  「支出の透明性が確保されていない事業」として減点対象にできる
