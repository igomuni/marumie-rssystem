# /sankey スライダー式支出先ドリルダウン

## 目的

ユーザーが支出先のランキングを**連続的にスクロール**できるようにするため。

現状は「支出先(TopN以外)」ノードをクリックして次の10件に進む操作で、位置の把握や任意位置へのジャンプができない。スライダーにより「上位何位から何位を見ているか」が直感的に分かり、自由な位置への移動が可能になる。

## スコープ

- **対象**: Global View の支出先列のみ
- **対象外**: Ministry View / Project View / Spending View の支出先表示（既存のまま）
- **ブランチ**: main から切る（feature/sankey2-block-node-layer とは独立）

## 現状の仕組み

### 支出先表示の流れ

1. `page.tsx` → API に `spendingLimit`（表示件数）と `spendingDrilldownLevel`（ページ番号）を送信
2. `sankey-generator.ts` の `selectData()` で `spendingOffset = spendingDrilldownLevel * spendingLimit` を計算
3. 全支出先を金額降順にソートし `slice(spendingOffset, spendingOffset + spendingLimit)` で切り出し
4. 残りは「支出先(TopN以外)」ノードに集約

### 現状の課題

- 「支出先(TopN以外)」ノードのクリックでしか次ページに進めない
- 前のページに戻るには画面リセットが必要
- 全体の何件中どこを見ているかの表示がない

## 設計

### UIコンポーネント: RecipientRangeSlider

Sankey図の下部（または支出先列の上部）に配置する水平スライダー。

**表示要素:**

- スライダー本体: 0〜最大支出先数の範囲を表すトラック上に、現在位置を示すつまみ
- 現在位置ラベル: 「11位〜20位 / 26,823件」のような表示
- 左右ボタン（任意）: 1ページ分（spendingLimit件）ずつ移動

**操作:**

- つまみをドラッグ → 開始位置が変わる → 表示件数は固定（topNSettings.global.spending の値）
- ドラッグ中はラベルのみ更新（API呼び出しは行わない）
- ドラッグ終了（mouseup/touchend）時にAPIを呼び出して Sankey を更新

**スライダーの値:**

- min: 0（1位から）
- max: totalRecipientCount - spendingLimit（末尾ページの開始位置）
- step: spendingLimit（ページ単位で移動。例: 10件ずつなら 0, 10, 20, ...）
- value: spendingOffset（= spendingDrilldownLevel × spendingLimit）

### API変更

既存パラメータをそのまま活用する。新しいAPIパラメータは不要。

| パラメータ | 現状 | 変更 |
|-----------|------|------|
| `spendingLimit` | 設定ダイアログから設定 | 変更なし |
| `spendingDrilldownLevel` | ノードクリックで+1 | スライダー値から算出: `Math.floor(sliderValue / spendingLimit)` |

### データ構造の変更

スライダーの max 値を決定するには「Global View でフィルタ後の支出先総数」が必要。

**現状の問題**: `metadata.summary.totalSpendings`（= 26,823）は全支出先の総数であり、Global View で実際にスライダー対象となる支出先数とは異なる。Global View では TopN 省庁からの支出がある支出先のみがソート対象で、かつ「その他」（spendingName === 'その他'）は除外される。この **フィルタ後の件数** は `selectData()` 内の `allRecipients.length` でのみ計算されており、現在は外部に返していない。

**変更箇所（3層を貫通）:**

| 層 | ファイル | 変更 |
|----|---------|------|
| Domain | `app/lib/sankey-generator.ts` | `DataSelection` に `totalFilteredRecipients: number` を追加。`selectData()` で `allRecipients.length` を設定 |
| Domain | `app/lib/sankey-generator.ts` | `generateSankeyData()` で `metadata.summary` にパススルー |
| Type | `types/preset.ts` | `PresetMetadata.summary` に `totalFilteredSpendings?: number` を追加 |
| API | `app/api/sankey/route.ts` | 変更不要（`metadata` はそのまま返却済み） |

**注意**: `totalFilteredRecipients` は TopN 省庁設定（`limit`）に依存するため、省庁数を変更するとスライダーの max も変わる。これは正しい動作（表示対象の支出先母数が変わるため）。

### 状態管理の変更（page.tsx）

| 状態 | 現状 | 変更 |
|------|------|------|
| `viewState.spendingDrilldownLevel` | ノードクリックで increment | スライダー操作でも更新 |
| `totalFilteredSpendings` | なし | APIレスポンスの `metadata.summary.totalFilteredSpendings` から取得 |

### 「支出先(TopN以外)」ノードとの共存

- 「支出先(TopN以外)」ノードは**そのまま残す**
- ノードクリック時の動作も維持（スライダー位置が連動して更新される）
- スライダーとノードクリックの両方から `spendingDrilldownLevel` を更新できる双方向連携

### ドラッグ中のパフォーマンス

- ドラッグ中はAPIを呼ばない（ラベルの位置表示のみ更新）
- `onChangeCommitted`（MUI）や `mouseup` イベントでのみAPIリクエスト発火
- デバウンスは不要（コミット時のみ発火のため）

## ロードマップ

### Step 1: データ層 — フィルタ後の支出先総数を返す

- `DataSelection` に `totalFilteredRecipients` を追加
- `selectData()` の Global View パスで `allRecipients.length` を設定
- `generateSankeyData()` で `metadata.summary.totalFilteredSpendings` にパススルー
- `types/preset.ts` の `summary` に `totalFilteredSpendings?: number` を追加
- **検証**: `npm run lint` + `npx tsc --noEmit` で型エラーなし

### Step 2: スライダーコンポーネントの実装

- `client/components/RecipientRangeSlider.tsx` を新規作成
- props: `value`（現在のoffset）, `max`（totalFilteredSpendings）, `step`（spendingLimit）, `onChange` コールバック
- HTML `<input type="range">` ベース（外部ライブラリ不使用）
- 現在位置ラベル表示:「{start}位〜{end}位 / {total}件」
- ドラッグ中は `onChange` を発火せず、`onMouseUp`/`onTouchEnd` でのみコールバック

### Step 3: page.tsx への組み込み

- `RecipientRangeSlider` を Sankey 図の下部に配置
- スライダーの `onChange` → `spendingDrilldownLevel` を更新 → API再取得
- 「支出先(TopN以外)」ノードクリック時にスライダー位置も連動更新
- Global View 以外ではスライダーを非表示

### Step 4: 動作確認・調整

- `npm run dev` でスライダー操作 → Sankey 図が正しく切り替わることを確認
- ノードクリックとスライダーの双方向連携を確認
- 末尾ページ（支出先が spendingLimit 未満）の表示を確認
- `npm run lint` + `npx tsc --noEmit` で最終チェック
