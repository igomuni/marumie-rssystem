# 支出先ラベリング確認UI 実装ドキュメント

作成日: 2026-02-28

---

## 概要

元CSV `data/year_2024/5-1_RS_2024_支出先_支出情報.csv` を直接読み込み、
「金額あり・支出先の合計支出額なし」に該当する支出先名（26,148件）に対してラベリングを行い、
その結果を閲覧・確認できる Read-only UI を新規に作成した。

既存の `entities-v2`、`generate-entity-labels.py`、`recipients_without_total.csv` は**流用しない**。

## generate-entity-labels-csv.py

### 入力・出力

| 項目 | パス |
|------|------|
| 入力CSV | `data/year_2024/5-1_RS_2024_支出先_支出情報.csv` |
| 辞書ディレクトリ | `public/data/dictionaries/` |
| 出力JSON | `public/data/entity-labels-csv.json` |

### フィルタ条件

```python
# 金額あり AND 支出先の合計支出額なし
if not name or not kinryo or total:
    continue
```

### 集計単位

支出先名（`支出先名`列）でグループ化:

| フィールド | 内容 |
|-----------|------|
| `amount` | 金額の合計（円） |
| `count` | レコード数 |
| `cn` | 最初に現れた法人番号 |
| `typeCodes` | 法人種別コードの重複排除リスト |

### ラベリングロジック

**Step 1（辞書一致）** → **Step 2（格パターン）** の順で適用。Step 2 が高優先（上書き）。

#### Step 1: 辞書マッチング

| 辞書ファイル | L1 | L2 |
|------------|----|----|
| `ministry_names.csv` | 国の機関 | 府省庁 |
| `ministry_from_ichiran.csv` | 国の機関 | 府省庁 |
| `ministry_supplement.csv` | 国の機関 | 府省庁 |
| `police_names.csv` | 国の機関 | 警察 |
| `prefecture_names.csv` | 地方公共団体 | 都道府県 |
| `municipality_names.csv` | 地方公共団体 | 市区町村 |
| `municipality_supplement.csv` | 地方公共団体 | 市区町村 |
| `foreign_agency_names.csv` | 外国法人・国際機関 | 外国政府機関 |
| `embassy_names.csv` | 外国法人・国際機関 | 大使館 |
| `international_org_names.csv` | 外国法人・国際機関 | 国際機関 |
| `country_names.csv` | 外国法人・国際機関 | 外国 |
| `aggregate_names.csv` | その他（集合） | 集合名称 |
| `beneficiary_names.csv` | その他（集合） | 受益者 |
| `special_corporation_names.csv` | 特殊法人・特別の法人 | 特殊法人 or 特殊会社 |

府省庁スキーマ（`ministry`, `bureau`, `division` 等の複数列）は列ごとに名称を抽出して辞書化。
`aggregate_names.csv` の `match_type=regex` エントリはスキップ（完全一致のみ辞書化）。

#### Step 2: 格パターンマッチング

`KAKU_PATTERNS`（68パターン）で `re.search()` マッチング。最初にヒットしたパターンを採用。

**複合名称スキップ**: `・` または `、` を含み、かつ区切られた複数セグメントのうち2つ以上に格ワードがマッチする名称はスキップ（例: `○○株式会社・△△株式会社`）。

**コンソーシアムバイパス**: スキップ対象でも `共同企業体`・`コンソーシアム` 等を含む場合は強制適用。

#### `source` フィールドの値

| 値 | 意味 |
|----|------|
| `dict` | Step 1 のみヒット |
| `kaku` | Step 2 のみヒット（Step 2 で上書き） |
| `both` | Step 1・Step 2 の両方でヒット（Step 2 優先） |
| `none` | 未ラベル |

### 出力 JSON スキーマ

```json
[
  {
    "name": "株式会社○○",
    "l1": "民間企業",
    "l2": "株式会社",
    "source": "kaku",
    "amount": 123456789,
    "count": 5,
    "cn": "1234567890123",
    "typeCodes": ["301"]
  }
]
```

- `l1` / `l2`: ラベルなしの場合は `null`
- `typeCodes`: 空の場合は `[]`

## API: app/api/entity-labels-csv/route.ts

### エンドポイント

`GET /api/entity-labels-csv`

### レスポンス型

```typescript
export interface EntityLabelItem {
  name: string;
  l1: string | null;
  l2: string | null;
  source: 'dict' | 'kaku' | 'both' | 'none';
  amount: number;
  count: number;
  cn: string;
  typeCodes: string[];
}

export interface EntityLabelsCsvResponse {
  items: EntityLabelItem[];
  summary: {
    total: number;
    labeled: number;
    labeledAmount: number;
    totalAmount: number;
    byL1: Record<string, { count: number; amount: number }>;
    bySource: Record<'dict' | 'kaku' | 'both' | 'none', { count: number }>;
  };
}
```

### 実装の特徴

- `public/data/entity-labels-csv.json` を読み込み、プロセス内キャッシュ（再起動まで保持）
- `summary` はサーバー側で計算して返す（クライアント側の計算量を削減）
- ファイル未存在時は `503` + エラーメッセージを返す

### 状態管理

| 状態変数 | 型 | 初期値 |
|---------|-----|--------|
| `data` | `EntityLabelsCsvResponse \| null` | `null` |
| `loading` / `error` | `boolean` / `string \| null` | `true` / `null` |
| `searchQuery` | `string` | `''` |
| `selectedL1` | `string \| null` | `null`（すべて） |
| `selectedSource` | `'all' \| 'dict' \| 'kaku' \| 'both' \| 'none'` | `'all'` |
| `sortField` | `'amount' \| 'count' \| 'name'` | `'amount'` |
| `sortDir` | `'asc' \| 'desc'` | `'desc'` |
| `page` | `number` | `1` |

フィルタ条件変更時は `page` を自動で `1` にリセット。

### UI 操作

- **L1チップ**: クリックで絞り込み。もう一度クリックで解除。「ラベルなし」= `l1 === null`
- **ソースフィルタ**: 辞書 / 格パターン / 両方 / 未ラベル で絞り込み
- **テキスト検索**: 支出先名の部分一致（大文字小文字区別なし）
- **テーブルヘッダーソート**: 支出先名・金額・件数の各列をクリックでソート（同列クリックで昇降切替）
- **ページネーション**: 50件/ページ
