# 支出先ノードID改善

## 目的

開発者が `sankey3-graph.json` のノードIDとクライアント側集約ノードのIDを安全に扱えるようにするため。現行は支出先名をそのままIDに使用しており、集約ノードとの衝突や特殊文字の問題がある。

## 現行の問題

`generate-sankey3-data.ts` は支出先名をそのままノードIDに使用している:

```
recipientId = `recipient-${recipient.name}`
例: recipient-三井不動産ビルマネジメント株式会社
```

これは以下の問題を引き起こす:

| 問題 | 詳細 |
|------|------|
| 集約ノードとのname衝突 | 実データに `recipient-その他の支出先`（1,203億円）が存在し、`filterTopN` が生成する集約ノード `{ id: 'recipient-other', name: 'その他の支出先' }` の `name` と衝突している |
| 特殊文字 | `&` を含む支出先名が67件（例: `みずほリサーチ&テクノロジーズ株式会社`） |
| IDサイズ | 名前ベースで261KB、連番なら74KB（JSON全体に対する割合は小さいが無駄） |
| 一意性の前提が脆弱 | 支出先名の一意性はCSVデータ品質に依存。法人番号にも誤記があるため法人番号でのID化も不確実 |

## 改善方針: 連番ID + name属性

支出先名をIDに使うのをやめ、安定した連番IDを付与する:

```
現行: id: "recipient-その他"             name: "その他"
改善: id: "r-1"                          name: "その他"
```

### 生成スクリプト側の変更（`generate-sankey3-data.ts`）

1. `recipientMap` を金額降順でソート後に連番を振る（IDがランキング順と対応して扱いやすい）
2. ID形式: `r-{連番}` （短く、特殊文字を含まず、集約ノードと衝突しない）
3. `name` 属性は従来通り支出先名を保持

### クライアント側の変更（`app/sankey-lab/real/page.tsx` の `filterTopN`）

1. `e.target.startsWith('recipient-')` → `e.target.startsWith('r-')` に変更
2. 集約ノードのIDを `__agg-` プレフィックスに変更

### 集約ノードのID命名規則

| 現行 | 改善後 | 用途 |
|------|--------|------|
| `ministry-other` | `__agg-ministry` | その他省庁（集約） |
| `project-budget-other` | `__agg-project-budget` | その他事業・予算側（集約） |
| `project-spending-other` | `__agg-project-spending` | その他事業・支出側（集約） |
| `recipient-other` | `__agg-recipient` | その他支出先（集約） |

`__agg-` プレフィックスにより、実データのIDと集約ノードのIDが構造的に衝突しないことが保証される。

### 他のノードタイプのID方式

| type | 現行ID形式 | 件数 | 判断 |
|------|-----------|------|------|
| `total` | `total` | 1 | 変更不要 |
| `ministry` | `ministry-{省庁名}` | 37 | **変更不要** — 37件で固定、衝突リスクなし |
| `project-budget` | `project-budget-{数値ID}` | 5,664 | 変更不要 — 既に数値IDベースで安全 |
| `project-spending` | `project-spending-{数値ID}` | 5,664 | 変更不要 — 同上 |
| `recipient` | `recipient-{支出先名}` | 12,413 | **要改善** — 本ドキュメントの対象 |

府省庁の `建制順`（1-1/5-1 CSVに存在）をIDに使う案も検討したが、外局が親省庁と同じ番号を共有するため不採用:
- 37省庁に対して23種のユニーク値（例: 総務省=15, 消防庁=15, 公害等調整委員会=15）
- 単独でのID化には不向き。省庁名ベースの現行方式で問題なし

### 影響範囲

| ファイル | 変更内容 |
|---------|---------|
| `scripts/generate-sankey3-data.ts` | 支出先IDを連番に変更 |
| `app/sankey-lab/real/page.tsx` | `filterTopN` 内のID参照パターン変更 + 集約ノードID変更 |
| `public/data/sankey3-graph.json` | 再生成が必要 |
