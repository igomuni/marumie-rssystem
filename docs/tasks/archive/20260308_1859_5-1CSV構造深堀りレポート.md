# 5-1 CSV 構造深堀りレポート

調査対象: `data/year_2024/5-1_RS_2024_支出先_支出情報.csv`
総レコード数: 194,133件

---

## 1. CSVの基本構造（2行1セット）

5-1 CSV は支出先1件につき原則 **2行** で構成される：

| 行 | `ブロックの合計支出額` | `支出先の合計支出額` | `金額` | 主な記載内容 |
|:--:|:--:|:--:|:--:|:--|
| 1行目 | ○ | - | - | ブロックの集計額（支出先名は空欄） |
| 2行目 | - | ○ | - | 支出先ごとの合計額 |
| 3行目以降 | - | - | ○ | 個別契約の内訳（複数契約がある場合） |

支出先1件・契約1件の場合は「合計行 + 明細行」の2行セットになる。

## 3-2. 支出先合計行と金額行のペア検証（追試）

`支出先の合計支出額` と `金額` の有無をグループ単位で検証した結果：

| 支出先合計 | 金額 | グループ数 | 割合 | 備考 |
|:---------:|:----:|----------:|-----:|:-----|
| ○ | ○ | 74,727 | 99.7% | 正常（常にペアで存在） |
| ○ | - | **0** | **0.0%** | 支出先合計のみ → **存在しない** |
| - | ○ | 0 | 0.0% | 金額のみ → 存在しない |
| - | - | 219 | 0.3% | どちらもなし（金額0のケース等） |

**「支出先合計のみ・金額なし」は実データ上ゼロ件**であることを確認。
支出先合計行（`支出先の合計支出額` のみ記載の行）を支出先一覧の表示対象から除外しても、情報欠落は発生しない。

## 4. 問題パターンの詳細

### パターンA: ブロックヘッダー行（18,797件）

- `支出先名` が空欄、`ブロックの合計支出額` のみ記載
- スコアリングループで `continue` されるため **影響なし**
- CSVフォーマット上の区切り行

### パターンB: 完全空欄行（1,977件）

- `支出先名` も3金額列もすべて空欄
- `支出先ブロック名` や `事業を行う上での役割` のみ記載のブロック定義行
- スコアリングループで `continue` されるため **影響なし**

### パターンC: 支出先名ありで金額なし（440件、0.2%）

- `支出先名` はあるが3金額列すべて空欄
- スコアリングループに **入る** → 影響あり
- 主な支出先名：「民間企業等」「民間団体等」「未定」「その他」など総称・仮置き名

  上位名称（件数）:
  - 民間企業等: 20件
  - 民間団体等: 14件
  - 未定: 14件
  - 民間事業者: 12件
  - 民間事業者等: 12件
  - その他: 10件

- 影響内容:
  - `row_count` の分母に加算
  - 軸1: 辞書に存在しない → `invalid` または `unknown` カウント → スコア下降
  - 軸2: 法人番号が空 → `cn_empty` カウント → スコア下降

## 5. スコアリングへの影響まとめ

| パターン | 件数 | スコアへの影響 |
|---------|-----:|:-------------|
| 正常行（個別契約・支出先合計） | 172,919 | 正常に処理 |
| ブロックヘッダー行 | 18,797 | なし（`continue`） |
| 完全空欄行 | 1,977 | なし（`continue`） |
| **支出先名あり・金額なし** | **440** | **軸1・軸2を不当に下降させる可能性** |

440件（全体の0.2%）が軽微な影響を与えているが、現時点では対処優先度は低い。
将来的に除外フィルタを追加する場合は `支出先の合計支出額` と `金額` の両方が空の支出先名ありレコードを対象とする。
