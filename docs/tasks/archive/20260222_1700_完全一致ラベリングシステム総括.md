## 1. 概要

`data/result/recipients_without_total.csv`（26,192 ユニーク支出先名、98,599 行、155.033 兆円）を対象に、**正規化なしの完全一致**でエンティティラベルを付与するための辞書 CSV 群を構築・整備した。

> **スコープ**: 金額規模よりも**ユニーク支出先名のカバレッジ**を重視。
> 1 件しかマッチしない名称でも、ラベル付け可能にすることが目的。

## 3. 共通スキーマ設計

全辞書は以下の原則に従う。

| 列 | 役割 |
|----|------|
| `xxx_id` | エンティティ一意ID（スネークケース） |
| `canonical_name` | 代表表記（UI 表示・集計用） |
| `name` | 完全一致マッチング対象の文字列 |
| `name_type` | `canonical` / `alias`（同一エンティティの表記ゆれ） |
| 種別列（辞書による） | `embassy_type`, `org_type`, `category` 等 |

### alias の使われ方

```
canonical: 在トリニダード・トバゴ日本国大使館
alias:     在トリニダード・トバゴ大使館
alias:     在トリニダード・トバコ大使館  ← 誤字バリアント
```

1 エンティティに複数の `name` 行を持ち、いずれかの完全一致でラベルを付与できる。

## 4. カバレッジ推移

対象: `recipients_without_total.csv`（26,192 ユニーク名称 / 155.033 兆円）

| フェーズ | 追加辞書 | ユニーク名称 | 名称カバレッジ | 金額カバレッジ |
|---------|---------|------------:|:------------:|:------------:|
| 初期 5 辞書 | country / prefecture / municipality / ministry×2 | 1,283 | 4.90% | 13.73% |
| +補完 4 辞書 | municipality_supplement / police / ministry_supplement / foreign_agency | +184 → 1,467 | 5.60% | 16.00% |
| +受給者・国際機関 | beneficiary / international_org / ministry_supplement 更新 | +204 → 1,671 | 6.38% | 51.39% |
| **+大使館（最終）** | embassy | **+71 → 1,742** | **6.65%** | **~51.4%** |

### 注目ポイント

- **金額カバレッジの急伸（13.73% → 51.39%）** は受給者辞書によるもの。
  年金受給者（24.7 兆）・年金受給者等（23.8 兆）の 2 名称だけで 49 兆円。
- **名称カバレッジが 6.65% に留まる**のは想定内。
  残り 93% の大半は株式会社・学校法人・個人等であり、別アプローチ（法人番号照合、部分一致等）が必要。

## 5. キュレーション判断基準

補完辞書を追加した際の選定基準を記録する。

### 追加対象としたもの

| 基準 | 例 |
|------|----|
| 同一パターンが多数出現する国の機関 | 都道府県労働局（47 局）、地方法務局（50 局）等 |
| データに明示的に現れる外国政府機関 | 在日米軍省（8,302 億円）、NASA |
| 集合名称として使われる受給者パターン | 年金受給者、求職者等、農業者等 |
| 実データ由来の完全一致が確認できる名称 | embassy_names.csv（全 71 件データ一致確認済み） |

### 追加対象外としたもの（現時点）

| 理由 | 例 |
|------|----|
| 法人番号照合で対応可能 | 株式会社・学校法人等（残り ~93%） |
| 部分一致・正規化が必要 | 略称・通称・旧組織名 |
| データ量が極小で収益が低い | 個別名称のワンオフ対応 |

## 6. 辞書の再生成手順

`public/data/dictionaries/` の CSV はすべてスクリプトから再生成可能。

```bash
# データ駆動型（DB / PDF が必要）
python3 scripts/generate-country-names.py
python3 scripts/generate-prefecture-municipality-names.py   # data/houjin.db 必要
python3 scripts/extract-ministry-from-pdf.py                # data/download/kikouzu_202508/ 必要 → ministry_from_pdf.csv
python3 scripts/generate-ministry-names.py                  # data/result/recipients.db 必要
python3 scripts/extract-ministry-from-ichiran.py            # data/download/kouji_ichiran/r071215kouji.pdf 必要

# キュレーション型（DB 不要・スクリプト内にデータ埋め込み済み）
python3 scripts/generate-ministry-supplement.py
python3 scripts/generate-municipality-supplement.py
python3 scripts/generate-police-names.py
python3 scripts/generate-foreign-agency-names.py
python3 scripts/generate-beneficiary-names.py
python3 scripts/generate-international-org-names.py
python3 scripts/generate-embassy-names.py
```

生成物は `public/data/dictionaries/` に出力（git 管理対象）。

## 7. 今後の展望

### 短期

- **カバレッジ調査スクリプトの整備**: `public/data/dictionaries/` を読み込んで自動でカバレッジを計算する `scripts/analyze-label-coverage.py` を作成する

- **法人番号照合との組み合わせ**: `houjin-lookup.json` を活用して法人番号が付与された支出先（株式会社・学校法人等）にラベルを付与し、カバレッジを大幅向上させる
- **辞書の統合 API**: `public/data/dictionaries/` を読み込んでラベルを返す TypeScript 関数の実装

### 長期

- **部分一致・正規化マッチング**: 表記ゆれ（略称・旧称・括弧書き等）へのフォールバック
- **機械学習分類**: 完全一致・法人番号照合で未解決の名称への分類器適用
