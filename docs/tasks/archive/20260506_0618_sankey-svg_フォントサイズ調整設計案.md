# sankey-svg フォントサイズ調整設計案

## 目的

`/sankey-svg` を閲覧・操作するユーザーが、ノードラベル・ツールチップ・サイドパネル・ツールバーのテキストを、Google Maps 相当の可読性で快適に読めるようにする。

---

## 現状の問題

### Google Maps との比較

Google Maps は地図 UI の可読性基準として広く参照される：

| 役割 | Google Maps | sankey-svg 現状 |
|------|-------------|-----------------|
| 主要ラベル（地名・施設名） | 13–14px | 11px（ノードラベル） |
| 補助ラベル（金額・カウント） | 11–12px | 9–10px（ツールチップ内） |
| UI コントロール・ボタン | 13–14px | 11–13px |
| サイドパネル・リスト | 13–14px | 11–12px |

sankey-svg は全体的に 1–2px 小さく、特にツールチップ内で **9px** まで下がる箇所は可読性が著しく低い。

## 設計方針

### 基本原則

1. **最小フォントサイズは 11px**（現状 9px → 11px へ引き上げ）
2. **ノードラベル基準を 12px に統一**（`labelFontPx: 11 → 12`）
3. **列ヘッダーを 14px に**（Google Maps の主要ラベル相当）
4. 右パネルとツールチップの **補助テキストは 11px 以上**

### 変更箇所一覧

#### 1. ノードラベル（SVG）

| 定数 / 変数 | ファイル | 行 | 変更 |
|-------------|----------|----|------|
| `labelFontPx` | page.tsx | 2090 | `11 → 12` |
| `labelFontPx` | page.tsx | 2222 | `11 → 12` |
| `fontSize={11 / zoom}` | page.tsx | 1966, 1994, 2005, 2058 | `11/zoom → 12/zoom` |

**レイアウト影響**: `labelFontPx` は `topShift` 計算（ノードが小さいときのラベル押し出し高さ）にも使われるため、ノード間隔が若干広がる可能性がある。LABEL_SLOT との兼ね合いを確認が必要。

## レイアウト崩れリスク

### 高リスク

| 箇所 | リスク内容 | 対処案 |
|------|-----------|--------|
| ノードラベル (`labelFontPx: 11→12`) | `topShift` が増加しノード間隔が広がる | zoom が 1× 付近の初期表示で全体が縦に伸びすぎないか確認 |
| 列ヘッダー `labelBlockH` | フォント拡大で背景ブロックが上部に食み出す | `labelBlockH` を +4px 調整して吸収 |
| ツールチップ幅 | 精細金額テキストの増大でコンテナ外にはみ出す | `whiteSpace: nowrap` は既存のまま、コンテナ右端クリップを確認 |

### 低リスク

- 右パネルは固定幅で縦スクロールあり → 文字拡大の影響は軽微
- ツールバードロップダウンは高さ固定でない → 自然に広がる

## Playwright による確認計画

既存の `tests/e2e/sankey-svg.spec.ts` に加えて、以下の視覚確認テストを追加する：

### 追加するテストケース

**スナップショット（視覚確認用）**

```
sankey-svg font size regression
```

1. 初期表示（zoom=1）でスクリーンショットを撮影し、列ヘッダーが SEARCH_BOX と重ならないことを確認
2. ズームイン 2× 後のスクリーンショットでノードラベルが適切なサイズで表示されることを確認
3. 省庁ノードをホバーしてツールチップが viewport 内に収まることを確認
4. 右パネル展開状態で右パネルの文字が overflow していないことを確認

**数値アサーション（CI-safe）**

- 列ヘッダー div の `fontSize` が `14px` であることを `getComputedStyle` で検証
- ノードラベル `<text>` の `font-size` 属性が `12/zoom` であることを検証
- ツールチップ内の最小フォントサイズが `11px` 以上であることをチェック

### 実行方法

```bash
npx playwright test tests/e2e/sankey-svg.spec.ts
```

ビジュアルリグレッションを使う場合:
```bash
npx playwright test --update-snapshots  # ベースライン更新
npx playwright test                      # 比較実行
```

---

## 実装順序

1. **定数変更**（`labelFontPx`）→ `topShift` 計算への影響をまず確認
2. **ツールチップの 9px → 11px 修正**（リスク最小・効果大）
3. **列ヘッダーの 13 → 14px + `labelBlockH` 調整**
4. **ツールバードロップダウン 11 → 12px**
5. **Playwright で各ステップ後に視覚確認**
