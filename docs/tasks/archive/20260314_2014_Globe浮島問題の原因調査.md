# Globe浮島問題の原因調査

## 目的

Globe可視化のパンゲア配置で、一部の小規模府省庁が大陸から離れた「浮島」として表示される問題の原因を特定する。

## 現象

- `水産庁`（603面）以下の小規模府省庁が、他の府省庁の領域内に飛び地として出現
- `消防庁`（41面）以下の極小府省庁が、海（厚労省）の中に孤島として出現
- パンゲア（超大陸）のはずが、小さな島が散在する見た目になっている

## 原因分析

### 原因1: seed配置と成長順の構造的矛盾

```
大陸中心(30°,20°)                        キャップ縁(~78°)
    |                                        |
    ■ 経産省(9°)                              |
    ■ 国交省(16°)                             |
    ■ 文科省(20°)                             |
    ■ 防衛省(24°)                             |
    ...                                      |
    □ 水産庁(50°)                             |
    ...                                      |
    □ 消防庁(67°)  ← 成長順6（早い）            |
    □ 消費者庁(70°) ← 成長順5（早い）            |
    □ 国税庁(71°)  ← 成長順4（早い）            |
    □ 金融庁(73°)  ← 成長順3（早い）            |
    ○ 公取委(74°)  ← 成長順0（最初）            |
    ○ 公安調(76°)  ← 成長順1                   |
    ○ その他(77°)  ← 成長順2                   |
                                      ──── 海 ────
```

- `fibonacciCap`はN点をキャップ内に均等配置するため、面数が少ない府省庁ほどキャップの外縁に配置される
  - 理由: 府省庁はAPIで支出額降順にソートされ（index 1=経産省, 2=国交省...）、seedは`continentSeeds[i-1]`で対応。Fibonacci格子は中心から外へ螺旋状に配置するため、**後ろのインデックス=小規模=外縁**
- 一方、成長順は面数昇順（小が先）なので、**外縁の小規模府省庁が最初に成長**する
- 周囲がすべて未割り当てのため、コンパクトな円形の島を形成
- その後、中心部の大規模府省庁が成長し、外側に向かって拡大
- **大規模府省庁のBFSが小規模府省庁の島を包囲・分断**する

### 原因2: 包囲後の断片化と再接続の限界

包囲された小規模府省庁の島は、connectivity fixで以下の処理を受ける：

1. **断片検出**: BFSで連結成分を検出し、最大成分以外を隣接府省庁に譲渡
2. **面の奪還**: 失った面数分、主成分の境界からBFS拡張で面を取り戻す

しかし**奪還制約**がある：

```typescript
// Only steal from ocean to avoid cascading island issues
if (assignment[nb] === oceanIdx) {
    assignment[nb] = m;
    ...
}
```

**海（厚労省）からしか奪還できない。** 小規模府省庁が他の非海府省庁に包囲されている場合、境界面はすべて非海府省庁であり、奪還が一切発生しない。

### 原因3: 「海の中の孤島」パターン

消防庁（41面、seed距離67°）のライフサイクル：

1. **成長順6で早期に成長** → 周囲未割り当て → コンパクトな41面の島
2. seedが大陸中心から67°離れているため、**大陸本体とは空間的に離れている**
3. 大規模府省庁（経産省→防衛省→...）が中心から外に成長
4. 消防庁のseed付近まで到達する前に成長制限に達するか、到達しても消防庁を**飛び越えて**成長
5. 最終的に海（厚労省）が残りを埋める
6. 消防庁は**海に囲まれた孤島**として残る

### 原因4: 「他府省庁内の飛び地」パターン

水産庁（603面、seed距離50°）のケース：

1. **成長順17で中期に成長** → 周囲は一部割り当て済み
2. 大規模府省庁（防衛省11,781面、農水省6,052面など）の成長圏内にseedがある
3. 水産庁の島が形成された後、防衛省・農水省のBFSが周囲を埋める
4. **水産庁の島が他の大陸府省庁の領域内に取り残される**
5. 断片が発生しても、周囲は非海府省庁なので奪還不可

## 根本原因のまとめ

```
fibonacciCap: 支出額順にseed配置 → 小規模ほど外縁
         ↓
BFS成長: 面数昇順 → 小規模ほど先に成長
         ↓
結果: 外縁で小さな島が先に形成され、後から大規模が包囲
         ↓
connectivity fix: 海からしか奪還できず、非海に包囲されると救済不能
```

**パンゲア（連続大陸）の実現には、小規模府省庁を大陸本体に「接着」させる仕組みが必要。**
現在のアルゴリズムは各府省庁を独立にseedから成長させるため、大陸内の位置関係（隣接性）が保証されていない。

## 改善の方向性（実装前メモ）

以下は検討すべきアプローチの方向性：

1. **成長順の見直し**: 大→小の順に成長させ、小規模は大陸の隙間に充填する方式
2. **大陸ベースの成長**: まず全非海府省庁を1つの「大陸ブロック」として成長させ、その後ブロック内をseedベースで分割
3. **奪還制約の緩和**: 非海府省庁からも奪還可能にし、連鎖的な再配置を行う
4. **seed配置の改善**: 小規模府省庁のseedを大陸中心寄りに配置する
