# Globe Phase 4B — 再帰二分割ツリーマップ方式による領域割り当て

## 目的

Globe可視化において、府省庁ごとの支出額に比例した面積を球面上に**矩形に近い形状**で割り当てる。
従来のBFS flood fill方式では経路ブロックによる浮島や面数の過不足が発生していたため、ツリーマップ的な機械的手法で置き換える。

## アルゴリズム概要

3フェーズ構成で327,680面（icosphere level 7）を33府省庁に割り当てる。

### Phase 1: 元面ベース海陸分離

icosphere level 7の327,680面は、元の正二十面体20面を各4^7=16,384面に細分割したもの。
`floor(faceIndex / 16384)` で任意の面が属する元面IDを特定できる。

1. 20元面の重心を計算
2. 大陸中心（経産省seed位置 `(30°E, 11°N)`）からの距離でソート
3. `continentFaceCount / FACES_PER_ORIGINAL` で必要な元面数を算出
4. 近い順に大陸、遠い順に海（厚生労働省=65.7%=oceanIdx:0）
5. 境界元面は内部の面を距離でソートし、必要面数だけ大陸に含める

**結果**: 海=215,264面 / 大陸=112,416面（面レベルで正確に分離）

### Phase 2: 再帰二分割ツリーマップ + エッジゾーン

#### 2D射影

大陸面を接平面上のローカル座標(u, v)に射影する:
- 法線ベクトル: 大陸中心方向 `(cnx, cny, cnz)`
- u軸（接線）: `up × normal` を正規化
- v軸（従法線）: `normal × tangent`
- 各面の(u, v) = 重心ベクトルを接線・従法線に射影

#### ツリーマップ分割

面リストを縦横交互にソート→面積比で二分割する再帰アルゴリズム:

1. 面リストを現在の軸(u or v)でソート
2. 府省庁リストを面積の半分で二分（前半/後半）
3. 面リストを面積比 `firstArea / totalArea` の位置で分割
4. 前半面→前半府省庁、後半面→後半府省庁で再帰
5. 府省庁が1つになったら、残り全面をその府省庁に割り当て

**面数ベース**で分割するため、球面の不均一密度に影響されない。

#### エッジゾーン（小規模府省庁の処理）

面積が小さい府省庁（< 1,200面 = `EDGE_THRESHOLD`）は矩形を維持できないため、
ツリーマップから外して大陸海岸線沿いに配置する:

1. 小規模府省庁の合計面積を「エッジゾーン」として1ブロックにまとめ、ツリーマップに参加
2. ツリーマップ後、エッジゾーンに割り当たった面を収集
3. 海に隣接するエッジ面をシードにBFS順序を構築
4. BFS順に小規模府省庁を面積降順で連続配置

**対象府省庁**（現在のデータで11省庁）:
復興庁(1,136), 国土交通省観光庁(1,019), 環境省(1,001), 財務省(991), 林野庁(815),
水産庁(603), 法務省(570), 国土交通省海上保安庁(521), 警察庁(402), デジタル庁(368), ...

### Phase 3: 境界滑らか化

ツリーマップの境界はソート軸に沿った直線的な切れ目になるため、局所スワップで滑らかにする:

1. 各面について隣接3面の府省庁を集計
2. 多数派が自分と異なる場合、スワップ候補
3. スワップ後の面数が目標値の±2%以内なら実行
4. 最大5イテレーション、スワップ0で早期終了

## ファイル構成

| ファイル | 役割 |
|---------|------|
| `client/components/GlobeView.tsx` | メインGlobe描画。`assignFaces()`内に3フェーズ実装 |
| `app/api/map/globe/route.ts` | Globe API。`fibonacciCap()`でseed生成、33府省庁データ提供 |
| `scripts/globe-assignment-check.mjs` | 検証スクリプト。`assignFaces()`を純JSで再現しCSV出力 |

## 定数

| 定数 | 値 | 意味 |
|------|-----|------|
| `ICO_SUBDIVISIONS` | 7 | icosphere分割レベル |
| `FACES_PER_ORIGINAL` | 16,384 (4^7) | 元面1つあたりの面数 |
| `SPHERE_RADIUS` | 1 | 球の半径 |
| `EDGE_THRESHOLD` | 1,200 | エッジゾーン判定閾値（面数） |
| `SMOOTH_ITERATIONS` | 5 | 滑らか化の最大反復回数 |
| `maxDeviation` | target × 2% | 滑らか化時の面数許容誤差 |
| `oceanIdx` | 0 | 厚生労働省 = 海（最大支出） |

## 検証結果

検証スクリプト実行結果（2026-03-15時点）:

- **全33府省庁が割り当て済み**（未割り当て面なし）
- **大規模〜中規模府省庁**: 全て1コンポーネント、面数差±15以内
- **小規模府省庁（エッジゾーン）**: 大半が1コンポーネント
- **極小府省庁**（金融庁10面、公正取引委員会3面）: 2-3コンポーネントの断片化あり

## 既知の課題

- **エッジゾーンのストライプ化**: 小規模府省庁がBFS順で海岸線に沿って細長いストライプ状に配置される。海岸線の幅が狭いため、各府省庁が帯状になり視認性が低下する
- **極小府省庁の断片化**: 3-10面程度の府省庁は滑らか化フェーズで断片化しやすい
