# 再委託チェーン — パネル階層表示

## 目的

/sankey2 のユーザーが、事業(支出)ノードを選択したとき、サイドパネルで直接支出と再委託の流れを分離して把握できるようにするため。

## 現状の課題

### グラフのsubcontractエッジにprojectIdがない

現在のsubcontractエッジは `recipient → recipient` の集約値であり、どの事業に関連するか判別できない。事業選択時に「この事業の再委託チェーン」を表示するには、事業ID単位の情報が必要。

### structured.jsonにはデータがある

`structured.json` の `outflows` には事業ID付きの再委託情報が存在する:

```json
{
  "projectId": 18,
  "sourceBlockName": "一般社団法人行政情報システム研究所",
  "targetBlockName": "富士通株式会社",
  "amount": 982320000,
  "recipients": [{ "name": "富士通株式会社", "amount": 982320000 }]
}
```

### 金額の内訳が不明

PID=18の場合、支出合計50.68億のうち:
- 直接支出: 28.01億（行政情報システム研究所のみ）
- 間接支出: 22.67億（再委託経由で富士通、富士通エフサス等に流れた分）

現在のパネルでは全支出先が同列で並び、この区別がつかない。

## 改善案

### 前提: グラフ構造・BFS・レイアウトは変更しない

[失敗記録](20260322_0537_再委託チェーン表示改善_失敗記録.md)に基づき、パネル表示のみで対応する。

### A. 再委託チェーンデータの事前計算

**対象**: `scripts/generate-sankey2-data.ts`

`structured.json` の `outflows` から事業IDごとの再委託フローを抽出し、グラフJSONに含める。

**新フィールド**: `Sankey2Graph.subcontractChains`

```typescript
interface SubcontractFlow {
  from: string;       // 委託元の支出先名
  to: string;         // 委託先ブロック名（「〜ほか」等）
  amount: number;
  recipients: { name: string; amount: number }[];
}

interface SubcontractChain {
  projectId: number;
  flows: SubcontractFlow[];
}
```

### B. サイドパネルの表示（project-spending選択時）

**対象**: `client/components/Sankey2/Sankey2View.tsx` の `DetailPanel`

`project-spending` ノード選択時、既存の「流出先」を「直接支出」と「再委託チェーン」に分離。

**表示イメージ**:

```
政府共通ネットワーク(情報通信技術調達等適正・効率化推進費)
支出合計: 50.68億

━━━ 直接支出 28.01億 ━━━

→ 一般社団法人行政情報システム研究所  28.01億

━━━ 再委託チェーン 18.82億 ━━━

一般社団法人行政情報システム研究所
├── → 富士通株式会社  9.82億
│   ├── → 富士通NW  7.22億
│   ├── → コマス  0.12億
│   └── → ジャステック  0.13億
├── → NTTデータグループほか  2.53億
│   ├── → NTTコミュニケーションズ  0.43億
│   └── (計16社)
└── → 民間企業Eほか  1.18億

富士通エフサス（2次再委託）
└── → JEXENSほか  1.53億
    ├── → ミツイワ  0.04億
    └── (計7社)
```

**ヘッダー部分の金額表示**:

| 項目 | 表示 |
|------|------|
| 支出合計 | 全支出先の合計（直接+間接） |
| 直接支出 | `isDirectFromGov === true` の合計 |
| 再委託合計 | outflowsのrecipients金額合計 |

**ツリー構築ロジック**:

1. `projectId` で `subcontractChains` から該当チェーンを取得
2. flowsの `from` フィールドでグループ化（委託元別）
3. 各委託元について `recipients` を子ノードとして表示
4. 子ノードの名前が別のflowの `from` に一致すれば、さらに再帰的に子を展開（多段再委託）

**インタラクション**:
- 各支出先名はクリック可能（`onNodeClick` でグラフ上の `recipient-{name}` にジャンプ）
- 5件以上の子は折りたたみ表示（「(計N社)」をクリックで展開）

### C. データサイズの影響

| 指標 | 値 |
|------|------|
| outflowを持つ支出先 | 約1,000件 |
| 総outflow数 | 約3,000件 |
| 追加データサイズ（概算） | 〜200KB（非圧縮）、〜30KB（gzip後） |

| `scripts/generate-sankey2-data.ts` | `subcontractChains` フィールドの生成 |

- グラフのノード・エッジ構造 — BFSハイライトロジックに影響なし
- `public/data/sankey2-layout.json.gz` — 再生成不要
