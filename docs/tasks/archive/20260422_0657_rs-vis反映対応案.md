## 対応方針（最低限の移行）

### 方針：rsync ベースの上書きコピー（docs/ 除外）

差分を個別にマージするより、marumie を正として rs-vis のソースコードを上書きする。両リポジトリの `app/sankey/` と `app/lib/sankey-generator.ts` は大きく乖離しており（差分1,200行超）、個別マージは現実的でない。

### 対象外とするもの

| 対象外 | 理由 |
|--------|------|
| `docs/` | ユーザー指定 |
| `node_modules/` | 環境依存 |
| `public/data/*.json`（非圧縮） | .gitignore対象・ビルド生成物 |
| `data/` | CSV生成物・.gitignore対象 |
| `.git/` | リポジトリメタデータ |
| `README.md` | rs-vis独自の説明に差し替えが必要な可能性あり（確認要） |

### 対象外の判断が必要なもの（要確認）

| ファイル | 確認事項 |
|----------|----------|
| `README.md` | rs-vis 側に独自の公開向け記述がある場合は上書き不可 |
| `.env*` / `vercel.json` | rs-vis の環境設定が異なる可能性（vercel.json は同一確認済） |
| `walkthrough.md` | rs-vis のみ存在・marumie には不在 |
| `CLAUDE.md` | marumie 固有の開発メモが含まれる場合は除外 |

## 実施手順案

### Step 1: ソースコードのコピー

```bash
rsync -av --exclude='.git' --exclude='node_modules' --exclude='docs' \
  --exclude='public/data/*.json' --exclude='data/' --exclude='README.md' \
  --exclude='walkthrough.md' \
  /Users/igomuni/MyGitHub/marumie-rssystem/ \
  /Users/igomuni/MyGitHub/rs-vis/
```

### Step 2: データファイルのコピー（.gz のみ）

必要な .gz ファイルを marumie の `public/data/` から rs-vis へコピー。  
`rs2024-structured.json.gz` は rs-vis 側にすでに存在するため上書き確認が必要。

```bash
cp /Users/igomuni/MyGitHub/marumie-rssystem/public/data/*.gz \
   /Users/igomuni/MyGitHub/rs-vis/public/data/
```

サブディレクトリ（`dictionaries/` 等）は移行対象ページが使用する場合のみコピー。

### Step 3: 依存関係の更新

```bash
cd /Users/igomuni/MyGitHub/rs-vis
npm install
```

### Step 4: ビルド確認

```bash
npm run build
```

prebuild フックで `.gz` → `.json` 展開が走ることを確認。

### Step 5: README.md の判断

`walkthrough.md` と `README.md` を比較し、rs-vis 向けの公開説明として残すか marumie 版で上書きするかを判断する。

## 留意事項

- **`app/sankey/page.tsx` と `sankey-generator.ts` は rs-vis 側が古い**。上書きにより rs-vis がすべて最新版に置き換わるため、上書き後は /sankey の動作確認が必須。
- **`CLAUDE.md`**: marumie のルート `CLAUDE.md` には開発内部向けのガイドが含まれる。rs-vis での公開運用に不要な記述（法人番号照合、内部ツール手順等）を含む場合は除外または整理が必要。
- **`scripts/` の分析系スクリプト**: marumie には調査用の `analyze-*.ts` / `analyze-*.py` が多数ある。これらはリリース版に不要な場合もあるが、移行コスト最小化のため一括コピーで問題ない（`.gitignore` や `package.json` の scripts に載っていない限りビルドには影響しない）。
