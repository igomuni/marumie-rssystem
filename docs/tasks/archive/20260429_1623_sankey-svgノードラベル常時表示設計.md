# /sankey-svg ノードラベル常時表示 設計

## 目的

ユーザーが「すべてのノードラベルを表示」トグルを操作しなくても常にラベルが読める状態にする。設定項目を減らしてUIをシンプルにする。

---

## 現状

### `showLabels` の役割

`showLabels`（URL param `sl`）は2つの箇所に影響している。

**① レイアウト計算（`page.tsx` ~L772）**

```text
showLabels=false → computeLayout(... /* minNodeGap なし */)
showLabels=true  → 2パス計算:
                    1) noGap = computeLayout(... /* minNodeGap なし */) で fitZoom を算出
                    2) computeLayout(... , minNodeGap = 14 / fitZoom)
```

2パス目の `minNodeGap = 14 / fitZoom` により、小さいノードの後ろのギャップが最低 14px（視覚）まで拡張される。これはノード高さは変えず、**ギャップのみ** を広げる。

**② ラベル描画（`page.tsx` ~L1739, ~L1811）**

```text
labelVisible = showLabels || (h + NODE_PAD) * zoom > 10 || isSelected
```

`showLabels=false` のとき、視覚高さが 10px 未満のノードはラベルを描画しない。

### `minNodeGap` の仕組み（`sankey-svg-filter.ts` L777〜）

```text
effectivePad = max(NODE_PAD, minNodeGap)

各ノードのギャップ:
  effectiveH < effectivePad → gap = effectivePad  // 拡張
  effectiveH ≥ effectivePad → gap = NODE_PAD      // 通常
```

ノード高さ（`effectiveH`）は変更されない。ギャップだけ広がる。

## 実装方針（採用した方針）

当初は `showLabels` の完全廃止を検討したが、既存の UI 互換性を保ちつつ段階的に進めるため、以下の方針を採用した。

| 項目 | 変更前 | 変更後（実装） |
|---|---|---|
| `showLabels` 初期値 | `true` | **`false`**（OFFをデフォルトに） |
| URLパラメータ `sl` | あり | 維持（後方互換） |
| UIチェックボックス | あり | 維持（手動 ON 可能） |
| レイアウト計算 | 条件分岐 | 常に2パス（extraRecipientGap / extraMinistryGap も追加） |
| ラベル表示方法 | showLabels=false で非表示 | **topShift** でノードを下方シフト→ラベルスペースを確保 |
| 縦方向フィット | fitZoom でビューポートに収める | shift 高さを含めて fitZoomWithShifts で計算 |

### topShift による表示方法

`showLabels=false` のまま、視覚高さが `LABEL_SLOT=12px` 未満のノードを描画上で下方向にシフトしてラベル空間を確保する。

```ts
// nodeShiftInfo（列ごとの累積シフト計算）
const topShift = h * zoom < LABEL_SLOT ? Math.max(0, LABEL_SLOT / zoom - h) : 0;
// <g> の translateY に cumShift を加算、rect/label の y に topShift を加算
```

- エッジの y 座標も `shiftedRibbonPath` で補正
- `project-budget` ノードは `max(予算高さ, 支出高さ)` を基準に topShift を決定

### 列間隔自動調整

`computeLayout` に `extraRecipientGapSVG` / `extraMinistryGapSVG` を追加し、fitZoom 基準でラベルスペースを確保する。

```ts
const extraRecipientGapSVG = 360 / fitZoom;  // 事業支出側 → 支出先
const extraMinistryGapSVG  = 310 / fitZoom;  // 府省庁 → 事業
```

## 注意事項

- `showLabels` state・`sl` URL param・UI チェックボックスは残存（手動での全ラベル表示が可能）。
- `topShift` はレンダリング座標のみを変更する。フォーカス・オーバーレイは引き続き生のレイアウト座標を参照しているため、ノードが大きくシフトした場合にフォーカス位置がずれる可能性がある（後続タスクで対応予定）。
- `fitZoomWithShifts` は収束ループ（最大6回）で zoom ↔ shift の循環依存を解消している。
