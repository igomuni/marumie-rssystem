# /sankey-svg 会計区分・予算執行データ構造の深堀り

作成日: 2026-05-16

関連Issue: feat: `/sankey-svg` 事業の会計区分内訳を表示 #188

## 背景

前回調査で、`2-1` は事業単位の総額・執行額、`2-2` は会計区分付きの予算内訳に向いていることが分かった。
ただし、実際にサイドパネルへ表示する情報は多く、単一の「予算・執行」タブに詰めると読みにくくなる。

このメモでは、会計区分と予算執行データをどう切ると理解しやすいかを整理し、サンプル事業で分類を試す。

---

## まず分けるべきレイヤー

予算・執行・支出先データは、同じ「お金」でも意味が違う。
UIでは以下のレイヤーを混ぜないほうがよい。

| レイヤー | 主CSV | 主な列 | 意味 |
|---------|-------|--------|------|
| 年度コンテキスト | `2-1`, `2-2` | `事業年度`, `予算年度` | 表示年度と予算年度の対応。`/sankey-svg` では基本 `YEAR - 1` |
| 予算・執行サマリ | `2-1` | 当初予算、補正予算、繰越し、予備費、予算現額、執行額、執行率、翌年度要求額 | 事業単位の総額 |
| 会計区分・会計・勘定 | `2-2` | 会計区分、会計、勘定、予算額 | どの会計から予算が出ているか |
| 予算種別 | `2-2` | 予算種別、予算額 | 当初・補正・繰越し・予備費等の構成 |
| 歳出項目 | `2-2` | 所管、組織・勘定、項、目、補足情報、予算額 | 財政上の費目・科目 |
| 執行・支出先 | `2-1`, `5-1`, `5-2` | 執行額、支出先、金額、担当組織からの支出 | 実際にどこへ出ているか。直接・間接の区別が必要 |
| データ品質・照合 | `2-1`, `2-2`, `5-1`, `5-2` | 各合計差分、0円行、支出先単純合計 | 誤読・重複・異常値の検出 |

特に重要なのは、`2-2` は会計区分別の予算額を持つが、会計区分別の執行額は持たないこと。
執行額は `2-1` の事業合計として見る。

## 推奨する情報設計

単一タブではなく、事業詳細の中に以下の小セクションまたはサブタブを分けるのがよい。

### 1. 概況

目的:
事業単位の全体像をすぐ把握する。

表示候補:

- PID
- 事業名
- 府省庁
- 予算年度
- 予算現額
- 執行額
- 執行率
- 翌年度要求額
- 直接支出額
- 間接支出額
- 支出先数

注意:
直接支出額は `5-1` と `5-2` の判定に依存する。
`5-1` の金額を単純合計した値は、執行額と一致しないことがある。

### 2. 会計

目的:
一般会計・特別会計・会計・勘定の構成を見る。

表示候補:

- 会計区分別予算額
- 会計別予算額
- 勘定別予算額
- 0円の会計メタデータ行の有無
- `accountCategory` フィルタ分類との差分

このセクションはIssue #188の中心。

注意:
`/sankey-svg` の会計区分バッジと会計区分フィルタは、表示対象の予算年度に揃える必要がある。
2024データなら予算年度2023、2025データなら予算年度2024を使う。
全年度の `2-1` 行から会計区分を拾うと、当年度や過年度のメタデータが混ざり、表示中の予算額とバッジがズレる。

2025データでは、全年度の `2-1` から判定した場合と、表示対象の予算年度2024だけから判定した場合で556事業の分類が変わった。
2024データでも666事業が変わる。
したがって `accountCategory` は `TARGET_BUDGET_YEAR` の行から生成する。

なお、さらに厳密に「金額がある会計区分」だけを見るなら `2-2` の `予算額(歳出予算項目ごと) != 0` を使う案もある。
ただし予算0円事業や0円メタデータ行の扱いが変わるため、これはバッジの意味を「会計メタデータ」から「金額付き会計区分」へ変える別設計として扱う。

### 3. 予算種別

目的:
予算が当初予算なのか、補正・繰越し・予備費なのかを見る。

表示候補:

- 当初予算
- 補正予算
- 前年度から繰越し
- 予備費等
- 予算種別ごとの歳出項目数
- マイナス予備費等のような調整行

防衛・公共事業・災害復旧系はここが複雑になりやすい。

### 4. 歳出項目

目的:
財政上の項・目レベルで予算の中身を見る。

表示候補:

- 所管
- 組織・勘定
- 項
- 目
- 歳出予算項目の補足情報
- 予算額
- 翌年度要求額

表示方法:
全行をただ並べるより、会計区分 → 予算種別 → 項・目の階層で折りたたむのがよい。

### 5. 執行・支出先

目的:
予算・執行額と支出先フローの関係を見る。

表示候補:

- `2-1` 執行額
- `5-1/5-2` 直接支出額
- `5-1/5-2` 間接支出額
- 直接支出先Top
- 間接支出先Top
- `5-1` 単純合計

注意:
支出先は直接・間接を分けないと、予算現額や執行額との比較が不安定になる。
`/sankey-svg` 本体は直接支出先中心なので、間接支出は「別レイヤー」として扱う。

### 5-補足. 「間接支出」と「再委託」の関係

このメモでは一部で「間接支出」という言葉を使っているが、厳密には注意が必要。

`/sankey-svg` の生成処理では、`5-2` の `担当組織からの支出=TRUE` でないブロックを直接支出から除外している。
この除外分を雑に「間接支出」と呼ぶことはできるが、ドメイン上はすべてが再委託とは限らない。

既存の `/subcontracts` データでは、ブロックを以下のように分けている。

| 分類 | 意味 | 再委託との関係 |
|------|------|----------------|
| `direct` | 担当組織から直接支出されたブロック | 再委託ではない |
| `subcontract` | 直接ブロックまたは再委託ブロックの下流にある通常のブロック | 狭義の再委託に近い |
| `separate-origin-broad` | 支出元として登場するが、直接支出ブロックでも通常の下流でもない広めの別起点 | 再委託と断定できない |
| `separate-origin-strong` | 別起点のうち、下流で他支出元と合流する強い別起点 | 再委託ではなく、別財源・借入・回収金等の可能性が高い |
| `indirectCosts` | `5-2` の「国自らが支出する間接経費」 | 再委託フローとは別枠 |

したがって、UI文言は以下のように分けるほうがよい。

| 避けたい表現 | 推奨表現 |
|-------------|----------|
| 間接支出 = 再委託 | 下流ブロック支出 |
| 間接支出額 | 直接外ブロック支出、または再委託等 |
| 再委託額 | `originKind === subcontract` のブロック合計に限定 |
| 支出計 - 直接支出 | 再委託・別起点等を含む下流ブロック合計 |

2025データの `subcontracts-2025.json` で集計すると以下になる。

| 分類 | 金額 | 備考 |
|------|------|------|
| 直接ブロック | 126.88兆円 | `/sankey-svg` の直接支出に近い |
| 再委託ブロック | 26.18兆円 | 通常の下流ブロック |
| 別起点ブロック | 3.46兆円 | 再委託と呼ぶと危険 |
| うち強い別起点 | 1.84兆円 | 他支出元と合流する別起点 |
| 国自らが支出する間接経費 | 418.37億円 | フロー図とは別の独立経費 |

このため、サイドパネルの「執行・支出先」では最低でも以下の4つを分ける。

- 直接支出
- 再委託ブロック
- 別起点ブロック
- 国自らが支出する間接経費

特に `directExpenseTotal` と `totalExpense` の差分をそのまま「再委託」と表示してはいけない。
既存の `/subcontracts` 一覧でも `支出計−直接` は「再委託・別財源など下流ブロック分」と説明しており、この表現に寄せるのが安全。

### 6. 照合・注意点

目的:
データの読み方を誤らないためのチェックをまとめる。

表示候補:

- `2-1` 予算現額と `2-2` 予算額合計の差分
- `2-1` 執行額と直接支出額の差分
- 直接支出額と支出先単純合計の違い
- 予算0円・支出あり
- 会計区分0円行
- `2-1` 会計メタデータと `2-2` 金額付き会計区分の差分

## 分類軸

サンプル事業を見た結果、少なくとも以下の分類軸がある。

| 分類軸 | 値の例 | 見るべき理由 |
|-------|--------|--------------|
| 会計構成 | 一般会計のみ / 特別会計のみ / 一般+特別 / 複数特別会計 | 財源・制度の違いが見える |
| 予算種別構成 | 当初中心 / 補正中心 / 繰越し大 / 予備費あり / 複数混在 | 政策の通常運用か、補正・災害・年度またぎかが見える |
| 歳出項目粒度 | 1項目 / 少数項目 / 多数項目 | UIで要約表示が必要か判断できる |
| 支出構造 | 直接中心 / 間接中心 / 直接+間接混在 / 支出先単純合計が大きい | Sankeyの表示値と予算執行表の対応を説明する必要がある |
| 執行状況 | ほぼ100% / 低執行 / 繰越し大 / 予算0・支出あり | 注意表示や並び替えに使える |
| データ品質 | 0円行あり / マイナス調整あり / 2-1と2-2差分 / 支出先合計の乖離 | 読み手への注記が必要 |

したがって「再委託が多い」ではなく「支出先データ照合注意」と分類する。

### PID 1406: 独立行政法人国際協力機構有償資金協力部門への出資

- 支出構造: 直接 + 再委託 + 別起点

## UIへの示唆

現在の「予算・執行」タブ案は、最初の入口としては使えるが、情報の構造としては分割したほうがよい。

候補:

| タブ/セクション | 主対象 | 内容 |
|----------------|--------|------|
| 概況 | 全事業 | 予算現額、執行額、執行率、翌年度要求、直接/間接支出 |
| 会計 | 混在・特別会計事業 | 一般/特別、会計、勘定、金額あり/メタデータのみ |
| 予算種別 | 補正・繰越し・予備費が多い事業 | 当初、補正、繰越し、予備費等 |
| 歳出項目 | 項目が多い事業 | 項・目の内訳、所管、組織・勘定 |
| 支出先照合 | 支出構造が複雑な事業 | 直接/間接、支出先Top、執行額との差分 |
| 注意点 | 異常・乖離がある事業 | 0円行、マイナス行、合計差、支出先金額の乖離 |

コンパクトにするなら、タブではなく「折りたたみセクション」でもよい。
特にサイドパネル幅が狭いので、タブ数を増やしすぎるより、事業タイプに応じて重要セクションを上に出すほうがよい。

## 次の対応案

### 短期

- 現在の「予算・執行」タブを、単一の長いリストから以下の見出しに分ける。
  - 概況
  - 会計
  - 予算種別
  - 歳出項目
  - 照合
- `2-2` の0円行を「金額あり」「メタデータのみ」に分ける。
- 会計区分サマリは `2-2` の金額付き行を優先する。
- `2-1` の `accountCategory` はフィルタ・バッジ用、金額内訳は `2-2` 用と明示する。

### 中期

- 事業タイプを自動分類し、重要セクションを上に出す。
- 直接/間接支出のサマリをサイドパネルに加える。
- `2-1` 執行額、直接支出額、間接支出額、支出先単純合計を照合する。
- 乖離が大きい事業には注意表示を出す。

### 長期

- 省庁単位・支出先単位の会計/予算種別内訳を集計表示する。
- 年度推移を追加し、同一PIDの予算現額・執行額・要求額の推移を見る。
- 支出先ビューと予算項目ビューを行き来できるようにする。

## 実装済みタブへの評価

現在実装済みの「予算・執行」タブは、`2-1` と `2-2` の素材を表示する最初の実装としては有効。
ただし、今回の深堀りを見る限り、最終形としては以下の改善が必要。

- 「予算・執行」という1タブではなく、内部を構造化する。
- 会計区分だけを中心にしすぎない。予算種別・歳出項目・直接/間接支出も同じくらい重要。
- 支出先金額は執行額と常に一致するわけではないので、照合セクションを別にする。
- 0円会計行やマイナス予備費等をデータ品質として扱う。

したがって、次の実装は「タブ追加」よりも「予算・執行タブ内部の情報設計の再構成」を優先するのがよい。
