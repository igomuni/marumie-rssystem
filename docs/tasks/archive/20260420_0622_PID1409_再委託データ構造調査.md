## データ構造の本質的な問題

### 1. 「参考標記フロー」が実フローと同じ形式で記載されている

5-2 には本来「再委託チェーン（下流への支出）」を記述するが、このデータでは**保険事業のメカニズム説明**として双方向・循環フローが記載されている。

```
財務省 ──[出資金]──▶ A(日本政策金融公庫)
         A ──[保険金支払 ※参考]──▶ B(信用保証協会)
         B ──[保険料支払]──────▶ A  ← 逆向き（Aに戻る）
         B ──[代位弁済]─────────▶ C(金融機関)
         C ──[融資]──────────── ▶ D(中小企業等)
         D ──[保証料支払弁済 ※参考]▶ B  ← 別サイクル
```

### 2. 参考標記の意味

補足欄の「(事業全体を把握するための参考標記。以下同様。)」は、**その接続は実際の国庫支出ではなく、事業の仕組みを説明するために記載した参考情報**であることを示している。

実質的な国庫支出は：
- 財務省 → A（46.7億円の出資金）のみ

### 3. 可視化への影響

現在の `/subcontracts/[projectId]` は参考標記フローと実フローを区別しないため：
- 循環グラフが発生（A→B→A, D→B）
- BFS がサイクルを巡回し続ける（修正済み：最小深さ採用で対処）
- ブロックB・C・Dには金額ゼロで支出先も空なのにノードが表示される

## 類似ケースの可能性

このパターン（参考標記付きフロー）は他の事業にも存在する可能性がある。  
以下のような特徴を持つプロジェクトは同様の構造を持つ可能性がある：

- `maxDepth` が MAX_DEPTH_LIMIT(30) に達している事業  
- 補足情報に「参考標記」「以下同様」を含む接続が存在する事業
- 5-1 の支出先が空のブロックが多い事業

---

## 対応案

### 短期（表示改善）
- 補足情報に「参考標記」を含むフローを破線表示・グレーアウトするなど視覚的に区別
- 支出先が空のブロックを「参考ブロック」として別スタイルで表示

### 中期（データ処理）
- `generate-subcontracts.ts` で「参考標記」フローにフラグを付与（`isReference: true`）
- 自己ループ（sourceBlock === targetBlock）を生成時に除去
- レイアウト計算・ノード表示時に参考フローを除外するオプションを提供

### 根本（データ側）
- RS System の 5-2 CSV において「参考標記」フローを実フローと分離する列の追加が望ましいが、データ提供元の仕様変更が必要
