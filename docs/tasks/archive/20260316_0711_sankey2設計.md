# /sankey2 設計ドキュメント

## 目的

予算・支出データの閲覧者が、TopN集約やドリルダウンの段階的操作なく、全事業・全支出先をスライダーとZoom/Panでシームレスに探索できるようにするため。

## 現行 /sankey の課題

| 課題 | 詳細 |
|------|------|
| TopN集約 + ドリルダウン | 固定件数でのページネーション。全体像と詳細を同時に見られない |
| 列レイヤー制約 | 5列固定で縦に長くなる。画面に収めるために情報量を犠牲にしている |
| APIリクエスト駆動 | ビュー切り替えごとにサーバーサイドで再計算。操作のたびに待ち時間が発生 |
| データ依存 | `rs2024-structured.json`（5,003事業）に依存。元CSVの5,664事業との差分が不透明 |

## 設計方針

### 既存との関係

| 項目 | 方針 |
|------|------|
| ページパス | `/sankey2`（既存 `/sankey` は維持） |
| データ | 新規パイプラインで生成。`rs2024-structured.json` とは独立 |
| 描画 | SVG（@nivo/sankey は使わず、自前SVG描画） |
| 参考実装 | `../../budget-flow-map`（deck.gl + Zustand）のUXパターンを参考 |

### データパイプライン再設計

#### 基準データ

元CSV `data/year_2024/` から直接生成する。既存の `generate-structured.js` とは別スクリプト。

| 元CSV | 用途 |
|-------|------|
| `1-1` 組織情報 | 府省庁 → 部局 → 課室の階層 |
| `2-1` 予算・執行サマリ | 事業別の予算額・執行額 |
| `5-1` 支出先・支出情報 | 支出先名・金額・法人番号・契約方式 |
| `5-2` 支出ブロックのつながり | 再委託構造（ブロック階層） |

#### 生成物

新スクリプト `scripts/generate-sankey2-data.ts`（仮称）で以下を生成:

| ファイル | 内容 |
|---------|------|
| `public/data/sankey2-graph.json.gz` | ノード + エッジのグラフ構造（座標なし） |
| `public/data/sankey2-layout.json.gz` | 事前計算済みレイアウト（座標 + パス） |

**ノード構造**:

| フィールド | 説明 |
|-----------|------|
| `id` | ユニークID |
| `label` | 表示名 |
| `type` | `total` / `ministry` / `project-budget` / `project-spending` / `recipient` |
| `amount` | 金額（1円単位） |
| `x, y` | 事前計算済み座標 |
| `width, height` | ノードサイズ |

**エッジ構造**:

| フィールド | 説明 |
|-----------|------|
| `source` | 始点ノードID |
| `target` | 終点ノードID |
| `value` | フロー金額 |
| `path` | 事前計算済みBezierパス座標 |

#### 対象件数

`data/year_2024/` の元CSVから、以下を基準として抽出:
- 事業: `2-1` に記載の全事業（現行パイプラインでスキップされていたものも含む）
- 支出先: `5-1` の個別行（「合計」行を除外）
- 支出先ユニーク数: 約26,192件（`data/result/no_total_stats.txt` 基準）

### 表示数スライダー

既存のTopN集約 + ドリルダウンを廃止し、**パーセンテージスライダー**で置き換える。

#### 仕様

| 項目 | 内容 |
|------|------|
| スライダー対象 | 事業（予算額順）、支出先（支出額順） |
| 範囲 | 0%〜100% |
| 基準 | 金額降順にソートし、上位N%を表示 |
| 集約 | 非表示分は「その他の事業」「その他の支出先」ノードに集約（金額合計を表示）。支出先名が「その他」の実データノード（~26兆円）とは別物 |
| 連動 | 事業スライダーの変更 → 関連する支出先も再フィルタ |

#### 動作例

- 事業スライダー 10% → 上位約566事業を表示、残り約5,098事業は「その他の事業」に集約
- 支出先スライダー 5% → 上位約1,310支出先を表示、残りは「その他の支出先」に集約

### Google Maps風 Zoom/Pan UI

#### 描画方式

SVGベースの自前描画。`budget-flow-map` のUXパターン（Zoom閾値によるラベル表示制御、Minimap、ハイライトBFS）を参考にする。

#### レイアウト

| 現行 /sankey | 新 /sankey2 |
|-------------|-------------|
| 5列固定（左→右） | 列の概念を廃止 |
| 列内で縦並び | 2D空間に自由配置 |
| 画面内に収める | 仮想空間を広く使い、Zoom/Panで探索 |

**ノードのタイプ識別**: 列の代わりに、ノードの色・形・ラベルで `type` を視覚的に区別する。

| type | ラベル | 色（案） |
|------|--------|---------|
| `total` | 全体 | グレー |
| `ministry` | 府省庁 | 青系 |
| `project-budget` | 事業(予算) | 緑系 |
| `project-spending` | 事業(支出) | 橙系 |
| `recipient` | 支出先 | 赤系 |

#### Zoom閾値によるラベル表示

`budget-flow-map` の実績を参考に、Zoomレベルに応じてラベルを段階表示:

| Zoomレベル | 表示されるラベル |
|-----------|----------------|
| 最小（全体俯瞰） | 全体、府省庁 |
| 中間 | + 事業(予算)、事業(支出) |
| 最大（詳細） | + 支出先 |

#### インタラクション

| 操作 | 動作 |
|------|------|
| ホイール | Zoom in/out |
| ドラッグ | Pan |
| ノードホバー | 接続されたノード・エッジをハイライト（BFS探索） |
| ノードクリック | 選択状態。サイドパネルに詳細表示 |

### レイヤー設計（CLAUDE.md準拠）

| Layer | ファイル | 役割 |
|-------|---------|------|
| Data Pipeline | `scripts/generate-sankey2-data.ts` | CSV → グラフJSON → レイアウトJSON |
| Domain Logic | `app/lib/sankey2-layout.ts` | レイアウト計算ロジック（スライダーフィルタ適用） |
| API Layer | `app/api/sankey2/route.ts` | 静的JSONの配信（動的計算は行わない） |
| UI Components | `client/components/Sankey2/` | SVG描画、Zoom/Pan、スライダー等 |
| Page | `app/sankey2/page.tsx` | 状態管理・レイアウト |

### 「列を廃止して縦に長くならない配置」について

列レイヤーの概念を廃止するため、レイアウトアルゴリズムの設計が必要。以下の方針:

- ノードのX座標は `type` ごとのベースライン（左→右の大まかな流れ）を維持しつつ、厳密な列ではなく**グループとしてのゆるい配置**にする
- Y座標は府省庁ごとにセクション分割し、関連するノードを近くに配置（`budget-flow-map` のアプローチと同様）
- 巨大ノード（厚生労働省等）と極小ノード（復興庁等）が混在するため、ノード高さの対数スケーリングを検討
- 具体的なレイアウトアルゴリズムは実装フェーズで段階的に詰める
