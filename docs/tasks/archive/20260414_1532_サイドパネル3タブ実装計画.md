# サイドパネル 3タブ実装計画

## 目的

[設計ドキュメント](./20260414_0759_サイドパネル再設計.md) に基づき、「流入元 / 流出先」2タブを **府省庁 / 事業 / 支出先** 3タブに置き換える。  
ユーザーが選択ノードの種類を意識せず、関連する省庁・事業・支出先を直感的に参照できるようにするため。

---

## 事業ノード統合後の変更点

PR #143（事業ノード統合）により、以下の点が設計ドキュメントの想定と変わっている。

| 事項 | 設計ドキュメントの想定 | 実装後の実態 |
|------|----------------------|------------|
| クリックで選択されるノード | `project-spending-{n}` | `project-budget-{n}`（merged shape の click target） |
| `project-spending` 型のパネル表示 | `project-spending` が独立して選択可能 | ユーザーから直接選択される機会はほぼない |
| 「事業セクション」の型 | `project-budget` / `project-spending` 両対応 | `project-budget` を正規型として扱う。`project-spending` も保持（URL・パネルリンク経由で到達しうる） |

**方針**: 事業セクションは `project-budget` 選択を基本とし、`project-spending` 選択でも同じ事業として表示できる共通処理関数でまとめる。

## ノード種別ごとのセクション内容

### `total`

| タブ | 内容 | データソース |
|------|------|-------------|
| 府省庁 | 全省庁（予算額降順） | `graphData.nodes` where `type='ministry'` |
| 事業 | 全事業（省庁グルーピング付き、予算額降順） | `graphData.nodes` where `type='project-budget'` |
| 支出先 | TopN 支出先 | `filtered.nodes` where `type='recipient'`（レイアウト内） |

### `ministry`

| タブ | 内容 |
|------|------|
| 府省庁 | **自身（1件）** |
| 事業 | この省庁の事業（`graphData.nodes` where `ministry === selectedNode.name && type='project-budget'`、支出額降順） |
| 支出先 | この省庁の支出先（事業経由で集計：省庁事業の spending エッジを graphData から辿る） |

### `project-budget` / `project-spending`（統合ノード）

| タブ | 内容 |
|------|------|
| 府省庁 | **所属省庁（1件）**: `selectedNode.ministry` |
| 事業 | **予算執行カード**（後述の再設計案参照） |
| 支出先 | この事業の支出先全件（`graphData.edges` where `source = project-spending-{pid}`、支出額降順） |

`project-budget` か `project-spending` かに関係なく同一 `projectId` の事業として扱う。

### `__agg-project-budget` / `__agg-project-spending`

| タブ | 内容 |
|------|------|
| 府省庁 | `filtered.aggNodeMembers.get('__agg-project-budget')` から省庁を重複除去・合算 |
| 事業 | `filtered.aggNodeMembers.get('__agg-project-budget')` をそのままリスト（省庁グルーピング付き） |
| 支出先 | 集約事業の spending エッジから集計（重複支出先は合算） |

### `recipient`

| タブ | 内容 |
|------|------|
| 府省庁 | この支出先に支出している省庁（事業経由で集計） |
| 事業 | この支出先に支出している事業（`graphData.edges` where `target = recipient.id`、支出額降順） |
| 支出先 | **自身（1件）** |

### `__agg-recipient`

| タブ | 内容 |
|------|------|
| 府省庁 | 集約支出先に支出している省庁（重複合算） |
| 事業 | 集約支出先に支出している事業 |
| 支出先 | `filtered.aggNodeMembers.get('__agg-recipient')` をリスト |

### `__agg-ministry`（集約省庁）

| タブ | 内容 |
|------|------|
| 府省庁 | `filtered.aggNodeMembers.get('__agg-ministry')` をリスト |
| 事業 | 集約省庁配下の事業（省庁グルーピング付き） |
| 支出先 | 集約省庁が支出する支出先 |

## 事業タブ 再設計案

事業ノード統合（PR #143）により、Sankey 図上で予算と支出が視覚的に統合された。  
サイドパネルの事業タブも、**予算額と支出額を並べて表示**する形に統一する。

---

### 統合ノード（`project-budget`）選択時：予算・支出カード

「自身（1件）」のリスト表示から、**単一カード形式**に変更する。

```text
┌──────────────────────────────────────┐
│ 予算額   1,234億円                    │
│ 支出額     987億円                    │
│ 支出先件数   42先                     │
└──────────────────────────────────────┘
```

- 予算額・支出額・支出先件数を縦並びで表示
- `isScaled` フラグが立っている場合は元の予算額を補足表示（現行ヘッダーと同様）
- 支出先件数は `projectRecipientCount` から取得（既存 useMemo 転用）
- カードはリスト形式でなく単一ブロックとして表示

**データソース**:
- 予算額: 同 `projectId` の `project-budget` ノードの `value`
- 支出額: 同 `projectId` の `project-spending` ノードの `value`（`graphData.nodes` から取得）

---

### 省庁（`ministry`）選択時：事業リストに予算額・支出額を両表示

現行：事業名 + 支出額（単一）  
変更後：事業名 + 予算額 + 支出額（2行または右端に並列）

```text
┌──────────────────────────────────────────────┐
│ 社会保険費       予算 1.2兆 / 支出 1.1兆      │
│ 医療保険給付費…  予算 0.8兆 / 支出 0.7兆      │
└──────────────────────────────────────────────┘
```

- 各行: `事業名（省略） | 予算額 / 支出額`
- ソートは支出額降順（現行と同じ）

**データソース**: `ministryProjectStats` は現状 `budgetTotal` / `spendingTotal` を持つが、`projects[]` は支出0円のみ格納（行 438）。  
→ **全事業を格納するよう `ministryProjectStats` の構築ロジックを変更する**:

```text
変更前: if (s === 0) st.projects.push(...)
変更後: st.projects.push(...)  ← 全事業を格納
```

---

### `total` 選択時

省庁グループヘッダーに「N件 予算X円 / 支出Y円」を表示。  
`ministryProjectStats` の `budgetTotal` / `spendingTotal` はすでに計算済みのため、ヘッダー表示の拡充のみ。

---

### `recipient` / その他のノード選択時

事業リストの各行には「支出額」のみ表示（他ノード視点では予算情報は主要ではないため単一金額で十分）。

---

### `PanelEntry` 型拡張

```typescript
type PanelEntry = {
  id: string; name: string; value: number;
  ministry?: string; aggregated?: boolean;
  budgetValue?: number;    // 追加: 予算額（project-budget に対応）
  spendingValue?: number;  // 追加: 支出額（project-spending に対応）
  recipientCount?: number; // 追加: 支出先件数
};
```

`budgetValue` / `spendingValue` が両方揃っている場合のみ両額表示。  
いずれかが欠ける場合は `value` の単一表示にフォールバック。

## タブ自動選択ルール

| 選択ノード | デフォルトタブ |
|-----------|--------------|
| `total` | `ministry` |
| `ministry` | `project` |
| `__agg-ministry` | `ministry` |
| `project-budget` / `project-spending` | `recipient` |
| `__agg-project-budget` / `__agg-project-spending` | `project` |
| `recipient` | `project` |
| `__agg-recipient` | `recipient` |

`selectedNodeId` 変更時に `panelTab` をリセット（`useEffect` で自動選択）。
