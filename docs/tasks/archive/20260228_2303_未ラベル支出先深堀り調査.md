## 解決策A': KAKU_PATTERNS の既存「民間企業（集合）」パターン拡張

**31件・265億円** を追加解消可能。

### 現状の問題

KAKU_PATTERNS の既存パターン:
```python
('民間企業', '民間企業（集合）',
 r'その他民間|^その他事業者$|その他[（(]?[0-9]+社[）)]?|その他[0-9]+社')
```

これは `その他〇〇` プレフィックスのみをカバーしており、**`^民間企業$`（180億円）や `民間企業等`（50億円）がマッチしない**。

### 分類

| 種別 | 件数 | 金額 | 例 |
|------|------|------|-----|
| 集合名称 | 13件 | 231億円 | `民間企業`、`民間企業等`、`民間企業・団体等` |
| 匿名化（特定企業） | 18件 | 34億円 | `民間企業A`、`民間企業B社`、`民間企業C` |

**参考**: 既存の「民間企業（集合）」ラベル付与済み件数は8件・7,203億円（`その他事業者`・`その他97社` 等）。

### 拡張案

```python
('民間企業', '民間企業（集合）',
 r'その他民間|^その他事業者$|その他[（(]?[0-9]+社[）)]?|その他[0-9]+社'
 r'|^民間企業$|^民間企業等|^民間企業[A-Za-z]|補助事業者\(民間企業\)'),
```

- `^民間企業$` → 集合名称（単体）
- `^民間企業等` → 「民間企業等」「民間企業等(1者)」等
- `^民間企業[A-Za-z]` → 匿名化名称（民間企業A〜M等）

## 解決策C: スキップ誤判定の修正（9件・8億円）

`is_skip_target` の複合名称スキップロジックが `一般社団法人○○協議会` の `・` 区切りで誤動作している。

### 問題の構造

`一般社団法人全日本コメ・コメ関連食品輸出促進協議会` を例に:
- `・` で分割 → `['一般社団法人全日本コメ', 'コメ関連食品輸出促進協議会']`
- 各セグメントに格ワードがマッチ（`一般社団法人` + `協議会`）
- → `kaku_segments >= 2` としてスキップ判定 ← **誤判定**

実態はコメ関連食品の`・`（中点）を名称内に含む単一法人。

### 誤判定一覧（9件・8億円）

| 金額 | 名称 |
|------|------|
| 6.15億円 | 一般社団法人全日本コメ・コメ関連食品輸出促進協議会 |
| 0.82億円 | 一般社団法人セキュリティ・キャンプ協議会 |
| 0.33億円 | 一般社団法人マリン・エコラベル・ジャパン協議会 |
| 0.20億円 | 一般社団法人住宅履歴情報蓄積・活用推進協議会 |
| 0.20億円 | 一般社団法人大手町・丸の内・有楽町地区まちづくり協議会 |
| 0.19億円 | 一般社団法人日本難病・疾病団体協議会 |
| 0.14億円 | 一般社団法人マリン・エコラベル・ジャパン協議会 |
| 0.05億円 | 一般社団法人九州半導体・デジタルイノベーション協議会 |
| 0.03億円 | 一般社団法人全国定期巡回・随時対応型訪問介護看護協議会 |

### 修正方針

`is_skip_target` に例外条件を追加:
- 名称が `一般社団法人`、`公益社団法人`、`一般財団法人`、`公益財団法人` で始まる場合は複合名称として扱わない

```python

    # 法人格名で始まる場合は単一法人として扱う（例: 一般社団法人○○・△△協議会）
    if re.match(r'^(?:一般|公益)(?:社団|財団)法人', name):
        return False
    if not SEPARATORS.search(name):
        return False
```
