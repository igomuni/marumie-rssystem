# 行政機関辞書化設計

## 目的

データ管理者が支出先の行政機関を正確に識別・追跡できるようにするため:
- 現在の補助辞書（government_branch 431件）を「所轄リンク付き行政機関辞書」に再構築する
- 教育委員会・農業委員会等の新カテゴリを加え、スコア評価の精度を高める
- 府省庁・都道府県・市区町村との所轄関係を辞書に記録し、分析に活用できるようにする

---

## 現状把握

### 補助辞書 government_branch (431件)の実態

| 属性 | 件数 | 内容 |
|------|------|------|
| cn_in_db=True | 342 | 省庁CN紐付き地方機関（厚生労働省86, 国土交通省61, 農林水産省26...） |
| cn_in_db=False | 89 | CN未照合（法務局29, 市区町村24, 公安調査局8, 出入国管理局8...） |

現在は `supplementary_valid_names.csv` に `name, category, note` の3列で管理。
所轄省庁・都道府県・市区町村との紐付き情報を持たない。

### 新スコープ: valid=False 行政機関（recipient_dictionary.csv）

`cn_in_db=True AND DB名=自治体名 AND 名称が行政機関サフィックスで終わる` パターン:

| 機関種別 | 件数 | 例 |
|---------|------|-----|
| 教育委員会 | 49 | 三重県教育委員会（CN:三重県）、亀山市教育委員会（CN:亀山市） |
| 農業委員会 | 37 | 三田市農業委員会（CN:三田市）、上越市農業委員会（CN:上越市） |
| 消防局 | 1 | 和歌山市消防局（CN:和歌山市） |

これらはCNが自治体CNに紐づく行政機関委員会。プログラム的に抽出可能。

## 設計方針

### 新ファイル: `public/data/dictionaries/government_agency_names.csv`

現在の `supplementary_valid_names.csv` の government_branch を廃止し、リッチなメタデータを持つ専用辞書に移行する。

#### 列設計

| 列名 | 型 | 説明 |
|------|----|------|
| `name` | str | 支出先名（辞書キー） |
| `agency_type` | str | 機関種別（下表参照） |
| `parent_name` | str | 上位組織名（法人番号DBの法人名） |
| `parent_cn` | str | 上位組織の法人番号（13桁） |
| `ministry` | str | 所轄府省庁（国の行政機関の場合は直接） |
| `prefecture` | str | 所属都道府県（都道府県・市区町村機関のみ） |
| `municipality` | str | 所属市区町村（市区町村機関のみ） |
| `note` | str | 備考・分類根拠 |

#### agency_type 値

| 値 | 意味 | 例 |
|----|------|----|
| `field_office` | 国の地方機関 | 関東地方整備局, 大阪労働局 |
| `prefectural_branch` | 都道府県の地方機関 | 都道府県警察, 教育委員会(都道府県) |
| `municipal_committee` | 市区町村の行政委員会 | 教育委員会(市区町村), 農業委員会 |
| `special_org` | 特殊法人・独立行政法人 | （補助辞書から移行の1件） |

#### サンプル行


## 生成スクリプト設計

新規スクリプト: `scripts/generate-government-agency-names.py`

### 生成対象の優先順位

1. **補助辞書 government_branch の移行（431件）**:
   - cn_in_db=True (342件): parent_name・parent_cn を recipient_dictionary の `db_name_by_cn` から引く
   - cn_in_db=False (89件): parent_name・parent_cn を法務局→法務省 等の既知マッピングで補完

2. **新規: 教育委員会（約54件）**:
   - recipient_dictionary で `valid=False AND cn_in_db=True AND name.endswith('教育委員会') AND db_name.endswith(都道府県市区町村サフィックス)` を抽出
   - agency_type: 都道府県機関→ `prefectural_branch`, 市区町村機関→ `municipal_committee`
   - ministry: `文部科学省`（固定）
   - prefecture・municipality: db_name から推定（都道府県が判明すれば自動設定）

3. **新規: 農業委員会（約47件）**:
   - 同上パターン。ministry: `農林水産省`（固定）
   - agency_type: `municipal_committee`（農業委員会は市区町村に設置）

4. **新規: 消防局・消防署（約10件）**:
   - ministry: `総務省消防庁`
   - agency_type: `municipal_committee`

### 生成不要・手動管理

- 法務局29件（CNなし → 法務省に紐付け手動設定が確実）
- 特殊法人1件

## スコアリング変更設計 (score-project-quality.py)

### 辞書ロード追加

```
gov_agency_map = {}  # name → {agency_type, ministry, prefecture, municipality}
with open(GOV_AGENCY_CSV, ...) as f:
    for r in csv.DictReader(f):
        gov_agency_map[r['name']] = r
```

### 軸1カウント変更（3層構造へ）

```
現在:
  strict_valid → valid_count++
  strict_invalid + in supp_map → supp_valid_count++
  その他 → invalid_count++

新:
  strict_valid → valid_count++
  strict_invalid + in gov_agency_map → gov_agency_count++   ← 新追加
  strict_invalid + not in gov_agency_map + in supp_map → supp_valid_count++
  その他 → invalid_count++
```

### 出力への影響

- JSON/CSV に `gov_agency_count` を追加
- axis1 = (valid + gov_agency + supp_valid) / total × 100（計算式は変わらず、gov_agencyが加わる）

## スコアへの期待影響

教育委員会54件・農業委員会47件が新たに有効カウントに加わる事業のaxis1が改善する。
補助辞書の government_branch(431件)は現在もsupp_valid_countとして有効カウントされており、
移行後は gov_agency_count として計上されるため、**全体スコアは変化なし**。

新規の教育委員会・農業委員会分は、各事業での `invalid_count` が減少することでaxis1向上。
これらが集中する文教・農林水産系事業でスコア改善が見込まれる。
