# 5-1/5-2 CSV 直接支出パターン調査

## 目的

開発者が `scripts/generate-sankey3-data.ts` の実装に先立ち、CSVにおける直接支出（国→支出先）と間接支出（再委託チェーン）のデータパターンを正確に把握するため。

## 調査対象

- `data/year_2024/5-1_RS_2024_支出先_支出情報.csv`（194,133行）— 支出先の詳細情報
- `data/year_2024/5-2_RS_2024_支出先_支出ブロックのつながり.csv`（22,662行）— ブロック間の資金フロー

## 直接支出の判定方法

5-2 CSVの `担当組織からの支出` カラムが `TRUE` である `(予算事業ID, 支出先の支出先ブロック)` ペアを「直接支出ブロック」と定義する。

- 5-2 CSVはブロック間の資金の流れを記述しており、`担当組織からの支出=TRUE` は国（担当組織）がそのブロックに直接支出していることを意味する
- ブロック番号（A, B, C, ...）自体には直接/間接の意味はない。どのブロックが直接支出かは事業ごとに異なる
- 5-1 CSVの各行を `(予算事業ID, 支出先ブロック番号)` で5-2の直接支出ペアとマッチさせて分類

### 4. ブロック番号と直接/間接の関係

ブロック番号は直接/間接を示すものではない。各ブロックの直接支出額と間接支出額:

| ブロック | 直接支出額 | 間接支出額 | 直接比率 |
|----------|-----------|-----------|---------|
| A | 112.89兆 | 0.08兆 | 100% |
| B | 7.12兆 | 13.07兆 | 35% |
| C | 6.80兆 | 6.05兆 | 53% |
| D | 0.65兆 | 2.13兆 | 23% |
| E | 0.32兆 | 0.73兆 | 31% |
| F〜Z | 1.94兆 | 3.17兆 | 38% |

Block A は結果的にほぼ100%直接だが、Block B にも 7.12兆円、Block C にも 6.80兆円の直接支出がある。国が同一事業内で複数の支出ルート（ブロック）を通じて直接支出しているケース。

### 5. パイプライン設計への示唆

新パイプライン（`generate-sankey3-data.ts`）で直接支出先を正確に判定するには、**5-2 CSVの `担当組織からの支出=TRUE` フラグの参照が必須**。

処理フロー:
1. 5-2 CSVを読み込み、`担当組織からの支出=TRUE` の `(予算事業ID, ブロック番号)` ペアを直接支出ブロックセットとして構築
2. 5-1 CSVを読み込み、各行の `(予算事業ID, 支出先ブロック番号)` を直接支出ブロックセットとマッチ
3. マッチした行 → `recipient` ノード（事業からの直接支出先）
4. マッチしない行 → 再委託チェーン（将来の `subcontract` ノード候補）

再委託チェーンの親子関係は、5-2 CSVの `支出元の支出先ブロック → 支出先の支出先ブロック` で構築可能。
