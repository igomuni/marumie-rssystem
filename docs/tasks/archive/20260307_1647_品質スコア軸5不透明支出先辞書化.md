# 品質スコア 軸5 の不透明支出先辞書化

## 目的

事業別支出先データ品質スコアの軸5（透明性）が、`その他支出先=TRUE` フラグを誤って「支出先名が不明」と解釈していたため、正しい評価基準に修正する。

## 背景・問題発見の経緯

### 発端: PID 140「戦略的イノベーション創造プログラム」の異常スコア

PID 140 の総合スコアは 59.6 点。データ自体はきちんと入力されているにもかかわらずスコアが冴えない原因を調査した。

| 軸 | スコア | 重み | 寄与 | 備考 |
|----|--------|------|------|------|
| 軸1 名称 | 78.4 | 40% | 31.4 | 422行中91行がinvalid |
| 軸2 CN | 0.9 | 20% | 0.2 | 422行中4行のみ法人番号記入 |
| 軸3 収支 | 100.0 | 20% | 20.0 | 完璧 |
| 軸4 構造 | 80.0 | 10% | 8.0 | 再委託階層2 |
| **軸5 透明性** | **0.0** | **10%** | **0.0** | **96.7% が TRUE** |
| **合計** | | | **59.6** | |

### `その他支出先=TRUE` の実態

当初、`その他支出先=TRUE` を「支出先名が『その他』（不明/非公開）」と解釈してスコアリングに使用していた。

しかし PID 140 の実データを確認したところ:

- `その他支出先=TRUE` (408行): KDDI株式会社、パナソニックHD、PwCコンサルティング等の **実在企業名**
- `その他支出先=FALSE` (14行): 国立研究開発法人量子科学技術研究開発機構、警察庁等

`その他支出先=TRUE` は「ブロック内の主要支出先以外の支出先」というRS Systemのデータ構造上の分類であり、支出先名の品質とは無関係だった。

### 本来の「不透明な支出先名」とは

データ全体を調査し、支出先名自体が不透明であるパターンを特定した:

| パターン | 延べ行数 | 例 |
|---------|---------|-----|
| 完全一致「その他」 | 6,968 | その他 |
| 「その他」で始まる集約表記 | 602 | その他労働局、その他自治体 |
| 匿名化表記 | 504 | A社、B社、個人 |
| 未確定表記 | 69 | 未定、未定(入札等により決定) |
| プレースホルダー | 6 | ○○府省庁 |
| **合計** | **8,149 / 173,359行 (4.7%)** | |

## 対応内容

### 1. 不透明支出先名キーワード辞書の作成

`public/data/dictionaries/opaque_recipient_keywords.csv` を新規作成。

| match_type | 説明 | ルール数 |
|-----------|------|---------|
| exact | 完全一致 | 5 |
| regex | 正規表現（A社パターン、○○等） | 4 |
| prefix | 前方一致（「その他」で始まる集約表記） | 1 |
| contains | 部分一致（「未定」「非公開」を含む） | 2 |
| **合計** | | **13ルール** |

不透明レベル（参考情報として記録、スコアリングには未使用）:

- Level 1: 完全不明（その他、未定、非公開）
- Level 2: 匿名化（A社、個人）、集約表記
- Level 3: プレースホルダー（○○）

### 2. 軸5 スコアリングの修正

**修正前**: `その他支出先=TRUE` フラグの割合で評価
**修正後**: 支出先名が不透明キーワード辞書にマッチする行の割合で評価

スコア計算式は変更なし: `ratio=0 → 100点, ratio>=0.5 → 0点（線形）`

### 3. フィールド名・ラベルの変更

| 箇所 | 修正前 | 修正後 |
|------|--------|--------|
| JSON | `unknownNameRatio` | `opaqueRatio` |
| CSV | `支出先名不明率` | `不透明支出先率` |
| UI 展開行 | 支出先名不明率（支出先名「その他」行の割合） | 不透明支出先率（不透明キーワード辞書にマッチする行の割合） |
| トップページ | 支出先名不明率 | 支出先名の透明性 |

## 修正後の影響

### PID 140 のスコア変化

| | 修正前 | 修正後 |
|--|--------|--------|
| axis5 | 0.0 | 87.7 |
| opaqueRatio | 96.7%（フラグベース） | 6.2%（辞書ベース） |
| totalScore | 59.6 | 68.3 |

### 全体への影響

| | 修正前 | 修正後 |
|--|--------|--------|
| 平均スコア | 80.3 | 80.5 |
| 中央値 | 90.9 | 90.9 |
| 軸5 平均 | - | 94.6 |
| 軸5 = 100 | - | 3,864事業 |
| 軸5 < 50 | - | 77事業 |

全体への影響は限定的（平均+0.2）。`その他支出先=TRUE` が多い事業（SIP等の大型研究事業）で大きな改善。

## 同セッションで実施した関連修正

### 孤立ブロック検出

5-1 に存在するが 5-2 に未記載のブロック（「孤立ブロック」）を検出・記録する仕組みを追加。

- 10事業・19ブロックが孤立ブロックとして検出
- ルートブロックとして補完はしない（リンクが辿れないため根拠がない）
- UI展開行の「ブロック数」に赤字で「(孤立: N)」を表示
- `orphanBlockCount` として JSON に記録

代表例: PID 6562「企業保有情報提出に関する導入支援システム」— Bブロック（NTTデータ経営研究所）が5-2に未記載

### UI改善

- ヒストグラムを10点刻みクリッカブルバーに統合（100点を独立ビンとして分離）
- フィルター（検索・府省庁）をヒストグラム横に配置
- 検索入力がウィンドウ幅に応じて伸縮
- 全角（）→ 半角()の正規化
