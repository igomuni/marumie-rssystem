# /sankey2 グリッドレイアウト設計（A案）

## 課題

現行レイアウトは37府省庁を縦に積み上げるため、仮想空間が 1,650 × 79,662 px になる。
ノード数が37,258件あり、1pxノードで単純積み上げでも26,000px超。線形・対数スケールの問題ではなく**ノード件数の問題**。

## セル内レイアウト

各セルは独立したミニSankeyフロー。4つのtype帯を横に並べる。

```
セル内構成（1セル = 1府省庁）:

  ←──── セル幅 ────→

  [府省庁] → [事業(予算)] → [事業(支出)] → [支出先]
     │         │  │  │         │  │  │        │  │  │
     │         │  │  │         │  │  │        │  │  │
     │         │  │  │         │  │  │        │  │  │
```

### セル内X座標

totalノードはグリッド外に配置（左端）。各セル内は相対座標:

| type | セル内X |
|------|--------|
| ministry | 0 |
| project-budget | セル幅 × 0.25 |
| project-spending | セル幅 × 0.50 |
| recipient | セル幅 × 0.75 |

### セル内Y座標

各type帯内で金額降順に積み上げ。ノード高さは**線形スケール**、ノード間余白は**0px**:

```
height = max(MIN_HEIGHT, amount * AMOUNT_SCALE)
  MIN_HEIGHT = 1px
  AMOUNT_SCALE = 1e-11（1兆円 = 10px）
GAP = 0px
```

#### スケール・GAP選定の根拠

セル高さの支配要因は**ノード間余白（GAP）**であり、ノード高さ自体ではない。
経産省の支出先3,138件の場合:

| GAP | GAP分の高さ | ノード高さ分 | セル高さ合計 |
|-----|-----------|------------|------------|
| 0px | 0px | ~3,200px | ~3,200px |
| 1px | 3,138px | ~3,200px | ~6,400px |
| 2px | 6,276px | ~3,200px | ~9,500px |

GAP=2pxではGAP分がノード高さの2倍を占める。Zoom/PanのUIではズームインでノードを見分けられるため、GAP=0pxが最適。

対数スケールは小額ノード（大半が1px）を拡大するため、ノード数が多い場合は線形より**逆に高さが増大**する。線形スケール + GAP=0が最もコンパクト。

## グリッド配置アルゴリズム

### 1. セルサイズの決定

セルの高さはセル内の最大ノード積み上げ高さで決まる。

**均一セルサイズ方式**: 全セルを最大セル（厚生労働省）に合わせる。
- メリット: グリッドが綺麗に揃う
- デメリット: 小規模庁のセルがスカスカ

**可変セルサイズ方式**: セルごとにノード数に応じた高さにする。
- メリット: スペース効率が良い
- デメリット: グリッドが不揃い

**推奨: 可変セルサイズ（行ごとに最大セル高さで揃える）**

各行の高さ = その行の6セルのうち最大のセル高さ。行内は揃うが行間は異なる。

### 2. セル内ノード配置

```
for each ministry in gridOrder:
  cell = getGridCell(ministry)

  # type帯ごとに金額降順で積み上げ
  for type in [ministry, project-budget, project-spending, recipient]:
    nodes = getNodesForMinistryAndType(ministry, type)
    nodes.sort(by amount descending)

    y = cell.top
    for node in nodes:
      node.x = cell.left + TYPE_X_OFFSET[type]
      node.y = y
      node.height = max(MIN_HEIGHT, node.amount * AMOUNT_SCALE)
      y += node.height  # GAP=0
```

### 3. 支出先の配置

支出先は複数府省庁から受け取る場合がある。**最大フロー元の府省庁セル**に配置する。

| 配置方式 | 説明 |
|---------|------|
| 最大フロー方式（推奨） | 支出先を最も多く支出している府省庁のセルに配置。クロスセルエッジは斜め線で表現 |
| 重複配置方式 | 支出先を関連する全府省庁のセルに複製。エッジは綺麗だがノード重複 |

### 4. totalノードの配置

totalノードはグリッドの左端に配置。37本のエッジが各府省庁セルに接続。

```
[total] ─┬→ [厚生労働省セル]
         ├→ [経済産業省セル]
         ├→ [国土交通省セル]
         └→ ...
```

### 5. クロスセルエッジ

支出先が別府省庁の事業からもフローを受ける場合、セルをまたぐエッジが発生する。

- **視覚的処理**: クロスセルエッジは薄い色・細い線で表示し、セル内エッジと区別
- **Zoom閾値**: クロスセルエッジはズームイン時のみ表示

## 想定される仮想空間サイズ

### 前提

- グリッド: 6列 × 7行（最終行は1セルのみ）
- セル幅: 400px（4type帯 × 100px）
- セル間余白: 50px
- 線形スケール + GAP=0px

### 試算

| 項目 | 値 |
|------|-----|
| グリッド幅 | 6 × 400 + 5 × 50 = **2,650px** |
| 行1（厚労・経産・国交・文科・内閣府・防衛） | 3,837px |
| 行2（こども・農水・総務・環境・外務・復興） | 2,804px |
| 行3〜7 | 1,620px（合計） |
| **グリッド高さ** | **8,261px + 行間余白300px = 8,561px** |
| **仮想空間** | **2,650 × 8,561 px** |

現行の 1,650 × 79,662 から **高さ1/9、面積比約1/6** に圧縮。

## Zoom/Pan との相性

| ズームレベル | 表示内容 |
|------------|---------|
| 最小（全体俯瞰） | 37セルのグリッド。各セルは府省庁名ラベルのみ |
| 中間 | 1行（6セル）程度。事業ノードのラベルが見える |
| 最大（詳細） | 1セル内部。全ノードのラベル・エッジが見える |

グリッド配置により、ズームアウト時に**全府省庁の比較**が可能になる。
これは縦積みレイアウトでは不可能だった（画面に3-4府省庁しか入らない）。
