## 1. 対象データの概要

**元CSV**: `data/year_2024/5-1_RS_2024_支出先_支出情報.csv`

| 項目 | 値 |
|------|----|
| 総レコード数 | 194,133行 |
| ブロック行（支出先名なし・ブロック名あり） | 19,783行 |
| スキップ行（支出先名・ブロック名ともに空） | 991行 |
| 個別支出先行（支出先の合計支出額 **あり**） | ― |
| 個別支出先行（支出先の合計支出額 **なし**） | 約98,159行 |

### 「金額あり かつ 支出先の合計支出額なし」の抽出結果

> ユーザー要件：`金額`フィールドが存在し、かつ`支出先の合計支出額`が空欄のレコードを対象とする

| 指標 | 値 |
|------|----|
| 該当レコード数 | **98,159行** |
| **ユニーク支出先名** | **26,148件** |
| 総金額合計 | **155.03兆円** |

### ※ `recipients_without_total.csv` との差異について

`analyze-label-coverage.py` の入力ファイル `data/result/recipients_without_total.csv` は、
`金額`の有無にかかわらず「支出先の合計支出額が空 かつ 支出先名が空でない」行をすべて含む。
そのため同ファイルのユニーク支出先名は **26,192件** となっており、
上記 26,148件との差分 **44件** は「支出先の合計支出額なし・金額なし」の行に
のみ現れる支出先名に相当する。

## 2. パイプライン処理フロー

```
[元CSV]
data/year_2024/5-1_RS_2024_支出先_支出情報.csv
            │
            ▼
  ┌─────────────────────────────────────────────┐

  │ 実行: npx tsx scripts/split-spending-csv.ts  │
  └─────────────────────────────────────────────┘
            │
            ├─▶ data/result/blocks.csv                  （ブロック集計行）
            ├─▶ data/result/recipients_with_total.csv   （合計支出額あり）
            ├─▶ data/result/recipients_without_total.csv（合計支出額なし ★）
            └─▶ data/result/skipped.txt                 （スキップ行）

                        ★ ここがラベリング対象
                            │
                            ▼
  ┌─────────────────────────────────────────────────────────────────┐
  │ Step 1 & 2: ラベルカバレッジ分析                                │
  │ スクリプト: scripts/analyze-label-coverage.py                   │
  │ 実行: python3 scripts/analyze-label-coverage.py                 │
  └─────────────────────────────────────────────────────────────────┘
            │
            ├─▶ data/result/label_coverage_report.txt   （カバレッジレポート）
            └─▶ data/result/uncovered_recipients.csv    （未カバー支出先一覧）
```

## 3. analyze-label-coverage.py の処理詳細

### 入力

| ファイル | 役割 |
|----------|------|
| `data/result/recipients_without_total.csv` | ラベリング対象の支出先行（26,192件） |
| `public/data/dictionaries/*.csv` | 完全一致用辞書（14ファイル） |

### Step 1: 完全一致辞書マッチング

支出先名を以下の辞書に対して**完全一致 または 正規表現フルマッチ**で照合する。

| 辞書ファイル | エントリ数 | 説明 | マッチ方式 |
|-------------|-----------|------|----------|
| `aggregate_names.csv` | 37 | 「その他」系集合名称 | exact + regex |
| `beneficiary_names.csv` | 35 | 受益者・補助金受給者など | exact |
| `country_names.csv` | 398 | 国名（日本語） | exact |
| `embassy_names.csv` | 71 | 在外公館名 | exact |
| `foreign_agency_names.csv` | 19 | 外国政府機関名 | exact |
| `international_org_names.csv` | 224 | 国際機関名 | exact |
| `ministry_from_ichiran.csv` | 1,702 | 府省庁（一覧PDF由来） | ministry/bureau/section列 |
| `ministry_names.csv` | 1,795 | 府省庁（基本名称） | ministry/bureau/division等7列 |
| `ministry_supplement.csv` | 242 | 府省庁補完辞書 | ministry/bureau/section等 |
| `municipality_names.csv` | 2,931 | 市区町村名 | exact |
| `municipality_supplement.csv` | 30 | 市区町村補完 | exact |
| `police_names.csv` | 48 | 警察関係機関名 | exact |
| `prefecture_names.csv` | 93 | 都道府県名（長・議会等） | exact |
| `special_corporation_names.csv` | 35 | 特殊法人名 | exact |

**Step 1 カバレッジ実績**（2026-02-28時点）:
- 件数: 1,825件 / 26,192件 = **7.0%**
- 金額: **63.5%**（大口の府省庁・受益者名が集中）

### Step 2: 格パターンマッチング（正規表現）

辞書に含まれない支出先名を、法人格を示す文字列パターンで分類する。
スクリプト内 `KAKU_PATTERNS` リスト（L1大分類 / L2中分類 / 正規表現）で定義。

**主な格パターン（抜粋）**:

| L1大分類 | L2中分類 | パターン例 |
|---------|---------|-----------|
| 民間企業 | 株式会社 | `株式会社｜（株）` |
| 民間企業 | 有限会社 | `有限会社｜（有）` |
| 公益法人・NPO | 一般社団法人 | `一般社団法人｜（一社）` |
| 公益法人・NPO | 特定非営利活動法人 | `特定非営利活動法人｜NPO法人` |
| 独立行政法人等 | 国立研究開発法人 | `国立研究開発法人` |
| 協同組合等 | 農業協同組合 | `農業協同組合` |
| 地方公共団体 | 市区町村 | `^[^\s]{2,5}(?:都|道|府|県)[^\s]{2,10}(?:市|区|町|村)$` |
| コンソーシアム・共同体 | 共同企業体 | `共同企業体｜協働企業体` |
| 協議会 | 協議会 | `協議会` |
| その他（集合） | プレースホルダー | `^その他$｜^その他の支出先$` 等 |

**スキップ対象**: 区切り文字（`・` or `、`）を含み、かつ2セグメント以上に格ワードがマッチする合算名称（例: `○○株式会社・△△株式会社`）はスキップ。ただしコンソーシアム・共同体系はバイパス適用。

**Step 2 カバレッジ実績**:
- 件数: 19,164件 / 26,192件 = **73.2%**
- 金額: **35.1%**

### Step 1 ∪ Step 2 合算

| 指標 | 値 |
|------|----|
| 件数カバレッジ | 20,967件 / 26,192件 = **80.1%** |
| 金額カバレッジ | **98.4%** |
| 両ステップ重複 | 22件 |
