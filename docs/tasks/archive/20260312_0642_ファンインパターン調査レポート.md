## 現状の問題と影響範囲

### `score-project-quality.py` の chain 計算バグ

```python
# 現在の実装（line 175-176）— 最初の親のみ保持
if dst not in parent_map:
    parent_map[dst] = src  # 2番目以降の親は無視
```

| 問題 | 影響 |
|---|---|
| `parent_map[K]` が最初の親（A）のみ保持 | chain 表示が "組織→A→K" のみ（B〜J→K が非表示） |
| Sankey 描画への影響 | `outflows` は全ルート保持済みなので **Sankey 図は正しい** |
| 金額整合性への影響 | 一致型は金額OK、不一致型は構造上の仕様差異 |

### 同様の問題が `generate-structured-json.ts` にも存在

```typescript
// line 476-477 — inflows も最後の親で上書き
inflows.set(targetKey, row.支出元の支出先ブロック名);
inflowsNumber.set(targetKey, sourceBlock);
```

こちらは `Map.set()` で **最後の親に上書き**される（Python 側とは逆順）。

### 対応方針（案）

1. **chain 表示改善**: ファンイン検出時に複数パスを列挙（例：`組織→{A,B,...,J}→K`）
2. **品質スコアへの影響軽減**: ファンインブロックを検出フラグで除外または別評価
3. **現状維持**: 件数が少ない（59事業）ため、既知の例外として文書化のみ
