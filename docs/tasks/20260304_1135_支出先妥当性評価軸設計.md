# 支出先 妥当性評価軸 設計

作成日: 2026-03-04

---

## 目的

データ分析者が各支出先レコードを見たとき、「実在する単一の外部エンティティへの明確な支出か否か」を一目で判別できるようにする。

---

## 評価軸の概要

| 項目 | 内容 |
|------|------|
| 軸名 | `validity`（妥当性） |
| 評価値 | `ok`（問題なし） / `ng`（問題あり） |
| 評価対象 | 支出先名（spendingName）単位 |
| 実装 | 新規スクリプト（既存スクリプトの流用なし） |

---

## 「問題なし（ok）」の定義

### 原則

> 支出先名が **単一の実在エンティティ** を明確に指しており、かつ以下の 3 つの判定基準のいずれかに該当するもの。

「単一」「明確」の 2 要件を**両方**満たす必要がある。

---

### 前提条件：単一性チェック（集合・混在の除外）

以下のいずれかに該当する名称は、判定基準を満たしていても `ng` とする。

| パターン | 例 |
|---------|-----|
| 「ほか」「など」「等」を含む | ○○県ほか、医療機関等 |
| 「・」「、」で複数名を並列している | ○○株式会社・△△株式会社 |
| 「その他」で始まる | その他37県、その他民間 |
| 数字＋法人種別（集合数表現） | その他97社、他12件 |
| 「(N市町村)」「(N機関)」等の括弧付き集合数 | 地方公共団体(43市町村等) |
| 末尾が「集合」「一覧」「グループ」 | 医療機関集合 |

---

### 判定基準 1：都道府県・国名辞書への登録

以下の辞書ファイルに完全一致するもの（alias 含む）を `ok` とする。

| 辞書ファイル | 対象 |
|------------|------|
| `country_names.csv` | 国名（canonical_name / name 列） |
| `prefecture_names.csv` | 都道府県名（canonical_name / name 列） |
| `municipality_names.csv` | 市区町村名（canonical_name / name 列） |
| `municipality_supplement.csv` | 市区町村補完（canonical_name / name 列） |

> 府省庁・特殊法人辞書は含めない（格による判定基準 2/3 で対応）

---

### 判定基準 2：承認格（ok_kaku）の含有

名称中に **承認格辞書（`ok_kaku.csv`）** に登録された格ワードが含まれるもの。

#### ok_kaku.csv の設計

新規作成するファイル: `public/data/dictionaries/ok_kaku.csv`

| カラム名 | 説明 |
|---------|------|
| `kaku` | 格ワード（名称中に含まれることで判定） |
| `l1` | 対応する L1 大分類（参考） |

#### 承認格から除外するもの（現時点）

格として機能するが名称内で曖昧性が生じうるため一旦対象外とする：

| 格ワード候補 | 除外理由 |
|-----------|---------|
| 協会 | 法人格なしの任意団体も多数 |
| 組合 | 法的根拠が多様（任意組合含む） |
| 事務所 | 個人・法人混在 |
| 研究所 | 法人内部組織の場合あり |
| センター | 法人内部組織・任意団体混在 |
| 学会 | 法人格なしの学術団体も多数 |
| 振興会 | 任意団体多数 |
| 連盟 / 連合会 | 任意団体多数 |
| 協議会 | 法人格なし |

### 判定基準 3：（将来拡張）

外国法人・国際機関の単一エンティティ判定は別タスクで設計。

---

## 判定フロー

```
spendingName
    │
    ├─ 集合・混在パターンに該当？ → YES → ng
    │
    ├─ country_names / prefecture_names / municipality_names に完全一致？ → YES → ok
    │
    ├─ ok_kaku.csv のいずれかの格ワードを含む？ → YES → ok
    │
    └─ それ以外 → ng
```

> **注**: ラベル未付与（source=none）の支出先も同フローで判定する（新スクリプトは entity-labels に依存しない）。

## 新規スクリプト設計

### ファイル名

`scripts/assess-spending-validity.py`

### 入力

| ファイル | 用途 |
|---------|------|
| `public/data/rs2024-structured.json` | 評価対象の支出先名一覧を取得 |
| `public/data/dictionaries/country_names.csv` | 判定基準 1 |
| `public/data/dictionaries/prefecture_names.csv` | 判定基準 1 |
| `public/data/dictionaries/municipality_names.csv` | 判定基準 1 |
| `public/data/dictionaries/municipality_supplement.csv` | 判定基準 1 |
| `public/data/dictionaries/ok_kaku.csv` | 判定基準 2（新規作成） |

### 出力

| ファイル | 内容 |
|---------|------|
| `public/data/spending-validity.json` | `{ spendingName: "ok" \| "ng" }` のマップ |

> `entity-labels.json` には手を加えず、別ファイルとして出力する。

### 出力フォーマット（案）

```json
{
  "株式会社○○": "ok",
  "都道府県集合": "ng",
  "その他": "ng",
  "北海道": "ok"
}
```

## 非対象事項

- 協議会・コンソーシアム・実行委員会等の ok/ng 判定（別タスク）
- 外国法人・国際機関の単一エンティティ判定（別タスク）
- UI 側の validity フィルター表示
- 金額ベースの妥当性評価（スコアリング等）
