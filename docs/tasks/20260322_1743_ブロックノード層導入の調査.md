# ブロックノード層導入の調査

## 背景

現在の構造: `事業(支出)` → `支出先` の2層。
サブコントラクトエッジはブロック名が支出先ノードに一致する場合のみ生成（カテゴリ名ブロックはスキップ）。

提案: `事業(支出)` → `支出ブロック` → `支出先` の3層にすることで、カテゴリ名ブロック経由のエッジも表現可能になるか？

## 3層構造の概要

```
事業(支出) ──→ ブロック ──→ 支出先
             │          │
             │ block→block（再委託）
             ↓
           ブロック ──→ 支出先
```

- **事業→ブロック**: 5-2 CSVの直接支出行（担当組織からの支出=TRUE）から生成。13,088エッジ
- **ブロック→ブロック**: 5-2 CSVの非直接支出行から生成。6,909エッジ
- **ブロック→支出先**: 5-1 CSVのブロック番号→支出先名から生成。74,946エッジ

### 解決される問題

1. **カテゴリ名ブロック（都府県、市町村等）がエッジのソースになれる** — ブロック自体がノードなので支出先名との一致不要
2. **間接支出先（13,391ノード）のレイアウト帰属** — 事業→ブロック→支出先の経路で府省庁帰属を計算可能
3. **Fan-In問題の正確な表現** — B(都府県)→D(市町村営事業)、F(北海道)→D(市町村営事業) が別々のエッジとして存在

### 追加コスト

| 項目 | 現在 | 導入後 |
|------|------|--------|
| ノード数 | ~37,000 | ~47,000（+9,831） |
| エッジ数 | ~62,000 | ~95,000（+13,088+6,909+74,946−現行subcontract 16,064） |
| 新ノードタイプ | - | `spending-block`（レイアウト・UI対応必要） |

### 考慮事項

- ブロック→支出先のエッジ74,946件が大きい（現在の全エッジ62,347件を超える）
- レイアウト計算で「事業(支出)」と「支出先」の間にブロック層をどう配置するか
- UIでのブロックノードの表示方法（色、サイズ、ラベル）
- ブロックノードのクリック時の挙動

## 仮説検証: 問題はソースブロックが複数支出先のときのみ発生するか

### ソースブロックの支出先数別の接続分類

| ソースブロックの支出先数 | 接続数 | 割合 | 問題 |
|------------------------|--------|------|------|
| 0件（支出先なし） | 263 | 3.8% | エッジ生成不可 |
| **1件** | **4,758** | **68.9%** | **なし — 唯一の支出先がソースとして一意に決まる** |
| **2件以上** | **1,888** | **27.3%** | **あり — どの支出先がソースか特定不能** |

### 支出先1件の場合の詳細

| 区分 | 件数 |
|------|------|
| 固有名（ブロック名=支出先名） | 4,225件 |
| カテゴリ名（ブロック名≠支出先名） | 533件 |

カテゴリ名533件はすべてブロック名と支出先名が異なるが、支出先が1件しかないため一意に決まる。

不一致の主な原因は**表記揺れ**:
- `日本電気(株)` → `日本電気株式会社`
- `(株)三菱総合研究所` → `株式会社三菱総合研究所`
- `スカパーJSAT(株)` → `スカパーJSAT株式会社`
- `指定金融機関` → `株式会社横浜銀行`

### 支出先2件以上の場合の詳細

| 区分 | 件数 | 問題の種類 |
|------|------|-----------|
| カテゴリ名 | 1,368件 (19.8%) | **捏造**: ブロック名が支出先に一致しないため代表選択が必要だった |
| 固有名 | 520件 (7.5%) | **情報欠落**: ブロック名の支出先は一致するが、同ブロックの他の支出先は委託元として表現されない |

### 結論

**仮説は正しい。** ソースブロックが支出先1件の場合（68.9%）は問題なく正確なエッジ生成が可能。

問題が発生するのはソースブロックが2件以上の場合（27.3%）のみ:
- ブロックノード層があれば、ソースブロック自体がノードとして存在するため、支出先数に関係なくエッジが生成できる
- ブロックノード層がなくても、ソース1件の場合は唯一の支出先を使えば正確（現行コードでは未対応）
