## Phase 1: 球体ハーネス

**ゴール**: Three.js + React Three Fiber で球体が表示され、ドラッグ回転・ズームが動作する

**成果物**:
- `app/globe/page.tsx` — データ取得なし、球体コンポーネントのみ
- `client/components/GlobeView.tsx` — 球体メッシュ + OrbitControls

**内容**:
- `three`, `@react-three/fiber`, `@react-three/drei` をインストール
- 単色の球体（半径1）を描画
- OrbitControls でドラッグ回転・スクロールズーム・右ドラッグチルト
- 背景は宇宙（黒）、環境光 + 方向光
- 球体表面にグリッド線（緯度経度風）をワイヤフレームで重畳

**ハーネス確認項目**:
- [ ] `/globe` にアクセスして球体が表示される
- [ ] ドラッグで回転、スクロールでズーム、右ドラッグでチルトが動く
- [ ] `npm run lint` エラーなし

## Phase 2: Globe API + 府省庁ダミー配置

**ゴール**: APIから37省庁データを取得し、球面上にFibonacci格子で点を配置して表示する

**成果物**:
- `app/api/map/globe/route.ts` — 府省庁レベルの球面データ配信
- GlobeView に府省庁マーカー（ラベル付きの点）を描画

**内容**:
- API: `rs2024-structured.json` から府省庁ごとの `totalSpending`, `areaFraction`, `projectCount` を計算
- Fibonacci格子で37省庁の `centroid [lat, lon]` を算出
- GlobeView: 各省庁を球面上のスプライト or ビルボードで表示（省庁名 + 金額）
- 省庁ごとに色相を割り当て（HSLのH値を37分割）
- クリックで省庁名・金額をコンソールに出力

**ハーネス確認項目**:
- [ ] 球体上に37個の色付きマーカーが表示される
- [ ] 各マーカーに省庁名が読める
- [ ] ズームインすると個別マーカーが大きく見える
- [ ] `/api/map/globe` が正しいJSONを返す

## Phase 3: 球面Voronoi — 府省庁領域

**ゴール**: 37省庁が面積比例のVoronoi領域として球面を分割する

**成果物**:
- API側でVoronoi計算 → ポリゴン頂点座標を配信
- GlobeView でポリゴンメッシュとして描画

**内容**:
- `d3-geo-voronoi`, `d3-geo` をインストール
- API: Fibonacci格子の初期配置から球面Voronoiを計算
- 面積比例補正: 反復的に重心を移動し、各領域面積を `areaFraction` に近づける（Lloyd緩和 + 面積重み）
- ポリゴン頂点座標をレスポンスに含める
- GlobeView: 各省庁をポリゴンメッシュとして球面に貼り付け
- 配色: 省庁色相 + 執行率で彩度/明度を変調
- マージン: ポリゴン間に隙間（球面の~30%を「海」として暗色表示）

**ハーネス確認項目**:
- [ ] 球体が37色のVoronoi領域で覆われている
- [ ] 大きな省庁（厚労省・財務省等）が大きな領域、小さな省庁が小さい領域
- [ ] 領域間に暗色のマージン（海）が見える
- [ ] ホバーで省庁名がツールチップ表示

## Phase 4: LOD Level 2 — 局・庁分割

**ゴール**: ズームインすると省庁領域が局・庁サブ領域に分裂する

**成果物**:
- APIレスポンスに局・庁レベルのVoronoiデータを追加
- GlobeViewにLOD切替ロジックを実装

**内容**:
- API: 各省庁の `children` に局・庁ごとのVoronoiポリゴンを追加
  - 省庁領域内を局・庁の支出比でサブ分割（親ポリゴン内でのVoronoi）
- GlobeView: カメラ距離を監視
  - 遠い（全球）: 省庁ポリゴンのみ
  - 近い: カメラに近い省庁のみ局・庁ポリゴンに切替
- 分裂アニメーション: 親ポリゴン → 子ポリゴン群にモーフィング（またはフェード切替）
- 局・庁間マージン（「川」）を省庁間マージン（「海」）より細く

**ハーネス確認項目**:
- [ ] 遠景で37省庁が見える
- [ ] 1つの省庁にズームインすると局・庁に分裂する
- [ ] 局・庁の面積が支出額に比例している
- [ ] ズームアウトすると省庁レベルに戻る

## Phase 5: 個別事業レベル + 情報パネル

**ゴール**: 最大ズームで個別事業（4,617件）が見え、クリックで詳細を表示

**成果物**:
- APIレスポンスに事業レベルを追加
- GlobeViewに事業ポリゴン + 情報パネルを実装

**内容**:
- API: 局・庁 `children` に個別事業のVoronoiポリゴンを追加
  - 中間階層（部・課・室・班・係）は `hierarchyPath` で保持するが、Voronoiは局・庁→事業で直接分割
  - 同じ課の事業は近接配置（重心のシード位置を課単位でクラスタリング）
- GlobeView LOD Level 3:
  - 近距離: 局・庁がさらに個別事業に分裂
  - 事業ポリゴン間のマージン（「垣根」）は極細
- 情報パネル（HTMLオーバーレイ）:
  - クリックした領域のパンくず（府省庁 › 局 › 課 › 事業名）
  - 実質支出額・予算額・執行率
  - 面積換算値
  - 品質スコア
- ダブルクリック → その領域にカメラをアニメーション移動

**ハーネス確認項目**:
- [ ] 最大ズームで個別事業の領域が見える
- [ ] 事業クリックで情報パネルが表示される
- [ ] パンくずの各段階がクリック可能でズームアウトする
- [ ] ダブルクリックでその領域にフォーカス移動

## Phase 6: パフォーマンス最適化 + 仕上げ

**ゴール**: 実用レベルのフレームレートとUI品質

**内容**:
- 遠景テクスチャベーク: 省庁レベルの配色を1枚のテクスチャにベーク → ポリゴン数削減
- Frustumカリング: カメラの裏側のポリゴンを描画スキップ
- InstancedMesh: 同一LODレベルのポリゴンをバッチ化
- `会計区分`による配色モード切替（一般=陸地、特別=海洋）
- ヘッダー: `/sankey`, `/quality`, `/map` へのナビゲーションリンク
- ステータスバー: 現在のズームレベル・表示領域数・LODレベル

**ハーネス確認項目**:
- [ ] 4,617事業表示時に60fps維持（または30fps以上）
- [ ] 全球 → 個別事業までのズームが滑らか
- [ ] `/globe` から他ページへのナビゲーションが動作
- [ ] `npm run build` が成功する
