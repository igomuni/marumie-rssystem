## Phase 2: Globe API + 府省庁マーカー配置 ✅

**PR**: #90

**ゴール**: APIから省庁データを取得し、球面上にFibonacci格子で点を配置して表示する

**成果物**:
- `app/api/map/globe/route.ts` — 府省庁レベルの球面データ配信
- GlobeView に府省庁マーカー（ラベル付きの点）を描画

**内容**:
- API: `rs2024-structured.json` から府省庁ごとの `totalSpending`, `areaFraction`, `projectCount` を計算
- Fibonacci格子で省庁の `seed [lon, lat]` を算出
- 省庁ごとに色相を割り当て（HSLのH値を等分割）
- クリック・ホバーで省庁名・金額を表示

## Phase 3: Icosphere面積比例マッピング ✅

**PR**: #91

**ゴール**: 府省庁が実質支出額に比例した面積で球面を分割する

**成果物**:
- GlobeView をicosphere方式に全面改修
- API に小規模省庁の集約ロジックを追加
- [府省庁別立体角一覧](20260314_1310_府省庁別立体角一覧.md)

**内容**:
- icosphere（正二十面体の再帰4分割、レベル6 = 81,920面）を生成
- 面を北極→南極の緯度帯順にソートし、支出額比率で連続割り当て
- レベル6で0面になる下位5省庁を「その他」に集約（33省庁 + その他 = 34エントリ）
- 単一メッシュ＋頂点カラーによる高速描画（`toNonIndexed()` で面ごとの色分け）
- ホバーハイライト（全面色変更）、ツールチップ（面数表示付き）
- ラベルを各省庁の北端境界に配置、ズーム連動スケーリング
- Zoom率UI（デフォルト1.00x、nearクリップ防止 `minDistance = SPHERE_RADIUS + camera.near`）

**経緯**:
- 球面Voronoi → multi-seed Voronoi → icosphere緯度帯割り当てと方式を試行
- Voronoiは等面積に収束する性質があり、厚労省65.7%の極端な偏りと相性が悪かった
- 緯度帯割り当てが面積比例を正確に実現でき、描画性能も良好

## Phase 4: 境界滑らか化・描画改善

**検討資料**: [Globe領域割り当てアルゴリズム検討](20260314_1553_Globe領域割り当てアルゴリズム検討.md)

**ゴール**: 府省庁間の境界を滑らかにし、視認性を向上する

**候補施策**:
- 境界面ローカルswap: 隣接グラフを構築し、境界長が短くなる方向に面を交換
- 細分割レベル上げ（7 = 327,680面）: 解像度向上だがGPU負荷増
- 配色改善: 隣接する府省庁の色が類似しないよう配色ロジックを調整

**ハーネス確認項目**:
- [ ] 境界線が現在よりジャギーが少ない
- [ ] 面積比例の精度が維持されている（±1%以内）
- [ ] フレームレートが低下していない

## Phase 5: LOD — 局・庁 → 個別事業

**ゴール**: ズームインで省庁領域が局・庁 → 個別事業に分裂する

**内容**:
- API: 各省庁の `children` に局・庁レベルの支出データを追加
- GlobeView: カメラ距離に応じたLOD切替（省庁 → 局・庁 → 個別事業）
- 省庁領域内をサブ分割（同じicosphere面を局・庁ごとに再割り当て）
- 情報パネル: クリックでパンくず（府省庁 › 局 › 課 › 事業名）、支出額・品質スコア表示
- ダブルクリック → 領域にカメラをアニメーション移動

**ハーネス確認項目**:
- [ ] 遠景で34エントリ（33省庁 + その他）が見える
- [ ] 1つの省庁にズームインすると局・庁に分裂する
- [ ] 最大ズームで個別事業の領域が見える
- [ ] 事業クリックで情報パネルが表示される
