# /sankey2 Phase 5: フィルタ・検索 設計

## 背景

Phase 4でクリック選択・BFS探索・サイドパネルを実装し、個別ノードの詳細確認とフロー追跡が可能になった。
しかし37,258ノードの中から目的のノードを見つけるにはZoom/Pan頼みで、特定の府省庁や事業、支出先に素早くたどり着けない。

Phase 5では**フィルタリング**と**検索**を追加し、目的のノードにすぐたどり着ける操作性を実現する。

## 5-1: 府省庁フィルタ

### 概要
特定の府省庁のみ表示するトグルフィルタ。

### UI設計
- ヘッダー情報エリアの下に「フィルタ」ボタンを配置
- クリックでドロップダウンパネル表示（全府省庁のチェックボックスリスト）
- 「すべて選択」「すべて解除」ボタン
- フィルタON時はボタンにバッジ表示（選択数）

### ロジック
- `ministryFilter: Set<string>` state（空 = フィルタなし = 全表示）
- LOD計算後の `visibleNodes` に追加フィルタ:
  - `total` ノード: 常に表示
  - `ministry` ノード: `node.label ∈ ministryFilter`
  - `project-*` / `recipient` ノード: `node.ministry ∈ ministryFilter`
- エッジ: 両端ノードが表示対象の場合のみ描画（既存ロジックで対応済み）
- フィルタなし（Set空 or 全選択）時はフィルタ処理をスキップ

### 府省庁リスト取得
- `data.nodes` から `type === 'ministry'` のノードを抽出、金額降順でソート
- 動的生成（ハードコード不要）

## 5-2: ノード名検索

### 概要
インクリメンタルサーチでノードを検索し、選択候補をリスト表示 → クリックでズーム移動。

### UI設計
- ヘッダーエリアに検索入力フィールド（`<input>`）
- 2文字以上入力で候補ドロップダウン表示（最大20件）
- 候補はtype別の色ドット + ノード名 + 金額
- 候補クリック → そのノードを選択状態 + ズーム移動（Phase 4の `handlePanelNodeClick` 相当）
- Escでドロップダウン閉じる

### ロジック
- `searchQuery: string` state
- 検索対象: `data.nodes` の `node.label`
- 部分一致（`label.includes(query)`）、大文字小文字区別なし不要（日本語のみ）
- 結果は金額降順ソート
- パフォーマンス: 37,258ノードの `includes` はO(n)だが十分高速（< 1ms）

### 検索結果ハイライト
- 検索結果のノードIDをSetで保持（`searchResultIds`）
- 該当ノードの矩形に点線ボーダー or 明滅で視覚的に強調（オプション）

## 5-3: 金額閾値フィルタ

### 概要
指定金額以下のノードを非表示にするスライダーフィルタ。

### UI設計
- フィルタパネル内にレンジスライダー配置
- 対数スケール（1万円 〜 1兆円）
- 現在の閾値を金額表示
- スライダー操作でリアルタイムフィルタ

### ロジック
- `minAmount: number` state（デフォルト: 0 = フィルタなし）
- `visibleNodes` フィルタに追加: `node.amount >= minAmount`
- `total` ノードは閾値に関係なく常に表示
- 対数スケール変換: `value = 10^(sliderPos * 12 + 4)` （10^4=1万 〜 10^16=1京）
  - 実用範囲: 1万円 〜 10兆円程度

### HUD表示
- フィルタ適用中は「最小金額: XX億円」をヘッダー情報に表示

## 5-4: URLパラメータ連携

### 概要
フィルタ・検索・選択状態をURLクエリパラメータに反映し、状態の共有・ブックマークを可能にする。

### パラメータ設計

| パラメータ | 型 | 例 | 説明 |
|-----------|------|-----|------|
| `m` | CSV | `m=厚生労働省,文部科学省` | 府省庁フィルタ（省略=全表示） |
| `min` | number | `min=100000000` | 最小金額（円単位） |
| `q` | string | `q=医療` | 検索クエリ |
| `s` | string | `s=ministry-厚生労働省` | 選択ノードID |

### ロジック
- `next/navigation` の `useSearchParams` + `useRouter` で読み書き
- State変更 → URLパラメータ更新（`router.replace` で履歴汚染なし）
- 初期表示時: URLパラメータからstateを復元
- デフォルト値の場合はパラメータ省略（URLをクリーンに保つ）

## 実装順序

1. **5-2: ノード名検索**（単体で最も実用的、依存なし）
2. **5-1: 府省庁フィルタ**（検索と組み合わせで強力）
3. **5-3: 金額閾値フィルタ**（スライダーUI）
4. **5-4: URLパラメータ連携**（全フィルタ完成後にまとめて対応）

## パフォーマンス考慮

- フィルタ条件変更時の再計算は `useMemo` の依存配列で自動対応
- 37,258ノードのフィルタリングはO(n)で十分高速（< 5ms）
- 検索のデバウンス: 150ms（入力のたびに再計算しない）
- 府省庁フィルタはSet.has()でO(1)判定
