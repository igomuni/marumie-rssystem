## 主な変更内容

### 1. ミニマップ座標の補正

- ラベルシフト後の描画位置をミニマップに反映する。
- 事業支出ノードは対応する事業予算ノード側のシフト情報を使い、ミニマップ上の y 座標ずれを抑える。
- ミニマップの world height を固定の `svgHeight` ではなく、実描画下端から計算する。

### 2. 列ラベル・列ツールチップの金額表示改善

- 事業列の集計で、事業予算ノードは `rawValue` を優先する。
- 事業列ホバー時のツールチップで、予算合計と支出合計を分けて表示する。

### 3. ズーム時ラベルフォントの安定化

- `getZoomLabelScale()` を追加し、fit zoom を基準にズームイン時だけラベルフォントを拡大する。
- ラベルシフト計算にもズーム連動スケールを反映する。
- 従来の隣接ラベル間隔ベースのフォント縮小ロジックを取り除き、最大倍率を `ZOOM_FONT_MAX_RATIO` に寄せてシンプルにする。

### 4. フォントサイズ操作 UI の移設

- サイドパネル内にあった基準フォントサイズ設定を、画面左下の独立コントロールへ移す。
- ミニマップやサイドパネルの開閉状態に応じて、フォント操作 UI の位置を調整する。
- 数値入力は一時入力値 `baseFontPxInput` を持ち、blur または Enter で範囲内に丸めて反映する。

## 反映方針

### 推奨: 小規模同期 PR として一括反映

今回の `docs/` 除外後の差分は `app/sankey-svg/page.tsx` のみで、データファイルやビルドスクリプトの変更はない。過去の同期と同じく rs-vis 側に同期ブランチを作り、対象ファイルを反映した 1 PR にまとめるのが適切。

利点:

- 差分が 1 ファイルに閉じており、レビューしやすい。
- データ生成や `.gz` コピーが不要。
- rs-vis 側の公開向け README / LICENSE / docs を触らずに済む。

注意点:

- 前回同期が rs-vis 側で本当に `3fc5635` 相当まで入っているかは、作業開始前に rs-vis の `main` で確認する。
- `app/sankey-svg/page.tsx` は差分が大きめなので、単純コピー後に build と主要画面の目視確認を行う。
