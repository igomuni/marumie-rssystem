## 2. 本タスクで使う機能との適合

スクリプト `scripts/score-project-quality-ai.py` は次を要求する。Nex-N2-Pro の `supported_parameters` と突合:

| スクリプトが使う機能 | パラメータ | Nex-N2-Pro 対応 | 効果 |
|----------------------|-----------|:--:|------|
| JSON強制 | `response_format` (`json_object`) | ✅ | 出力をJSONに固定 |
| 構造化出力 | `structured_outputs` | ✅ | スキーマ準拠を後押し |
| 決定的化 | `temperature` (=0) | ✅ | 再現性向上 |
| 推論抑制 | `reasoning` (`effort: low`) | ✅ | 分類タスクの推論コスト削減 |
| 長文バッチ | 262K ctx | ✅ | 20件バッチでも余裕 |

→ スクリプトが前提とする **JSON強制＋構造化出力＋temperature=0** をフル対応。これが「疎通でフォールバック補完ゼロ」の技術的裏付け。

## 4. 適性評価

### 4-1. 向いている用途（今回の使い方）

- **疎通確認・プロンプト調整・少数サンプルでの品質目視**。$0 で response_format/structured_outputs/temperature を完全サポートし、JSONパイプラインの動作確認に最適だった。
- 262K の長コンテキストとMoEの素性で、判定品質も実用域。

### 4-2. 本番一括採用には非推奨（理由）

- **⚠️ 提供終了が確定（最重要）**: `expiration_date: 2026-06-22`（OpenRouter照会・ページの「Going away」バッジと一致）。本レポート作成（6-20）の**2日後に消滅**するため、全件確定生成の基盤には使えない。**今後の追加検証も期限前提**で、別の `:free` への切替を見込むこと。
- **レート制限**: 無料枠は同時実行・分間リクエストの制限が厳しめ。87,653タプル＋5,642事業の全件では `--concurrency` を絞る必要があり、所要が伸びる／途中で 429 が出やすい（キャッシュで再開は可能）。
- **可用性リスク**: 無料モデルは予告なく提供停止・仕様変更されうる（本モデルはまさに終了予定）。年度をまたいだ再現生成の基盤としては不安定。
- **基準の不一致**: 2025年度は `google/gemini-3.5-flash` で生成済み。**年度間でモデルを揃えないとスコア基準が揃わない**（同一事業でも判定の癖が異なりうる）。
- **データ方針**: 無料モデルはプロバイダ側でログ・学習利用される可能性がある一般的な前提。本データは公開済みの行政事業レビュー情報のため実害は小さいが、運用ポリシーとして認識しておく。

### 4-3. 推奨

- **疎通・検証フェーズ**: 本モデル（または他の `:free`）で十分。
- **本番（2024確定生成）**: **`google/gemini-3.5-flash`（2025と同一）**で全件生成し、年度間の基準を揃えるのが本筋。free で全件試行する場合も「暫定スコア」と割り切る。

## 6. まとめ

`nex-agi/nex-n2-pro:free` は、**$0 で JSON強制・構造化出力・temperature=0 を完全サポートする MoE モデル**で、今回の疎通検証では補完フォールバックゼロ・根拠全付与・良好な弁別を示し、検証用途として十分だった。ただし **`expiration_date: 2026-06-22` で2日後に提供終了が確定**しており、本番はもちろん追加検証の基盤にも継続採用できない。**本番の2024確定生成は `google/gemini-3.5-flash`（2025と同一）**、追加の無料検証が要る場合は終了予定のない別 `:free`（例: `qwen/qwen3-next-80b-a3b-instruct:free`）に切り替えること。
