# 府省庁別 品質スコア比較（2025年度データ）

## 1. 結論

**データ上は比較可能。ただし現行 `/quality` ページには府省庁横断の比較UIが無い**（単一府省庁で絞り込むセレクトのみ: [app/quality/page.tsx:328-337](../../app/quality/page.tsx#L328-L337)）。
`public/data/project-quality-scores-{year}.json` の各事業レコードに `ministry` と5軸スコア・`totalScore` が入っているため、集計はローカルで即座に可能。本ドキュメントはその集計結果。

**重要な注意**: 年度をまたいだ比較はしないこと。2024年度データは旧世代のAIスコアリングで生成されており、2025年度と採点基準が異なる（2024の平均は全体的に高く出る）。以下は **2025年度単独**の比較。

## 3. 読み取れること

- **レンジは狭い**: 事業数10以上の機関に限ると平均は 69〜87 の18ポイント幅に収まる。「良い省庁／悪い省庁」の断絶は無く、差は主に**特定（axisIdentify）と成果（axisEffective）の2軸**から生まれている。
- **構造（axisStructure）はほぼ全機関で満点**。比較軸として機能していない＝現状は識別力が無い。
- **下位は外務省(69.4)・法務省(69.5)**。両者とも `axisIdentify` が 46〜57 と突出して低い。支出先の特定可能性（相手先名・法人番号の記載）が弱い構造的要因があり、外務省は在外公館・国際機関向け、法務省は個人・矯正関連の支出が多いという性質を反映している可能性が高い。スコアの低さ＝不正ではなく、**報告様式と事業性質のミスマッチ**として読むべき。
- **予算規模と質は相関しない**: 厚労省は93兆円で平均77.8、国交省は11兆円で78.7、気象庁は0.08兆円で87.2。金額の大きい省庁が特に低いわけでも高いわけでもない。
- **こども家庭庁の `axisIdentify` 52.9 / `axisPurpose` 59.4** は事業数112に対して明確に低く、新設官庁として報告様式の運用が定着していない可能性がある。要個別確認。

## 4. 前提と限界

- スコアは報告の**形式品質**（特定可能性・説明性・成果設計の明確さ）であり、資金経路の透明性や事業の妥当性そのものではない（`metadata.notes` と同旨）。
- `axisEffective` はAI（`aiSource`: `openrouter:google/gemini-3.5-flash` 等）による判定を含む。同一モデル・同一プロンプトで全件生成されている前提での相対比較のみ有効。
- 単純平均のため、事業数の少ない機関ほどブレる。予算加重平均は別途必要なら算出可能。

## 5. UI化するなら（未実装・提案）

現状 `/quality` は単一府省庁の絞り込みのみ。比較を出すなら:

1. `summary` に `byMinistry`（機関別の n / 平均 / 中央値 / 5軸平均）を追加 — API側で集計してレスポンスに同梱
2. `/quality` に府省庁別の横棒チャート（5軸スタック or 総合スコア降順）を追加
3. 事業数の閾値（n>=10 など）でのフィルタを既定にし、少数事業機関の外れ値を隠す

いずれも未着手。実施するかは要判断。

## 再現コマンド

```bash
node -e '
const d=require("./public/data/project-quality-scores-2025.json");
const m={};
for (const it of d) { const k = it.ministry || "(不明)"; (m[k] = m[k] || []).push(it); }
const rows = Object.entries(m).map(([k, v]) => {
  const avg = f => v.reduce((s, x) => s + (x[f] ?? 0), 0) / v.length;
  return { m: k, n: v.length, avg: +avg("totalScore").toFixed(1) };
}).sort((a, b) => b.avg - a.avg);
console.table(rows);
'
```
