## Phase 0: 土台作り（破壊的変更なし）

並行ルート骨格とトークン定義を、既存 `/sankey-svg` を一切触らずに用意する。

- `app/sankey-svg-next/page.tsx` を `app/sankey-svg/page.tsx` からコピーで生成（最初は完全に同一の見た目）
- `app/sankey-svg-next/_layout.ts` を新設し、以下を定義
  - `DisplayMode` 型: `'compact-mobile' | 'compact-tablet' | 'standard-desktop' | 'dense-laptop' | 'presentation-tv' | 'projection'`
  - `LayoutTokens` 型: 設計書 §2 の項目
  - `MODE_TOKENS: Record<DisplayMode, LayoutTokens>` 定義（既定値）
  - `resolveDisplayMode(width: number, pointerCoarse: boolean, override?: DisplayMode): DisplayMode`
- `app/sankey-svg-next/page.tsx` に `displayMode` 状態と `tokens` を導入。**この時点ではトークンを参照するだけで挙動は変えない**（既存定数を `tokens.xxx` 経由に置き換えるリファクタのみ）
- 検証: 旧ルートと新ルートが視覚的に同一であることを Playwright で確認

**成果物**: コピーされた page.tsx、`_layout.ts`、視覚回帰なし

## Phase 1: トークン駆動への切り替え

既存ロジックの「直書き定数」をトークン参照に置き換え、デスクトップでの回帰がないことを担保する。

- `SEARCH_BOX_RESERVE` の `svgWidth < 1100 ? 92 : 56` 分岐を撤去、`tokens.searchBoxReservePx` に置換
- `SIDE_PANEL_WIDTH_DEFAULT = 310` を `tokens.sidePanelWidthPx` に置換
- `COLUMN_LABEL_FONT_PX_DEFAULT = 12` を `tokens.columnLabelFontPx` に置換、`fontScaleClampMin` でクランプ
- 各モードのトークン値は **standard-desktop / dense-laptop は現状値そのまま** を入れる
- 他モードは仮値（Phase 2以降で確定）
- 検証: 1440 / 1280 / 1024 で旧ルートと差分ゼロ

**成果物**: 数値の散在解消、デスクトップ回帰なし

## Phase 2: DisplayMode 自動判定と手動上書き

URL クエリ・localStorage・自動判定の3層で `displayMode` を解決する仕組みを通す。

- `?mode=compact-mobile` 等のクエリで強制指定（最優先）
- localStorage に保存された最後の手動選択を次点（クエリ無しのとき）
- `containerWidth` と `matchMedia('(pointer: coarse)')` で自動判定
- 設定UI上の DisplayMode 切替セグメント（モック準拠）は **置き場所だけ用意**、Phase 3 で見た目を作る
- 検証: クエリ切替でトークン値が切り替わり、`SEARCH_BOX_RESERVE` 等が動的に変化する

**成果物**: モード切替フックの基盤完成、まだ UI は素のまま

## Phase 3: compact-mobile レイアウト実装

採用モック [Mobile Sankey Layout.html](_assets/20260528_responsive/Mobile%20Sankey%20Layout.html) に沿って 390/360px の見た目を作る。

- 上部ツールバーを「検索（flex-1） + 年度ピル + ⚙ 設定」の3要素・全44px高に再構築（モード時のみ分岐）
- ⚙ 設定シートを実装: フォント・TopN・表示設定・DisplayMode 切替セグメント
- 列見出しを `with-amount` モードで再描画。`hidden` モードへの切替も `_layout.ts` レベルで仕込んでおく
- サイドパネルを `bottom-sheet` に切替（3段スナップ: 閉/半開/全開）
- 360px幅では `fullscreen` モードへ昇格（モック Frame D 準拠）
- **ラベル衝突回避**: フォーカス状態での非選択ノードラベル非表示、支出先ラベルの右ガッター退避（モック準拠の compact-mobile 専用ブランチ）
- 検証: Playwright で 390/360 を撮影し、モックとの構造差分を目視確認

**成果物**: モバイルで設計書の受け入れ条件を満たすレイアウト

## Phase 4: compact-tablet レイアウト実装

採用モック [Tablet Portrait Sankey Layout.html](_assets/20260528_responsive/Tablet%20Portrait%20Sankey%20Layout.html) に沿って 768px縦の見た目を作る。

- 上部ツールバー2段化（1段目: 検索+年度ピル、2段目: TopNスライダー+フォント+設定アイコン直接配置）
- `compact-tablet` モード時は `showTopNSliders` を常時 true 相当で固定（設計書「実装時の判断ポイント」）
- 列見出しを `name-only` モードで再描画
- サイドパネルは bottom-sheet を初期既定、docked は ⚙ ポップオーバーから手動切替可能（設計書「実装時の判断ポイント」）
- DisplayMode 切替を ⚙ アイコン直下のポップオーバーに格納（モバイルのフルシートと差別化）
- 検証: Playwright で 768/1024 を撮影し、モックとの構造差分を目視確認

**成果物**: タブレットで設計書の受け入れ条件を満たすレイアウト

## 受け入れ条件チェックリスト（Phase 完了時に確認）

- [ ] タブレット縦・スマホ幅で上部UIと列見出しが重ならない（Phase 3, 4）
- [ ] スマホ幅で検索・年度・フォント/TopN/表示設定すべてが2タップ以内（Phase 3）
- [ ] ノード詳細が docked / bottom-sheet / fullscreen それぞれで機能する（Phase 3, 4）
- [ ] 主要ボタンが最小44pxヒットエリア（Phase 6）
- [ ] レイアウトトークンのモード切替で表示密度を一括変更可能（Phase 2）
- [ ] `npm run lint` / `npx tsc --noEmit` パス（各 Phase 末）
- [ ] デスクトップでの視覚回帰なし（Phase 1, 7）

## リスク

- **page.tsx 4,500行のコピー由来のドリフト**: Phase 0〜6 の間に旧ルートに修正が入ると同期が必要。期間中の旧ルート修正は最小化する方針で進める。
- **モード判定とコンテナリサイズの競合**: DisplayMode 切替時にレイアウト計算が一瞬崩れる可能性。Phase 2 で `useLayoutEffect` 等で同期化が必要かもしれない。
- **bottom-sheet のスクロール衝突**: Sankey のパン操作とシートのスワイプ操作が同じジェスチャになる。Phase 3 でタッチイベントの優先順位設計が必要。
