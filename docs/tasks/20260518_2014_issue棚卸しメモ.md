## #135 の位置づけ

#135 は当初「一括計上予算科目の事業に予算額が反映されない問題」として起票された。

現在の実装では、予算0・支出ありの事業がまだデータ上に残っている。

| 年度 | 予算0・支出あり件数 | 支出合計 |
|---|---:|---:|
| 2024 | 224件 | 約6,564億円 |
| 2025 | 288件 | 約9,463億円 |

この状態は単純な表示バグというより、RSシステムのデータ構造上、予算が一括計上側の事業に載り、執行・直接支出が個別事業側に現れるケースを示している可能性が高い。

そのため、#135 は「予算額を機械的に補完して消すべき不具合」ではなく、以下の観点で残す。

- RSデータの一括計上・個別執行のねじれを観測できる Issue
- 会計・予算執行データの説明 UI を検討するための起点
- 将来的に `予算額は0だが執行/直接支出あり` をツールチップやサイドパネルで明示する改善候補

## #135 対応時の注意

`2-1` の `執行額(合計)` をフォールバック予算額として使う案は過去に検討済みだが、安易に採用すると次の問題がある。

- 元データの「予算0」という事実が見えなくなる
- 一括計上側の予算と個別事業側の執行の関係を、存在しない明示リンクとして誤読させる可能性がある
- 府省庁ノード・総額ノードとの整合性を別途再設計する必要がある

次に進めるなら、補正よりも先に「データ由来の状態を説明する表示」を検討するのがよさそう。
