(B) bottom-sheet (half) — alt

ユーザー設定で切替可。Sankey本体が画面いっぱいに広がる代わり、詳細閲覧時は半開きシートで2列リスト表示。 横長スペースを活かして事業/支出先を並べて読める。「自動」モード時は 768px 縦は docked が既定。

01 — Two-row toolbar (§3)

クラスタを縦に積む

1段目（検索 + 年度）は画面コンテキストの起点、2段目（TopN + フォント + ⚙）は分析パラメータ。 モバイルと違い TopN / フォント は直接配置で1タップで触れる。 searchBoxReservePx は 60+56 = 116px 確保。

02 — Side panel (§4)

docked と sheet を併記

既定は docked (sidePanelWidthPx=310)。Sankey が常に左 458px に収まり、選択ノードと並列比較できる。 bottom-sheet 案は本体最大化重視で、2列リストで横幅を活用。⚙ポップオーバーから即時切替。

03 — DisplayMode (§1)

フルシートではなくポップオーバー

モバイルは画面遷移コストが高いのでフルシート、タブレットは ⚙ 直下の小ポップオーバーで DisplayMode / 列見出し / パネル方式を切替。閉じても Sankey が見え続け、変更結果を即確認できる。
