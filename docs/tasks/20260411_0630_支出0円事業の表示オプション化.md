# 支出0円事業の表示オプション化

## 目的

ユーザーが「支出が0円の事業」をSankey図から除外できるようにすることで、
実際に支出が発生した事業のみを対象にした予算・省庁比較が可能になる。

---

## 背景・現状

### データ規模

- 全事業数（project-spending）: **5,664件**
- うち支出が0円: **1,047件**（約18.5%）
- 0円事業の予算合計: **約1.44兆円**

### 現状の挙動

`project-spending` ノードの `value` が 0 の事業（CSV に実績がない事業）は、
現在の `filterTopN` でも除外されず、以下の計算に参入している。

| 計算対象 | 影響 |
|---------|------|
| `ministryBudgetValue` | 省庁の予算合計に0円事業の予算分が含まれる |
| `totalBudget` | 総計に0円事業の予算分（1.44兆円）が含まれる |
| `otherProjectBudgetTotal` | 集約ノードの予算合計に含まれる |
| `topMinistryAllProjects` | TopN事業選択の母集団に含まれる（ただし windowValue=0 のため選ばれにくい） |
| 集約ノードの「N事業」件数 | 0円事業がカウントされる |

※ `effectivelyHiddenIds` は `n.value > 0` を条件にしているため、`value === 0` の事業は対象外。

---

## 要件

### 設定オプション

| 項目 | 内容 |
|-----|------|
| 名称 | 「支出が0円の事業を対象にする」チェックボックス |
| 場所 | 設定パネル（他の TopN スライダーと同じ領域） |
| デフォルト | **OFF**（0円事業を除外する） |
| State 名 | `includeZeroSpending: boolean`（`false` = 除外） |

### OFF時（`includeZeroSpending === false`）の除外対象

「支出が0円の事業」＝ `n.type === 'project-spending' && n.value === 0` のノード。

これらを **`zeroSpendingProjectIds`** として `filterTopN` の先頭で集合を作り、
以下の計算すべてから除外する。

| 計算箇所 | 変更内容 |
|---------|---------|
| `topMinistryAllProjects` フィルタ | `zeroSpendingProjectIds` を除く条件を追加 |
| `otherProjects` フィルタ（2か所） | `zeroSpendingProjectIds` を除く条件を追加 |
| `ministryBudgetValue` 集計（project-budget ループ） | 対応する budget ノードを除く（`zeroSpendingBudgetIds` を作成して skip） |

`ministryBudgetValue` の変化は `totalBudget`・`otherMinistryBudgetValue`・`otherProjectBudgetTotal` に自動伝播するため、追加の変更は不要。

## 変更ファイルと変更内容

### `app/lib/sankey-svg-filter.ts`

#### `filterTopN` シグネチャ

```
filterTopN(
  allNodes, allEdges,
  topMinistry, topProject, topRecipient,
  recipientOffset,
  pinnedProjectId,
  includeZeroSpending: boolean = true   ← 追加
)
```

#### 変更箇所（ステップ順）

1. **先頭（step 1 以前）**: `includeZeroSpending === false` のとき
   ```
   zeroSpendingProjectIds = Set<string>  // project-spending で value===0 のもの
   zeroSpendingBudgetIds  = Set<string>  // 対応する "project-budget-{projectId}"
   ```
   `true` のときは空の Set（既存コードに影響なし）。

2. **step 4 `topMinistryAllProjects` フィルタ（L68）**:
   ```diff
   - n.type === 'project-spending' && topMinistryNames.has(n.ministry || '')
   + n.type === 'project-spending' && topMinistryNames.has(n.ministry || '') && !zeroSpendingProjectIds.has(n.id)
   ```

3. **`otherMinistryProjects` フィルタ（L106）**:
   ```diff
   - n.type === 'project-spending' && !topMinistryNames.has(n.ministry || '') && !topProjectIds.has(n.id) && !effectivelyHiddenIds.has(n.id)
   + ... && !zeroSpendingProjectIds.has(n.id)  を追加
   ```

4. **`ministryBudgetValue` 集計ループ（L152）**:
   ```diff
     if (effectivelyHiddenBudgetIds.has(n.id)) continue;
   + if (zeroSpendingBudgetIds.has(n.id)) continue;
   ```

### `app/sankey-svg/page.tsx`

- `const [includeZeroSpending, setIncludeZeroSpending] = useState(false);` を追加
- `filterTopN` 呼び出しに引数を追加
- 設定パネルに「支出が0円の事業を対象にする」チェックボックスを追加

## 注意点

- `effectivelyHiddenIds` は `n.value > 0` の事業が対象なので、`zeroSpendingProjectIds` と重複しない。両方を独立して管理する。
- デフォルト OFF（除外）にすることで、初期表示が「実支出のある事業のみ」になる。ON にすると現状と同じ挙動。
- 支出が0円かつ `topN` に選ばれている事業（ピン留め等）はエッジがないため視覚上の問題はないが、ONの場合でも除外はしない（既存の `pinnedProjectId` ロジックと整合するため）。
