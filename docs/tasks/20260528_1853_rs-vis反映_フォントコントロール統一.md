## 変更内容（移植項目）

### 1. 基準フォントサイズコントロールの UI/UX をオフセット調整に揃える

`app/sankey-svg/page.tsx` のフォントコントロール部（`showFontControls` トグルパネル）周り。

- **リピートタイマー追加**: `fontRepeatRef` / `stopFontRepeat`（既存の `offsetRepeatRef` / `stopOffsetRepeat` と同パターン）
- **編集状態の state 追加**: `isEditingBaseFont`
- **スライダー幅**: `flex: 1, minWidth: 0` → `width: 60` 固定（オフセット調整と同幅）
- **数値表示**: 常時 input → 「クリックで input に切り替わるボタン」形式
  - 非編集時: `aria-label="基準フォントサイズ編集を開始"`、`color: '#999'`、`fontSize: META_FONT_PX_DEFAULT`（**固定値**、`baseFontPx` に連動させない）、borderless
  - 編集時: `aria-label="基準フォントサイズ(数値)"`、`autoFocus`、Enter で確定、Escape で取消、blur で確定
- **上下リピートボタン**: 縦並び 16×16、SVG 12×12、長押し（400ms→150ms 間隔）対応
- **リセットボタン**: 枠あり 32×32 → アイコンのみ（`background: transparent`, `border: 'none'`, 14×14 SVG, `color: '#555'`）
- **パネルレイアウト**:
  - 固定幅 `FONT_CONTROL_PANEL_W_PX` を削除（コンテンツ幅で自動）
  - `minHeight: FONT_CONTROL_BUTTON_PX`, `display: 'flex', alignItems: 'center'` を追加（スライダー行を縦中央配置）
- **開閉時の縦位置ジャンプ解消**: コンテナの `bottom: showMinimap || showFontControls ? 8 : 16` → `bottom: showMinimap ? 8 : 16`
- **未使用定数削除**: `FONT_CONTROL_PANEL_W_PX`, `FONT_CONTROL_FONT_PX`

### 2. 省庁ラベルのクリップ幅を次列の左側予算ラベル幅に応じて動的に制限

ズーム＋大きいフォント時に省庁ノードのラベルが事業ノード左側の予算金額ラベルと重なる問題に対処。

`clip-col-N` 生成部（line 2417 付近）を以下のように変更:

```tsx
const nextX0 = nextColNodes.length > 0
  ? Math.min(...nextColNodes.map(n => getNodeInnerX0(n)))
  : labelStart + screenWToInner(layout.colSpacing * horizontalScale - screenNodeW);
// 次列ノードが左側ラベル（事業予算金額）を持つ場合、その想定幅分を予約
const leftLabelChars = nextColNodes
  .filter(n => n.type === 'project-budget')
  .reduce((m, n) => {
    const main = formatYen(n.value);
    const raw = n.isScaled && n.rawValue != null ? ` / ${formatYen(n.rawValue)}` : '';
    return Math.max(m, (main + raw).length);
  }, 0);
const labelScale = getZoomLabelScale(zoom, baseZoom);
const innerFontPx = (mapLabelFontPx * labelScale) / zoom;
const leftLabelReserve = leftLabelChars > 0
  ? innerLabelGap * 2 + leftLabelChars * innerFontPx * 0.7
  : 0;
const clipEnd = nextX0 - leftLabelReserve;
```

### 3. e2e テスト調整

`tests/e2e/sankey-svg.spec.ts:131` 付近、フォントサイズ入力フロー:

```diff
 await page.getByLabel('表示設定を開く').click();
+await page.getByLabel('基準フォントサイズ編集を開始').click();
 await page.getByLabel('基準フォントサイズ(数値)').fill('24');
 await page.getByLabel('基準フォントサイズ(数値)').press('Enter');
```

ボタンと input の aria-label を分離したため、`fill` 前に編集モードへ切替えるクリックを挟む。

1. rs-vis で feature ブランチ作成（例: `chore/font-control-style-align`）
2. 上記 3 セクションを順に適用
3. `npm run lint` / `npx tsc --noEmit` で品質チェック
4. `npm run dev` で実機確認:
   - フォントコントロールの開閉位置がジャンプしないこと
   - 数値クリックで input 切替、Enter/Escape/blur で確定・取消
   - 上下ボタン長押しで連続調整
   - フォント上げ＋ズームで省庁ラベルと事業予算ラベルが重ならないこと
5. PR 作成
