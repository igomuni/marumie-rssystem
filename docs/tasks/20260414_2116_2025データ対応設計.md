# /sankey-svg 2025データ対応 設計

## 目的

ユーザーが `/sankey-svg` 上で2025年度のRSシステムデータを2024年度データと切り替えて参照できるようにするため。

---

## 現状

### データフロー（2024版）

```
data/download/RS_2024/*.zip
    ↓ npm run normalize (normalize_csv.py)
data/year_2024/*.csv
    ↓ tsx scripts/generate-sankey-svg-data.ts   ← リネーム後
public/data/sankey-svg-graph.json               ← リネーム後
    ↓ npm run compress-data
public/data/sankey-svg-graph.json.gz            ← リネーム後
    ↓ prebuild (decompress-data.sh)
/data/sankey-svg-graph.json (fetch target)      ← リネーム後
```

### ハードコーディング箇所

| ファイル | 値 | 用途 |
|---------|-----|------|
| `scripts/normalize_csv.py` line 39-40 | `RS_2024` / `year_2024` | 入出力ディレクトリ |
| `scripts/generate-sankey3-data.ts` line 24 | `year_2024` | CSVディレクトリ |
| `scripts/generate-sankey3-data.ts` line 26 | `sankey3-graph.json` | 出力ファイル名 |

| `package.json` `compress-data` | `sankey3-graph.json` | gzip対象 |

### 2025データの状態

- `data/download/RS_2025/` に ZIP 形式で取得済み
- CSV ファイル名パターン: `1-1_RS_2025_*`、`2-1_RS_2025_*` … 2024 と同一構造
- `normalize_csv.py` は `RS_2024` の ZIP を対象にハードコードされているため未展開

---

## 設計方針

**年度別ファイルを並存**し、UI の URL パラメータで切り替える。

- `sankey3` という命名を `sankey-svg` に統一する（スクリプト・JSON ファイル名）
- `public/data/sankey-svg-graph.json` を `sankey-svg-2024-graph.json` にリネーム（`.gz` も同様）
- `public/data/sankey-svg-2025-graph.json` を新規生成
- `/sankey-svg?year=2025`（デフォルト）/ `?year=2024` で切り替え

---

## 変更スコープ

### 1. `scripts/generate-sankey3-data.ts` → リネーム

`scripts/generate-sankey-svg-data.ts` に改名する。

CLI 引数（`process.argv`）で年度を受け取り、以下を動的に構成する:

| 定数 | 現行値 | 変更後 |
|------|--------|--------|
| `DATA_DIR` | `data/year_2024` | `data/year_{YEAR}` |
| `OUTPUT_FILE` | `sankey3-graph.json` | `sankey-svg-{YEAR}-graph.json` |
| `TARGET_BUDGET_YEAR` | `2023` | `YEAR - 1` |
| CSV ファイル名 | `1-1_RS_2024_*` | `1-1_RS_{YEAR}_*` |
| スクリプト冒頭コメント | `/sankey3` | `/sankey-svg` |

```
tsx scripts/generate-sankey-svg-data.ts 2025
→ public/data/sankey-svg-2025-graph.json
```

### 2. `scripts/normalize_csv.py`

CLI 引数 `--year`（デフォルト `2024`）を追加し、`INPUT_DIR` / `OUTPUT_DIR` を動的に構成する。

```
python3 scripts/normalize_csv.py --year 2025
→ data/download/RS_2025/ → data/year_2025/
```

既存 `npm run normalize` の動作は変えない（デフォルト `2024`）。

### 3. `package.json` — スクリプト追加・変更

```json
"normalize-2025": "python3 scripts/normalize_csv.py --year 2025",
"generate-sankey-svg": "tsx scripts/generate-sankey-svg-data.ts 2024",
"generate-sankey-svg-2025": "tsx scripts/generate-sankey-svg-data.ts 2025",
"compress-data": "... && gzip -9 -k -f public/data/sankey-svg-2024-graph.json && gzip -9 -k -f public/data/sankey-svg-2025-graph.json"
```

既存 `compress-data` の `sankey3-graph.json` 参照を上記2ファイルに置き換える。

### 4. `scripts/decompress-data.sh`

`sankey3-graph.json.gz` の展開ブロックを `sankey-svg-2024-graph.json.gz` と `sankey-svg-2025-graph.json.gz` の2ブロックに置き換える（各ファイルが存在する場合のみ展開）。

### 5. ファイルリネーム

Git の追跡を維持するため `.gz` は `git mv` を使用する:

```bash
git mv public/data/sankey3-graph.json.gz public/data/sankey-svg-2024-graph.json.gz
```

`.json`（展開済み、`.gitignore` 対象）は単純なファイル操作で対応。

| 現在 | 変更後 |
|------|--------|
| `public/data/sankey3-graph.json` | `public/data/sankey-svg-2024-graph.json` |
| `public/data/sankey3-graph.json.gz` | `public/data/sankey-svg-2024-graph.json.gz` |
| `scripts/generate-sankey3-data.ts` | `scripts/generate-sankey-svg-data.ts` |

### 6. `app/sankey-svg/page.tsx`

**URL state に `year` を追加**

| 変更点 | 内容 |
|--------|------|
| `SankeyUrlState` | `year: '2024' \| '2025'` を追加 |
| デフォルト値 | `'2025'` |
| fetch パス | `/data/sankey-svg-${year}-graph.json` |
| ローディングメッセージ | `sankey3-graph.json` → `sankey-svg-{year}-graph.json` |
| URL エンコード | `p.set('year', year)` / `p.get('year')` |
| UI | 既存の設定パネル内に年度切り替えセレクタを追加 |

年度変更時は `graphData` をリセットしてリロードする（既存の `fetch` useEffect が `year` state の変化を検知）。

