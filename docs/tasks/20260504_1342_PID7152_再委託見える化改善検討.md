# PID7152 再委託見える化改善検討

## 対象

- 事業: 起業家等の海外派遣・シリコンバレー拠点形成事業
- PID: 7152
- 年度: 2025
- marumie URL: `https://marumie-rssystem.vercel.app/subcontracts/7152?year=2025`
- RSシステム URL: `https://rssystem.go.jp/project/ecd9934a-82db-49aa-a4db-8d1a0b670bb3?activeKey=payment`

## 確認した構造

この事業の再委託構造は、深いチェーンではなく次の2段構造。

```text
経済産業省
  -> A 独立行政法人日本貿易振興機構
      -> B デロイトトーマツファイナンシャルアドバイザリー、Plug&Play、CIC Innovation Communities、500 Startupsほか
```

ローカルデータ `public/data/subcontracts-2025.json` 上の概要:

- 最大深度: 2
- ブロック数: 2
- 表示内訳の支出先数: 13
- フロー数: 2
- Aブロック金額: 61.99億円
- Bブロック金額: 61.99億円

ブロック別の意味:

- A: 独立行政法人日本貿易振興機構、直接支出、役割は「全体統括」
- B: 再委託先グループ、役割は「J-StarX運営事務局・広報業務の実施、J-StarXプログラム運営等」

## 現画面で見づらい点

### 1. 構造より内訳の詰め込みが目立つ

現画面はBブロック内に支出先のtreemapを描画している。  
この案件では再委託階層自体は単純だが、Bブロック内の小さな矩形が多く、視線が「構造理解」より「読めない内訳」に向かう。

### 2. A/Bが同額で二重計上に見えやすい

AとBがどちらも61.99億円で表示される。  
これは `経産省 -> JETRO -> 再委託先群` の通過フローとして読めば自然だが、画面上では「Aにも61.99億円、Bにも61.99億円がある」ように見え、合計の扱いが直感的でない。

### 3. 件数の意味が元RSとずれて見える

元RSシステムの支出先上位者リストでは、Bブロックの支出先の数が `360` と表示されていた。  
一方、marumieの再委託ページではヘッダーに `支出先 13`、Bブロック内では表示内訳が12件程度に見える。

これは集約・名寄せ・表示対象の違いと思われるが、画面上に説明がないため、データ欠落のように見える可能性がある。

### 4. Bブロック名と役割が読み切れない

Bブロック名は長く、現画面では省略表示になる。  
本来重要なのは法人名の列挙より「J-StarX運営事務局・広報業務・プログラム運営等」という役割だが、役割テキストは小さく、初見では読みにくい。

### 5. 予算・執行額の表示とブロック金額の関係が分かりにくい

marumie画面ヘッダーでは予算・執行が `—` になっている一方、ブロックには61.99億円が表示される。  
元RSシステムでは予算・執行額テーブルに補正予算・計・執行額として `6,199,412千円` が表示されている。  
どの金額をどの粒度から取得しているかを補助表示した方がよい。

## 元RSシステムから参考になる点

元RSシステムは可視化としては強くないが、以下は参考になる。

- 「支出先上位者リスト」として、ブロック単位の表を先に見せている
- ブロック名、合計支出額、支出先の数、事業を行う上での役割が横並びで読める
- A/Bの2ブロック構造が、表だと比較的分かりやすい
- 予算・執行額テーブルで `6,199,412千円` の出どころが見える

## 改善方針

この案件では、サンキーやtreemapを強化するよりも、デフォルト表示を「構造の要約 + 読める内訳」に寄せるのがよい。

### 方針A: 上部に資金ルート要約を出す

画面上部またはSVGキャンバス直上に、次のような要約を出す。

```text
経済産業省 -> JETRO -> J-StarX運営等の再委託先群
```

併せて、以下をチップまたは小さな統計行で表示する。

- 最大2層
- 直接支出ブロック 1件
- 再委託ブロック 1件
- 表示内訳 13件
- 元RS支出先数 360件
- 支出額 61.99億円

### 方針B: Bブロックの内訳は表をデフォルトにする

treemapは面積比較には有効だが、名前や契約概要を読むには弱い。  
この案件の初期表示では、Bブロックの内訳を右側または下部に表で常時表示する。

推奨列:

- 支出先名
- 金額
- 構成比
- 契約概要
- 法人番号
- 国内法人/海外等の区分が取れる場合は区分

上位例:

- 過年度執行額、次年度執行確定額、繰越額: 19.66億円
- その他: 16.28億円
- デロイトトーマツファイナンシャルアドバイザリー合同会社: 5.84億円
- Plug & Play, lLC: 4.68億円
- 株式会社JTB: 3.32億円
- CIC Innovation Communities LLC: 2.43億円

### 方針C: 図は「2段の資金フロー」として簡素化する

この案件では、初期表示の図は3ノードで十分。

```text
経済産業省
  -- 61.99億円 -->
JETRO
  -- 61.99億円 / 再委託先群 -->
J-StarX運営等
```

改善ポイント:

- エッジに金額ラベルを常時表示する
- A/Bが同額である理由を「JETROを経由した再委託先群への支出」として補足する
- ブロック内treemapは「内訳マップ」モードに切り替えたときだけ表示する

### 方針D: 件数ラベルを明確にする

`支出先 13` だけでは元RSの `360` と矛盾して見える。  
表示ラベルを次のように分ける。

- `元RS支出先数: 360`
- `表示内訳: 13`
- `再委託先上位/集約後: 12`

データ上取得できない場合でも、少なくとも `表示内訳` や `集約後` という言葉に変える。

### 方針E: ブロック詳細パネルを「クリック後」から「補助ペイン」へ寄せる

現在はブロックをクリックすると左から詳細パネルが出る。  
この案件のような小規模構造では、初期表示で右ペインにBブロックの内訳を出した方が探索コストが低い。

推奨:

- 右ペイン: 選択ブロック詳細
- 初期選択: 最大金額の再委託ブロック、今回ならB
- Aを選ぶとJETROの直接支出情報に切り替わる
- Bを選ぶと再委託先ランキングに切り替わる

## 実装優先度

### 優先度1: すぐ効く改善

- ヘッダーの `支出先` を `表示内訳` などに変更する
- Bブロックを初期選択し、支出先一覧を常時見せる
- エッジに `61.99億円` ラベルを出す
- ブロックホバーまたは詳細に「A/Bが同額なのは通過フロー」である旨を追加する
- treemap内で読めない小矩形のラベル表示を抑制し、表へ誘導する

### 優先度2: レイアウト改善

- `構造図` と `内訳表` の2ペイン構成にする
- Bブロックのtreemapをタブ化し、初期タブは `リスト`
- ブロック内訳表に構成比バーを追加する
- 長いブロック名は法人列挙より役割を優先して表示する

### 優先度3: データ説明改善

- 元RSの支出先数とmarumieの表示内訳件数の関係を明示する
- `その他`、`過年度執行額、次年度執行確定額、繰越額` を通常の法人支出先とは異なる表示にする
- 国内法人、海外法人、その他/調整額を分類して見せる

## 推奨する最初の実装

最初の実装は、既存のSVG描画を大きく作り替えずに次を行うのがよい。

1. 初期表示で最大金額の再委託ブロックを選択状態にする
2. 左スライドパネルではなく、右側固定ペインに選択ブロックの支出先リストを表示する
3. Bブロックのtreemapは残しつつ、細かい矩形ラベルは減らす
4. ヘッダーに `表示内訳 13件`、可能なら `元RS支出先数 360件` を出す
5. エッジ中央に金額ラベルを表示する

これにより、現行の資産を活かしながら「見えない・読めない」問題をかなり減らせる。
