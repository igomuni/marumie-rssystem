# 間接経費のサンキー非表示問題と、ブロック連結の可視化アイデア

再委託UI刷新 Phase 3 の続き。ユーザーからの2つの問いへの調査・検討メモ。

## 1.【確定】間接経費はサンキー図に「乗らない」

**結論: 国が直接支払う間接経費（支出先ブロックを持たない行）は、メインSankeyにもフローUIにも現れない。**
これはデータ上の事実で、両パイプラインで独立に確認した。

### 根拠（パイプライン2本で一致）

- `scripts/generate-subcontracts.ts:155-167`: `targetBlock` が空の間接経費行は
  `indirectCosts[]` に退避して `continue` する → フロー（支出先）にならない。
- `scripts/generate-sankey-svg-data.ts:343-352`: `支出先名` が空／`支出先ブロック番号` が空の行は
  skip/continue。さらに `indirectOnlyBlocks` として間接専用ブロックを明示的に除外（`indirectRows++`）。

### 実データ検証（PID:1 内閣人事局経費(研修事業)・2025）

| 指標 | 値 |
|------|-----|
| ブロック支出先8社の合計 | 9,353,000円 |
| subcontracts `totalExpense` / `directExpenseTotal` | 9,353,000円 |
| Sankey の project-spending ノード値 | 9,353,000円 |
| 間接経費（講師謝金 1,034,000＋委員等旅費 9,000） | 1,043,000円（上記に**含まれない**） |

間接経費（講師・委員など個人への直接支払い）は `blockHint` が空で、支出先ブロックを持たないため
サンキーの支出額から外れている。

### 規模（2025・全事業集計）

- 間接経費を持つ事業: **1,045 / 5,794 件（約18%）**
- 間接経費の総額: **約329億円**（総支出156.5兆円の約0.02%）
- ただし事業単位では比率が大きいものがある（PID:1 で直接費の約11%）
- カテゴリ: 事務費・職員旅費・委員等旅費・行政経費・謝金 等（＝人件費・庁内経費の性格）

※ 329億円は `targetBlock` を持つ間接経費（フローにも残る分）を含む上限値。
純粋にオフサンキーな額はこれ以下だが、大半は直接支払い分とみられる。

### 意味づけ（ユーザーの直感は正しい）

サンキー図は「事業 → 支出先」の**資金の受け手がいる流れ**しか描けない。
間接経費は受け手（支出先ノード）を持たない支出なので構造的に不可視になる。
**再委託詳細ページの「間接経費」タブは、この“サンキーに乗らない金額”を可視化できる唯一の場所**であり、
現状スコア・フロー中心のUIの中で相対的に軽い扱い（0件時 disabled・最後尾タブ）になっているのは
情報価値と釣り合っていない。重要観点として格上げする余地がある。

### 対応アイデア（未着手・要判断）

- **フロー図に「国が直接支出」レーンを追加**: 事業ノードから、支出先を持たない終端ノード
  「間接経費（講師謝金・旅費 等）」へグレーのリボンを引く。金の全体像が1枚で閉じる。
- **事業ヘッダに内訳を明示**: 「支出 X（うち支出先 Y／間接経費 Z）」。
  現在ヘッダは予算・執行のみで、間接経費が数字として見えない。
- **間接経費タブの位置づけ強化**: 0件でも「なし」を明示（disabledで消さない）、
  タブラベルに金額バッジ。

## 2.【アイデア募集中】ブロック↔支出先↔再委託ブロックの連結可視化

ユーザーの関心: 「ブロックと、ブロック内の支出先、その支出先が作る再委託ブロックの支出先、の
つながりを可視化したい」。まだ確定案なし。方向性の候補:

- **A. フォーカス／トレース**: フロー図でブロックを選ぶと、その部分木（当該ブロック→内包支出先→
  下流の再委託ブロック→その支出先）だけを残して他を減光（/sankey-svg の focus-related と同作法）。
  実装は既存リボンレイアウトへのハイライト追加で比較的軽い。**第一候補**。
- **B. 支出先を中間ノードに昇格**: 現在フロー図の支出先は終端列。再委託する支出先を
  「支出先かつ次段ブロックの入口」として二役で描き、支出先→再委託ブロックの線を明示。
  情報は増えるが列数・交差が増え、サンキーの可読性を損ないやすい。
- **C. パンくず／経路リスト**: 図は触らず、選択支出先について
  「事業 → Aブロック → 当該支出先 →（再委託）→ Bブロック → 終端支出先」の経路をパネルに列挙。
  実装最小・確実だが視覚的インパクトは弱い。

現行フロー図は既に 事業→ブロック(第1層)→再委託ブロック(第2層)→支出先 を描けているので、
不足しているのは「特定ブロック／支出先を起点にした経路の追跡性」。A が費用対効果が高そう。

## 次アクション候補

- 間接経費: まず「事業ヘッダに支出先/間接経費の内訳を明示」＋「間接経費タブを0件でも表示」の
  軽量2点を Phase 3 の締めとして入れる。フロー図レーン追加は別途。
- ブロック連結: A（フォーカス・トレース）のプロトタイプをフロー図に実装して感触を見る。
