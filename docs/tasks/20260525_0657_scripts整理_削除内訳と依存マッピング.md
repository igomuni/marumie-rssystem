## 目的

リリース対象4ページ（`/sankey-svg`・`/subcontracts`・`/mof-budget-overview`・`/quality`）を保守するために必要な `scripts/` のみを残す。**後で「何を、なぜ消したか」を再検証できる**ように、削除候補と判断根拠、残すべきパイプラインの依存マッピングを記録する。

復元: 本作業前のスナップショットは `archive/pre-release-20260525` タグおよび同名ブランチに保全済み。

### 辞書チェーンの扱い（重要）

`score-project-quality.py` は以下4つのCSVを直接読む:

- `public/data/dictionaries/recipient_dictionary.csv`
- `public/data/dictionaries/government_agency_names.csv`
- `public/data/dictionaries/supplementary_valid_names.csv`
- `public/data/dictionaries/opaque_recipient_keywords.csv`

これら22個の辞書 CSV は **すべて Git 管理対象** (`git ls-files public/data/dictionaries/` で22ファイル確認済) であり、runtime は committed CSV のみで完結する。

辞書ジェネレータ群（`build-recipient-dictionary.py`、`generate-*-names.py` 等）は以下の理由で **DELETE 対象** とした:

1. これらは `data/houjin.db`・`data/result/*` 等の**Git管理外の中間成果物**を入力とする。フレッシュクローンから実行できない
2. 辞書 CSV は committed であり、runtime には影響しない
3. 性質上、実験的に作成された一度限りのスクリプト群

万一辞書再生成が必要になった場合は `archive/pre-release-20260525` から復元する。
