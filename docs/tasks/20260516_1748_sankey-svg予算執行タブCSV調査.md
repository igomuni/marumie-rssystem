# /sankey-svg 予算・執行タブ CSV調査と対応方針

作成日: 2026-05-16

関連Issue: feat: `/sankey-svg` 事業の会計区分内訳を表示 #188

## 目的

`/sankey-svg` のサイドパネルに「予算・執行」タブを追加し、選択した事業について予算額・執行額・会計区分内訳を確認できるようにする。

まずはCSVデータの構造を確認し、どのファイルからどの情報を取るべきかを整理する。

---

## 結論

予算・執行タブは `2-1` と `2-2` を併用する。

| 用途 | CSV | 理由 |
|------|-----|------|
| 事業単位の総額、執行額、執行率 | `2-1_RS_{YEAR}_予算・執行_サマリ.csv` | `計(歳出予算現額合計)`、`執行額(合計)`、`執行率` がある |
| 会計区分・会計・勘定・予算種別・項目ごとの予算内訳 | `2-2_RS_{YEAR}_予算・執行_予算種別・歳出予算項目.csv` | `予算額(歳出予算項目ごと)` が会計区分付きで入っている |

重要な制約として、`2-2` には執行額の内訳がない。
そのため「会計区分ごとの執行額」は、このCSV群からは直接表示できない。

初期実装では以下を表示するのが妥当。

- 上部: `2-1` 由来の総額
  - 当初予算
  - 補正予算
  - 前年度からの繰越し
  - 予備費等
  - 計(歳出予算現額合計)
  - 執行額(合計)
  - 執行率
  - 翌年度への繰越し
  - 翌年度要求額
- 下部: `2-2` 由来の予算内訳
  - 会計区分
  - 会計
  - 勘定
  - 予算種別
  - 所管
  - 組織・勘定
  - 項
  - 目
  - 予算額(歳出予算項目ごと)
  - 翌年度要求額(歳出予算項目ごと)

## 年度の扱い

`data/year_2024` や `data/year_2025` の `year` は、RSシステムの事業年度・公開データ年度に近い。
各CSVの中には複数の `予算年度` が含まれる。

`/sankey-svg` の主表示に使うべき予算年度は、既存実装どおり `YEAR - 1`。

| 表示データ | 主に使う予算年度 | 理由 |
|-----------|----------------|------|
| `sankey-svg-2024` | 2023 | 執行額が入り、支出先データと対応する |
| `sankey-svg-2025` | 2024 | 執行額が入り、支出先データと対応する |

`YEAR` と同じ `予算年度` の行も存在するが、これは予算額や翌年度要求額中心で、執行額は基本的に0。
支出先までつながる完了データとしては `YEAR - 1` を使う。

既存の `scripts/generate-sankey-svg-data.ts` も以下の方針になっている。

```ts
const TARGET_BUDGET_YEAR = YEAR - 1;
```

### 年度間のつながり

同一事業の推移は基本的に `予算事業ID` で追える。
ただし、事業の新設・終了・統廃合により、年度ごとにPID集合は変わる。

`翌年度要求額` は翌年度予算への要求額であり、翌年度の実際の `計(歳出予算現額合計)` と一致するものではない。
補正予算や繰越しがあるため、年度をまたぐ比較では「要求額」と「翌年の予算現額」は別物として扱う。

サイドパネルでは、初期表示は選択中の表示年度に対応する `YEAR - 1` に絞る。
将来的に年度推移を見る場合は、同じ `予算事業ID` の過年度行を折りたたみセクションで表示するのがよい。

## `2-1` の構造

`2-1_RS_{YEAR}_予算・執行_サマリ.csv` は、事業・予算年度ごとの総額を持つ。

主な列:

| 列 | 内容 |
|----|------|
| `予算年度` | 予算の年度 |
| `当初予算(合計)` | 事業単位の当初予算合計 |
| `補正予算(合計)` | 補正予算合計 |
| `前年度からの繰越し(合計)` | 前年度からの繰越し |
| `予備費等(合計)` | 予備費等 |
| `計(歳出予算現額合計)` | 予算現額の合計 |
| `執行額(合計)` | 執行額の合計 |
| `執行率` | 執行率 |
| `翌年度への繰越し(合計)` | 翌年度への繰越し |
| `翌年度要求額(合計)` | 翌年度要求額 |
| `会計区分` | `一般会計` / `特別会計` / 空 |
| `会計` | 会計名 |
| `勘定` | 勘定名 |

### 注意点

`2-1` では、金額が入っている行の `会計区分 / 会計 / 勘定` は空である。
会計区分行は別に存在するが、そこには金額が入っていない。

例: PID 3522、2025データの予算年度2024

| 会計区分 | 会計 | 勘定 | 予算額 | 執行額 |
|---------|------|------|--------|--------|
| 空 | 空 | 空 | 990,178,535,000 | 990,178,535,000 |
| 特別会計 | エネルギー対策 | エネルギー需給勘定 | 空 | 空 |
| 一般会計 | 一般会計 | 空 | 空 | 空 |

そのため、`2-1` だけから「会計区分ごとの金額内訳」を出すことはできない。
既存の `accountCategory` は事業がどの会計区分を持つかの分類には使えるが、金額配賦には使えない。

## `2-2` の構造

`2-2_RS_{YEAR}_予算・執行_予算種別・歳出予算項目.csv` は、予算内訳を持つ。

主な列:

| 列 | 内容 |
|----|------|
| `予算年度` | 予算の年度 |
| `会計区分` | 一般会計 / 特別会計 |
| `会計` | 会計名 |
| `勘定` | 勘定名 |
| `予算種別` | 当初予算 / 補正予算 / 前年度から繰越し / 予備費等など |
| `所管` | 所管 |
| `組織・勘定` | 組織または勘定 |
| `項` | 歳出予算の項 |
| `目` | 歳出予算の目 |
| `歳出予算項目の補足情報` | 補足 |
| `予算額(歳出予算項目ごと)` | 項目ごとの予算額 |
| `翌年度要求額(歳出予算項目ごと)` | 項目ごとの翌年度要求額 |

`2-2` の `予算額(歳出予算項目ごと)` を `予算年度 = YEAR - 1` で合計すると、`2-1` の `計(歳出予算現額合計)` と一致する。

## 事例

### PID 3522

事業名: ポスト5G情報通信システム基盤強化研究開発事業(AI基盤モデル及び先端半導体関連技術開発事業)

2025データ、予算年度2024では、`2-1` の総額は 990,178,535,000円。

`2-2` では以下の内訳に分かれる。

| 会計区分 | 会計 | 勘定 | 予算種別 | 項 | 目 | 予算額 |
|---------|------|------|----------|----|----|--------|
| 一般会計 | 一般会計 | 空 | 第1次補正予算 | 空 | 空 | 832,562,535,000 |
| 特別会計 | エネルギー対策 | エネルギー需給勘定 | 第1次補正予算 | 脱炭素成長型経済構造移行推進対策費 | 脱炭素成長型経済構造移行推進対策費補助金 | 157,616,000,000 |

この事例は、`2-1` の会計区分メタデータだけでは内訳を正しく説明できないが、`2-2` を使うと金額付きで説明できる。

### PID 7152

事業名: 起業家等の海外派遣・シリコンバレー拠点形成事業

2024データ、予算年度2023では、`2-1` の総額は 6,199,412,000円。

`2-2` では以下の1行。

| 会計区分 | 会計 | 勘定 | 予算種別 | 予算額 |
|---------|------|------|----------|--------|
| 一般会計 | 一般会計 | 空 | 第1次補正予算 | 6,199,412,000 |

## 対応方針

### 1. グラフデータ生成に `2-2` を追加で読み込む

対象: `scripts/generate-sankey-svg-data.ts`

現在は `1-1`, `2-1`, `5-1`, `5-2` を読み込んでいる。
ここに `2-2_RS_{YEAR}_予算・執行_予算種別・歳出予算項目.csv` を追加する。

読み込み後、`予算年度 === TARGET_BUDGET_YEAR` の行だけを対象に、`予算事業ID` ごとに内訳配列を作る。

```ts
type BudgetBreakdownItem = {
  fiscalYear: number;
  accountCategory: string;
  account: string;
  subAccount: string;
  budgetType: string;
  jurisdiction: string;
  organizationAccount: string;
  item: string;
  subItem: string;
  note: string;
  amount: number;
  nextYearRequestAmount: number;
};
```

### 2. `project-budget` ノードへ予算・執行詳細を付与する

対象: `types/sankey-svg.ts`, `scripts/generate-sankey-svg-data.ts`

候補:

```ts
budgetSummary?: {
  fiscalYear: number;
  initialBudget: number;
  supplementaryBudget: number;
  carryoverBudget: number;
  reserveFund: number;
  totalBudget: number;
  executedAmount: number;
  executionRate: number | null;
  carryoverToNext: number;
  nextYearRequest: number;
};
budgetBreakdown?: BudgetBreakdownItem[];
```

`budgetSummary` は `2-1` の `TARGET_BUDGET_YEAR` 行を集計して作る。
`budgetBreakdown` は `2-2` の `TARGET_BUDGET_YEAR` 行から作る。

`project-spending` ノードには付与しない。
サイドパネル側では `project-spending` 選択時も同じ `projectId` の `project-budget` ノードを参照する。

### 3. サイドパネルに「予算・執行」タブを追加する

対象: `app/sankey-svg/page.tsx`

既存のサイドパネルは省庁・事業・支出先の3タブ構成。
これに `budget` タブを追加する。

タブ候補:

- 省庁
- 事業
- 支出先
- 予算・執行

自動タブ選択は、事業ノード選択時に「予算・執行」へ直接飛ばすか、従来どおり支出先タブを維持するかはUI確認が必要。
初期実装では従来挙動を保ち、ユーザーが明示的に「予算・執行」を選べる形が安全。

### 4. 予算・執行タブの表示内容

事業ノード選択時:

- ヘッダー
  - 事業名
  - PID
  - 予算年度
  - 会計区分バッジ
- 総額カードまたはコンパクト表
  - 予算現額
  - 執行額
  - 執行率
  - 当初予算
  - 補正予算
  - 繰越し
  - 予備費等
  - 翌年度要求額
- 会計区分別サマリ
  - `budgetBreakdown` を `accountCategory` で集計
  - 一般会計・特別会計ごとの予算額
- 内訳テーブル
  - 会計区分
  - 会計
  - 勘定
  - 予算種別
  - 項
  - 目
  - 予算額
  - 翌年度要求額

省庁・支出先・集約ノード選択時:

- 初期実装では、紐づく事業一覧から各事業を選ぶ導線に留める。
- 省庁単位や支出先単位の予算内訳集計は、後続対応でよい。

### 5. データサイズ対策

`2-2` の行数は多い。

- 2024: 55,609行
- 2025: 70,915行

ただし `/sankey-svg` グラフJSONには既に全事業ノードが入っているため、事業ノードに内訳配列を持たせるとJSONサイズが増える。
初期実装では以下のどちらかを選ぶ。

#### 案A: graph.json に同梱

実装が簡単。
サイドパネル表示が即時。

欠点は `sankey-svg-{YEAR}-graph.json.gz` が肥大化すること。

#### 案B: 別ファイル化

`public/data/sankey-svg-{YEAR}-budget-details.json.gz` のように分け、事業選択時に読み込む。

初期ロードが軽い。
実装は少し増える。

サイドパネルの詳細データなので、長期的には案Bが望ましい。
ただしIssue #188の最初の実装では、スコープを抑えるなら案Aでもよい。

## 検証観点

- `2-1` の `計(歳出予算現額合計)` と `2-2` の `予算額(歳出予算項目ごと)` の事業別合計が一致すること
- 事業ノード選択時に `YEAR - 1` の予算年度だけが表示されること
- `project-spending` ノード選択時も同じPIDの予算・執行タブが表示できること
- PID 3522 のような一般会計・特別会計混在事業で、会計区分別の金額が表示されること
- PID 7152 のような単一一般会計事業で、内訳が1行でも自然に表示されること
- 2025データの予算年度2025のような未執行年度を誤って主表示しないこと
- グラフJSONまたは詳細JSONのサイズ増加が許容範囲に収まること

---

## 残課題

- 会計区分ごとの執行額はCSV上直接存在しないため、表示しないか、「執行額は事業合計のみ」と明示する。
- 省庁単位・支出先単位・集約ノード単位の予算内訳集計は後続対応とする。
- 年度推移表示を入れる場合、同一PIDの過年度行をどこまで見せるかを別途設計する。
- `accountCategory` フィルタの `both` 分類は現在 `2-1` の会計区分メタデータを使っている。金額付きの分類に寄せるなら `2-2` ベースへの見直し余地がある。
