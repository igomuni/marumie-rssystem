# 事業ノードの統合表示 設計

## 目的

ユーザーが事業の「予算」と「支出」の差を一目で把握できるようにするため。  
現状は `project-budget`（予算）と `project-spending`（支出）が別々のノードとして列に渡って表示されており、両者の関係が視覚的に分かりにくい。

---

## 現状

### 列構成（現行）

| Col | type | 役割 |
|-----|------|------|
| 0 | `total` | 総計 |
| 1 | `ministry` | 府省庁 |
| 2 | `project-budget` | 事業（予算） |
| 3 | `project-spending` | 事業（支出） |
| 4 | `recipient` | 支出先 |

- `project-budget → project-spending` エッジが列2→3の内部リボンとして存在する
- 両ノードは `NODE_W = 18px` の同幅矩形
- `project-budget` の高さは予算額（`scaleBudgetToVisible` 適用後）
- `project-spending` の高さは支出額（オフセット分を除いた可視支出）

---

## 表示仕様（統合後）

### 列構成（統合後）

| Col | type | 役割 |
|-----|------|------|
| 0 | `total` | 総計 |
| 1 | `ministry` | 府省庁 |
| 2 | `project` | 事業（統合ノード） |
| 3 | `recipient` | 支出先 |

### 統合ノードの形状

```
左辺高さ = 予算高さ (H_budget)
右辺高さ = 支出高さ (H_spending)
ノード幅 = NODE_W × 2
```

- 左辺（省庁側）: 高さ H_budget の垂直線
- 右辺（支出先側）: 高さ H_spending の垂直線
- 上辺: (x0, y0_budget_top) → (x1, y0_spending_top) を結ぶ3次ベジェ曲線
- 下辺: (x1, y1_spending_bottom) → (x0, y1_budget_bottom) を結ぶ3次ベジェ曲線

整列方針: **上端揃え**（y0_budget_top = y0_spending_top = ノードの y0）。  
高さの差（予算 > 支出が典型）は下辺のカーブで吸収する。

```
(x0) ─────────────── (x1)
|   ↑ H_budget        | ↑ H_spending
|                     |
(x0) ～～～～～～～～～ (x1)   ← 下辺はベジェ
```

### ラベル配置

| 位置 | 内容 |
|------|------|
| 左側（省庁寄り） | 予算額（`formatYen`） |
| 右側（支出先寄り） | 事業名 + 支出額 |

現行の `showLabels` トグルは引き続き有効。

### リボン接続

- 省庁 → 事業: `x0`（左辺）に接続、縦方向を H_budget 内で分配
- 事業 → 支出先: `x1`（右辺）から出発、縦方向を H_spending 内で分配

---

## 実装方針

### 採用アプローチ: 隣接配置 + 描画オーバーライド

データモデル（`filterTopN` の出力）は基本維持し、`computeLayout` とSVG描画層を変更する。

**理由**: `project-budget / project-spending` の二ノード構造は `filterTopN` 内の複雑なスケーリング・集計ロジックと密結合している。型とエッジ構造を変えると変更スコープが大きくなるため、レイアウト・描画層の変更で完結させる。

### 変更ファイル一覧

#### `app/lib/sankey-svg-constants.ts`

- `COL_MAP` の `'project-spending'` を `2` → `2`（変更なし。`project-budget` も `2` に変更）
  - **具体的には**: `'project-budget': 2, 'project-spending': 2` とし、両者を同列に収める
- `NODE_W` はそのまま（統合ノードの幅 = `NODE_W * 2` は描画時に計算）
- ラベル用の `COL_LABELS` から `'事業(支出)'` を削除し `'事業'` に統合

#### `app/lib/sankey-svg-filter.ts`

- 変更なし（`filterTopN` 出力は現行どおり）

#### `app/sankey-svg/page.tsx`（描画層）

**1. `computeLayout` の変更**

`project-budget` と `project-spending` は同じ列（col 2）に収める。ただし描画では左右に分けるため、`LayoutNode` に以下のフィールドを追加する：

```typescript
pairedNodeId?: string   // budget → spending, spending → budget の対応ID
pairSide?: 'left' | 'right'
```

配置ルール:
- `project-budget` ノード: `x0 = 2 * colSpacing`, `x1 = x0 + NODE_W`（左半分）
- `project-spending` ノード: `x0 = x0_budget + NODE_W`, `x1 = x0 + NODE_W`（右半分 = 同じ `x0` オフセット + NODE_W）
- 列間隔は col 4（`recipient`）が col 3 扱いになるよう `maxCol = 3` で計算する

Y 座標は既存ロジックどおり（列ごとに独立して計算）。  
ただし `project-budget` と `project-spending` の y0 を上端揃えで同期させる後処理を追加する:

```text
各 project-budget n に対して:
  paired = project-spending ノード（同一 projectId）
  paired.y0 = n.y0  // 上端を揃える
```

**2. 内部リボンのスキップ**

`project-budget → project-spending` エッジは `computeLayout` 内で link として保持するが（高さ計算に必要）、**SVG描画時にフィルタして非表示**にする。

```typescript
// 描画対象外エッジ
const isInternalProjectEdge = (link: LayoutLink) =>
  link.source.type === 'project-budget' && link.target.type === 'project-spending';
```

**3. 統合ノードの SVG パス生成**

新規ヘルパー関数 `mergedProjectPath(budget: LayoutNode, spending: LayoutNode): string` を追加:

- budget.x1 = spending.x0 が前提（隣接）
- 上辺ベジェ: control point は x 方向中央、y は budget.y0 / spending.y0 それぞれ
- 下辺ベジェ: control point は x 方向中央、y は budget.y1 / spending.y1 それぞれ

SVG描画ループで `project-budget` ノードを検出した際に、対応する `project-spending` ノードと合わせて `mergedProjectPath` で描画。`project-spending` 単体のノード描画はスキップ。

**4. ラベル変更**

各統合ノードのラベル:
- 右側（`project-spending` の x1 付近）: 事業名 + 支出額（現行と同位置）
- 左側（`project-budget` の x0 付近）: 予算額のみ

## 対象外ステートの影響確認

| 機能 | 影響 | 対処 |
|------|------|------|
| `pinnedProjectId` | `project-spending-{n}` IDを使用 | 変更なし（ノードIDは保持）|
| `selectedNodeId` | budgetまたはspendingのID | 統合ノードクリックはspending側のIDで処理 |
| `computeFocusPins` | `nodeId.startsWith('project-spending-')` | 変更なし |
| サイドパネル | budgetとspendingを個別表示 | 統合ノード用の表示に変更が必要 |
| `focusRelated` / フォーカスモード | `project-spending` typeで判定 | 変更なし |
| URL params `pp` | `project-spending-{n}` | 変更なし |
| `scaleBudgetToVisible` | budgetノードのvalue調整 | filterTopN内で完結、変更なし |
