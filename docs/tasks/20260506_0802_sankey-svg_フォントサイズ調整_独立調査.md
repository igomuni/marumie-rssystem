# sankey-svg フォントサイズ調整 独立調査

## 前提

- 既存の設計案は参照せず、現在の実装と Playwright の実画面確認だけで調査した。
- 対象画面: `/sankey-svg?year=2025`
- 参考にする方向性: Google Maps のように、図本体のラベルは控えめだが視認でき、操作UIは少し大きく、補助情報は小さくする。

- 1024 x 768: `/sankey-svg?year=2025`

結果:

- 1440px幅では現状でも読めるが、事業・支出先が密な箇所はやや小さく、Google Maps 的な地図ラベルとしては 12px 程度まで上げる余地がある。
- 1024px幅でも図本体は成立しているが、上部の検索・年度・TopN設定が横一列に密集している。
- 390px幅では検索ボックス、年度選択、TopN設定、列見出しが重なり、フォントサイズ調整以前に上部UIのレスポンシブ整理が必要。

## Google Maps のPlaywright確認

確認URL:

- `https://www.google.com/maps`

確認条件:

- 1440 x 900
- 日本語UI
- 初期表示された地図画面を対象に、DOM上の操作UIの `computed font-size` を取得した。

制約:

- Google Maps の地図本体ラベルは、DOMテキストではなくタイル画像/Canvas/内部描画に近い形で表示されるため、PlaywrightのDOM計測では正確なフォントサイズを取得できない。
- そのため、地図本体ラベルはスクリーンショット上の相対比較、検索欄・チップ・サイドカード・ボタン類はDOM計測、という分け方で見る。

取得できた代表値:

| 対象 | Google Maps 実測/観察 |
| --- | ---: |
| 検索入力プレースホルダ | 14px |
| 検索・ルートなどのアイコン | 24px |
| 交通状況カードの主要テキスト | 16px |
| 交通状況カードの補助テキスト | 12px |
| 上部カテゴリチップ | 14〜16px相当 |
| 小さな補助UI | 10〜12px |
| 地図本体ラベル | DOM計測不可、視覚上は小さくてもコントラストと背景処理で読ませている |

観察:

- 操作UIは思ったより大きい。検索入力は 14px、カードの主要テキストは 16px、アイコンは 24px で、タッチ/クリックしやすいサイズになっている。
- 一方で地図本体ラベルは控えめ。地図上に多数のラベルが出るため、画面上の文字を大きくするより、色・太さ・背景との分離で読ませている。
- 上部カテゴリチップは横スクロール的に並ぶため、狭幅で全要素を1画面に押し込まない設計になっている。
- 左カードは読み物として成立するサイズ感で、図本体ラベルより明確に大きい。

`sankey-svg` への示唆:

- 図本体ラベルは 12px程度で抑え、密度を守る。
- 検索入力、主要操作、ツールチップ、サイドパネルは 12〜14px中心に寄せる。
- 読ませる場所と俯瞰する場所でフォントスケールを分ける。
- 狭幅では上部UIを押し込まず、チップ/設定/検索を再配置する。これは Google Maps 参考でも、単純な縮小ではなくレイアウト切替が自然。

### 具体例: Google Maps の検索結果/サイドパネル

確認シナリオ:

- Google Maps を開く
- 検索欄に `東京駅` を入力
- Enterで検索
- 検索結果リスト/左サイドパネルを確認

スクリーンショット:

- `/private/tmp/google-maps-search-tokyo-station.png`

代表値:

| 対象 | Google Maps 実測/観察 |
| --- | ---: |
| 左ナビ幅 | 約72px |
| 検索結果パネル幅 | 約408px |
| 左ナビ込みの占有幅 | 約480px |
| 検索結果1行の高さ | 約100px |
| 検索結果の地点名 | 16px / weight 500 |
| 種別・住所などのメタ情報 | 14px / line-height 20px |
| アクションボタンのアイコン | 18〜24px |
| アクションボタンのラベル | 12px / weight 500 |
| 左ナビのラベル | 12px / weight 500 |

観察:

- 検索結果リストは、地図本体よりかなり読みやすさ寄りのスケールになっている。
- 地点名は 16px、メタ情報は 14pxで、行高も 20px程度あり、1件あたり約100pxを使う。
- アクションはアイコンを大きく、ラベルを小さめにし、ボタン領域は広く取っている。
- 左ナビは 12pxラベルでも、アイコンと縦方向の余白で認識しやすくしている。

`sankey-svg` への示唆:

- サイドパネルの「選択中ノード名」は現在13pxだが、検索結果や詳細の主タイトルとしては 14〜16px も候補になる。
- 一覧行は現在かなり高密度なので、読みやすさ重視のモードでは「名前 14px、金額 12〜13px、補助 12px」程度まで上げてもよい。
- ただし Google Maps の検索結果は1件100px程度を使うため、`sankey-svg` の関連ノード一覧へそのまま適用すると一覧性が落ちる。通常モードでは現状寄り、閲覧/プレゼンモードではゆったり、のように表示密度プリセットを持つのが良い。
- アクションボタンは、文字を大きくするよりアイコン24px級・ヒットエリア40px以上に寄せるほうが Google Maps 的。

## 現状実装の観察

### 図本体ラベル

- `app/sankey-svg/page.tsx` のノードラベルは `fontSize={11 / zoom}` で統一されている。
- ズームしても画面上のラベルサイズを一定にする思想は良い。Google Maps 風の「地図を拡大してもラベルが巨大化しない」挙動に近い。
- ただし 11px 固定は少し細く、特に日本語の長い事業名と金額が並ぶ箇所で読みにくい。
- ラベル表示判定が `maxH * zoom > 10` / `(h + NODE_PAD) * zoom > 10` に固定されているため、フォントを上げる場合は表示閾値も合わせて変更しないと重なりやすい。
- 小さいノードを押し広げる `LABEL_SLOT = 12` も現在の 11px 前提。12pxへ上げるなら 14px 程度に広げたほうが安定する。

### 列見出し

- 列見出しは DOM overlay で、見出し 13px、金額 11px。
- 見出し位置は `SEARCH_BOX_RESERVE = 56` を基準に避けている。
- 390px幅では上部UI自体が重なるため、列見出しも巻き込まれて視認性が落ちる。
- 見出しのフォントだけを上げるより、見出しは 12px程度に抑え、金額は 11px、狭幅時は金額行を省略してホバー/クリックへ逃がすのが良さそう。

### サイドパネル

- パネル幅は 310px。
- タイトル 13px、主金額 15px、本文・リスト 11〜12px、補助ラベル 10〜11px。
- 現状の情報密度は悪くないが、10pxの補助ラベルが多く、少し弱い。
- リスト行の名前 12px、金額 11pxは、Google Maps の地点リスト相当としては許容範囲。ただし金額を読み取り対象にするなら金額も 12pxへ上げたい。
- Google Maps の検索結果例では地点名16px、メタ情報14px、行高20px、1件約100pxだった。`sankey-svg` で同じ密度にすると一覧性が落ちるため、通常時は高密度を維持しつつ、選択中タイトルや検索結果の主語だけを少し大きくするのが現実的。

### ツールチップ

- リンクツールチップ: タイトル 11px、金額 10px、円表記 9px。
- ノードツールチップ: タイトル 11px、金額 10px、円表記 9px。
- 9pxはかなり小さい。ツールチップは「読むために出すUI」なので、タイトル 12px、金額 11px、円表記 10px程度が良い。
- ただし高さが `tipH = 58` や `(both ? 78 : 68) + ...` のように固定計算されているため、フォントを上げる場合は高さ計算も見直す必要がある。

### 上部操作UI

- 検索入力と年度選択は 13px。
- フィルタターゲット、会計/省庁フィルタ、金額入力、TopN補助表示などは 10〜11pxが多い。
- デスクトップでは許容範囲だが、390px幅では検索、年度、TopN設定が重なっている。
- フォントサイズを上げるなら、上部UIの幅・配置を先に整理しないと崩れが増える。

## 推奨フォントスケール

まずは大きくしすぎず、1段だけ上げる。

| 用途 | 現状 | 推奨 |
| --- | ---: | ---: |
| SVGノードラベル | 11px相当 | 12px |
| SVG補助`tspan` | 11px相当 | 11px相当 |
| 列見出し | 13px | 12px |
| 列見出しの金額 | 11px | 11px |
| 検索入力 | 13px | 14px |
| 年度 | 13px | 13px維持 |
| 上部フィルタの通常テキスト | 10〜11px | 12px |
| 上部フィルタの補助テキスト | 10px | 11px |
| サイドパネルタイトル | 13px | 13px維持 |
| サイドパネル主金額 | 15px | 15px維持 |
| サイドパネル検索/選択タイトル | 13px | 14〜16px |
| サイドパネルリスト名 | 12px | 12〜14px |
| サイドパネルリスト金額 | 11px | 12〜13px |
| サイドパネル補助情報 | 10〜11px | 12px |
| ツールチップタイトル | 11px | 12px |
| ツールチップ金額 | 10px | 11px |
| ツールチップ円表記 | 9px | 10px |
| ズーム率 | 10px | 10〜11px |

方針:

- 図本体は 12pxを上限にする。13px以上にすると、密な事業列・支出先列でラベル密度がかなり苦しくなる。
- 操作UIは 12〜14pxに寄せる。Google Maps の検索入力が 14pxであることを踏まえ、検索は 14pxまで上げてよい。
- 10pxは原則、ズーム率や補助的な記号表示だけに限定する。
- 金額は読み取り対象なので、サイドパネルとツールチップでは 11px以下にしすぎない。

## 実装案

### 1. フォント定数を導入する

`app/sankey-svg/page.tsx` 内に散らばる数値を段階的に定数化する。

候補:

```ts
const MAP_LABEL_FONT_PX = 12;
const MAP_LABEL_SLOT_PX = 14;
const MAP_LABEL_VISIBLE_MIN_H_PX = 11;

const COLUMN_LABEL_FONT_PX = 12;
const COLUMN_AMOUNT_FONT_PX = 11;

const SEARCH_FONT_PX = 14;
const CONTROL_FONT_PX = 13;
const CONTROL_SMALL_FONT_PX = 12;
const META_FONT_PX = 11;

const PANEL_TITLE_FONT_PX = 14;
const PANEL_PRIMARY_VALUE_FONT_PX = 15;
const PANEL_LIST_NAME_FONT_PX = 12;
const PANEL_LIST_VALUE_FONT_PX = 12;
const PANEL_META_FONT_PX = 12;

const TOOLTIP_TITLE_FONT_PX = 12;
const TOOLTIP_VALUE_FONT_PX = 11;
const TOOLTIP_META_FONT_PX = 10;
```

まずは画面全体の一括テーマ化まではしない。現在は単一ファイル内の inline style が多いため、定数化だけでも影響範囲を追いやすくなる。

### 2. SVGラベルを 12px 相当に変更する

変更対象:

- `fontSize={11 / zoom}` を `fontSize={MAP_LABEL_FONT_PX / zoom}` に変更。
- `labelFontPx = 11` を `MAP_LABEL_FONT_PX` に変更。
- `LABEL_SLOT = 12` を `MAP_LABEL_SLOT_PX` に変更。
- ラベル表示判定の `> 10` を `> MAP_LABEL_VISIBLE_MIN_H_PX` に変更。

注意:

- 表示閾値を上げすぎるとラベルが減りすぎる。
- まず `MAP_LABEL_VISIBLE_MIN_H_PX = 11`、`MAP_LABEL_SLOT_PX = 14` で確認するのが良い。

### 3. ツールチップはフォントと高さを同時に調整する

変更対象:

- リンクツールチップの `tipH = 58`
- ノードツールチップの `tipH = (both ? 78 : 68) + ...`
- 各 `fontSize: 9/10/11`

提案:

- タイトル 12px、主要金額 11px、円表記 10pxへ変更。
- 固定高さはそれぞれ 6〜10px程度増やす。
- 可能なら `tipH` は実DOM高さで測る構造にする。ただし今回は変更量が大きくなるため、最初は余裕を足すワークアラウンドで良い。

### 4. 上部UIはレスポンシブ整理を先に入れる

390px幅では現状でも以下が重なっている。

- 検索ボックス
- 年度セレクト
- TopN設定パネル
- 列見出し

フォント調整のPR内で最低限やるなら:

- `SEARCH_BOX_RESERVE` を固定 56pxではなく、上部UIの実測高さまたは viewport 幅で切り替える。
- 720px未満では TopN設定を初期折りたたみ、右上の設定ボタンから開く方式に寄せる。
- 720px未満では列見出しの金額行を非表示にし、見出しのみ表示する。
- Google Maps のカテゴリチップのように、狭幅では全操作を横一列に押し込まず、主要操作だけを残して残りはメニュー/シートへ逃がす。

この対応なしにフォントだけ上げると、狭幅でさらに崩れる。

### 5. サイドパネルは小さな改善に留める

今回の主対象は `sankey-svg` 全体のフォント調整だが、サイドパネルは情報密度が高いため大きく変えない。

変更候補:

- リスト金額を 11px -> 12px。
- 補助ラベルの 10pxを 12pxへ上げる。
- 選択中ノード名は 13px維持でも成立するが、詳細パネルの主タイトルとしては 14pxへ上げる候補がある。
- 検索結果や関連ノード一覧を Google Maps 寄りにする場合は、名前 14px、金額 12〜13px、行高 28〜36px程度の「読みやすさ優先モード」を別に持つ。
- パネル幅 310pxは一旦維持。
- もし金額の折り返しや詰まりが出る場合だけ 320〜360pxへ広げる。

## 優先順位

1. フォント定数化
2. SVGノードラベル 12px化と `LABEL_SLOT` / 表示閾値の連動
3. 検索入力の 14px化と上部UIの余白確認
4. ツールチップの 12/11/10px 化と高さ調整
5. サイドパネルの金額・補助ラベル微調整
6. 狭幅時の上部UI整理

ただし、狭幅対応まで同じPRに入れると差分が大きい。実装PRとしては、デスクトップ/タブレットのフォント改善を主目的にし、狭幅の上部UI再配置は別Issue化しても良い。

## 検証観点

Playwrightで最低限以下を確認する。

- 1440 x 900 `/sankey-svg?year=2025`
  - 事業列、支出先列のラベルが重なりすぎないこと。
  - 列見出しと検索/TopN UIが重ならないこと。
- 1440 x 900 選択状態
  - サイドパネルの金額、タブ、リスト金額が欠けないこと。
  - 図側の選択状態ラベルが読めること。
- 1024 x 768
  - 上部UIと列見出しが衝突しないこと。
  - 支出先列の長い名前が過度に読みづらくならないこと。
- 390 x 844
  - 現状の重なりが残る場合は既知課題として明記する。
  - 同一PRで対応するなら、TopN初期折りたたみと列見出し簡略化まで確認する。

## 結論

`sankey-svg` の図本体ラベルは、現在の「ズームに対して画面上の文字サイズを固定する」方針を維持したまま、11px相当から12px相当へ上げるのが良い。Google Maps を参考にするなら、地図本体の文字を大きくしすぎず、操作UI・ツールチップ・サイドパネル側を 12〜14px中心に寄せるのが自然。特に検索入力は Google Maps の実測に近づけて 14pxを候補にしてよい。

一方で、狭幅ではフォントサイズ以前に上部UIが重なっている。フォント調整PRではデスクトップ/タブレットを主対象にし、モバイル/狭幅の上部UI再配置は別タスクとして切り出す判断が現実的。
