# 国の機関 L2 局種別ラベリング設計

## 目的

データ分析者が支出先一覧を見たとき、`前橋地方法務局` や `中部地方整備局` が `府省庁` と同じ L2 に分類されていては機関の性質が判断できない。これらを `法務局`、`地方整備局` 等の局種別 L2 に振り分けることで、地方支分部局（=国の出先機関）と中央省庁の本省・内部部局を区別できるようにする。

## 変更方針

### 1. field_office_names.csv に `l2` 列を追加

現在の `field_office_names.csv` は `name, match_type` の2列スキーマで、DICT_LABEL で全エントリに一律 `('国の機関', '行政機関')` が付与されている。

このファイルを拡張して `name, match_type, l2` の3列スキーマにする。
省庁 CSV (`ministry_from_ichiran.csv` 等) より先に読まれること（`f` < `m`）を活かし、
法務局・整備局等の正確な L2 で **辞書段階でオーバーライド** できるようにする。

**load_dict_labels のロジック変更**（generate-entity-labels-csv.py / generate-entity-labels.py）:
- `field_office_names.csv` 読み込み時、行に `l2` 列が存在すれば DICT_LABEL の L2 より行の値を優先する

### 2. field_office_names.csv の内容拡張

労働局エントリの L2 を `行政機関` → `労働局` に変更（列追加で表現）。

### 3. KAKU_PATTERNS への regex フォールバック追加

辞書に収録されていない局種エントリを KAKU_PATTERNS で補足する。
（辞書より後に実行されるため、辞書収録済みエントリには影響しない）

追加パターン（案）：

| パターン | L2 |
|---------|----|
| `^.+法務局(\w+局)?$` | `法務局` |
| `^.+地方整備局` | `地方整備局` |
| `^.+農政局$` | `農政局` |
| `^.+[運]輸局$` | `運輸局` |
| `^.+経済産業局$` | `経済産業局` |
| `^.+管区警察局` | `管区警察局` |
| `^.+厚生局` | `厚生局` |
| `^.+公安調査局$` | `公安調査局` |
| `^.+防衛局$\|^.+防衛支局$` | `防衛局` |
| `^.+森林管理局$` | `森林管理局` |
| `^.+出入国在留管理局$` | `出入国在留管理局` |
| `^.+総合通信局$` | `総合通信局` |
| `^.+航空局$` | `航空局` |
| `^.+国税局$` | `国税局` |
| `^.+開発局$` | `開発局` |

### 4. 未ラベリングエントリへの対応

KAKU_PATTERNS または aggregate_names.csv への追加で対応。

| 名称 | 対応方法 |
|------|---------|
| `経済産業局` | KAKU regex `^経済産業局$` → `経済産業局` |
| `地方経済産業局` | KAKU regex → `経済産業局` |
| `その他(37局)` | aggregate_names.csv に exact エントリ追加 → `行政機関集合` |
| `各経済産業局及び沖縄総合事務局` 等（集合表現） | aggregate_names.csv に exact または regex 追加 → `行政機関集合` |
| `財務省北海道財務局ほか` | KAKU regex → `地方出先機関` |
| `総務省東北総合通信局` | KAKU regex（既存の `総合通信局$` パターンで捕捉されるはず） |
| `地方運輸局` | KAKU regex → `運輸局` |
