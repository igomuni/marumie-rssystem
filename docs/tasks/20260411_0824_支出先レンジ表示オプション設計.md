# 支出先の集約ノード表示オプション

## 目的

ユーザーが「支出先ランキングの特定区間（TopN + オフセット）」を指定したとき、
ウィンドウ外の支出先を集約した `その他の支出先` ノードを非表示にし、
**ウィンドウ内支出先と直接接続がある事業のみ** に絞り込んで Sankey 図を確認できるようにする。

---

## 背景・検討経緯

当初「rangeStart / rangeEnd のデュアルスライダー」による支出先レンジ指定を検討したが、
総支出先数 12,413 件に対して TopN=100 でもスライダー幅の 0.8% 程度にしかならず、
2つの Thumb が重なって操作不能になるため、スライダー UI は断念した。

現行のオフセット + TopN スライダーが実質「レンジ」と同義であるため、
UI は既存のままとし、**集約ノードを表示するかどうかのチェックボックスのみ追加** する方針に切り替えた。

---

## 実装内容

### UI

設定パネルに「支出先の集約ノードを表示」チェックボックスを追加。

| 状態 | 挙動 |
|------|------|
| **ON（デフォルト）** | 現行と同じ。ウィンドウ外支出先は `__agg-recipient` に集約表示 |
| **OFF** | 下記の変更が適用される |

### OFF 時の変更内容

#### 1. `__agg-recipient` ノードを非表示

- 集約支出先ノード（`その他の支出先`）を出力しない
- それに向かうエッジ（`project-spending → __agg-recipient`、`__agg-project-spending → __agg-recipient`）も出力しない

#### 2. ウィンドウ内支出先に接続のない事業を非表示

通常モードの `effectivelyHiddenIds` はウィンドウ・テール両方への流出がゼロの事業を隠す。
OFF 時はテールへの流出があっても **ウィンドウへの流出がゼロの事業を隠す**。

```text
effectivelyHiddenIds の追加条件（OFF 時のみ）:
  projectWindowValue === 0  （テールのみへの支出は無視）
```

#### 3. 事業(支出)ノードの高さをウィンドウ内支出のみで算出

通常モードでは `node.value - aboveWindowSpending`（テール分込み）。
OFF 時は `projectWindowValue`（ウィンドウ内支出のみ）を使用し、
事業(支出)と支出先の合計が一致するようにする。

`__agg-project-spending` ノードも同様に `otherProjectWindowTotal` を使用し、
作成条件も `otherProjectWindowTotal > 0` のみに絞る。

## 関連バグ修正

### `ky` フォールバック不具合（同時修正）

`computeLayout` のバイナリサーチで、ノード数が多く（例: 事業TopN=100 + ノードラベル表示 ON）
ギャップオーバーヘッドだけで `innerH` を超える場合、`ky = 0` に収束していた。

従来の `if (!isFinite(ky) || ky <= 0) ky = 1` フォールバックが `ky = 1` に上書きし、
億・兆円単位の value を持つノードが画面全体を覆うバグが発生していた。

`ky = 0` は「全ノードが最小高さ1px」を意味する正当な状態のため、
フォールバック条件を `if (!isFinite(ky)) ky = 1` に修正した。
