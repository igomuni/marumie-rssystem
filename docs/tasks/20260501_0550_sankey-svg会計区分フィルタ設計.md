# /sankey-svg 一般会計・特別会計フィルタ設計

作成日: 2026-05-01

## 目的

閲覧者が「一般会計のみ」「特別会計のみ」に絞って予算・支出フローを確認できるようにする。
現状は会計区分の区別なく全事業が混在して表示されており、一般会計と特別会計を分けた分析ができない。

---

## 現状調査

### CSVデータ

`2-1_RS_2024_予算・執行_サマリ.csv` に3列が存在する：

| CSV列名 | 型 | 説明 |
|--------|----|------|
| 会計区分 | string | `一般会計` / `特別会計` / `''` |
| 会計 | string | 特別会計の場合は「エネルギー対策特別会計…」などの名称。複数ある場合はカンマ区切り |
| 勘定 | string | 会計内の勘定名（例: 年金勘定） |

**会計区分の分布（行レベル・全37,981行）:**

| 会計区分 | 行数 |
|--------|------|
| 一般会計 | 15,639 |
| 特別会計 | 3,816 |
| 空 | 18,526 |

行レベルで空が多く見えるが、**同一事業の別予算年度行が空になっているだけ**。
事業レベルで集計すると実態は下記の通り。

**事業レベルの会計区分分類（全5,664事業）:**

| 分類 | 事業数 | 割合 | 説明 |
|-----|------|------|------|
| 一般会計のみ | 4,769 | 84.2% | すべての行が一般会計 |
| 特別会計のみ | 837 | 14.8% | すべての行が特別会計 |
| 両方（both） | 57 | 1.0% | 一般会計・特別会計両方の行を持つ |
| 完全空欄 | **1** | 0.02% | PID=18717「防災・安全交付金(都市公園・緑地等事業)」（国土交通省） |

**重要:** 事業レベルで判定すると「不明（完全空欄）」は実質1件のみ。
行レベルの空欄は分類上問題にならない。

**both 57件の内訳:**
- 厚生労働省 30件（一般会計 + 労働保険特別会計にまたがる雇用・就労系事業が中心）
- 経済産業省 9件（一般会計 + エネルギー対策特別会計）
- 環境省 8件
- 国土交通省 4件、文部科学省 2件、農林水産省 2件 他

一つの事業が複数区分にまたがる場合（例: 一般会計 + 特別会計の両方から支出）、
`generate-structured-json.ts` ではカンマ区切り文字列で `accountCategory` に保存される。

### 既存の型・データフロー

```
CSV（会計区分列）
  → generate-structured-json.ts
      → rs2024-structured.json の BudgetRecord.accountCategory（string）
          → generate-sankey-svg-data.ts
              → sankey-svg-{year}-graph.json の RawNode
                  → sankey-svg-filter.ts（filterTopN）
                      → app/sankey-svg/page.tsx（UI）
```

**現状のギャップ:**

| 箇所 | 状態 |
|------|------|
| `types/structured.ts` → `BudgetRecord.accountCategory` | ✓ 実装済み |
| `generate-sankey-svg-data.ts` → graph.json への出力 | ✗ **未出力** |
| `types/sankey-svg.ts` → `RawNode.accountCategory` | ✗ **未定義** |
| `sankey-svg-filter.ts` → accountCategory によるフィルタ | ✗ **未実装** |
| `app/sankey-svg/page.tsx` → 会計区分フィルタUI | ✗ **未実装** |

## 設計

### 1. `types/sankey-svg.ts` — RawNode に accountCategory を追加

```
RawNode に以下フィールドを追加：
  accountCategory?: string   // 'general' | 'special' | 'both' | undefined
```

`project-budget` ノードのみに付与する（ministry / recipient / total ノードには不要）。

**accountCategory の値の定義:**

| 値 | 意味 |
|----|------|
| `'general'` | 一般会計のみ |
| `'special'` | 特別会計のみ |
| `'both'` | 一般会計と特別会計の両方にまたがる |
| `undefined` | CSVに会計区分情報がない（不明） |

CSV の `会計区分` 列の値を以下のルールで変換する：
- `'一般会計'` → `'general'`
- `'特別会計'` → `'special'`
- カンマ区切りで両方含む（例: `'一般会計, 特別会計'`）→ `'both'`
- 空文字列 → `undefined`

### 2. `scripts/generate-sankey-svg-data.ts` — project-budget ノードに accountCategory を出力

`BudgetRecord.accountCategory` の値を上記変換ルールで変換し、
`project-budget` ノードの `accountCategory` フィールドに格納する。

`ministry` / `project-spending` / `recipient` ノードは付与しない
（これらは間接的に会計区分に関連するが、自身は会計区分を持たないため）。

### 3. `app/lib/sankey-svg-filter.ts` — filterTopN にフィルタ引数を追加

`filterTopN` に以下引数を追加する：

```
accountFilter: Set<'general' | 'special' | 'none'>
  // デフォルト: new Set(['general', 'special', 'none'])（全チェック）
```

**フィルタ動作（project-budget ノードに対して）:**

| project-budget の accountCategory | 表示条件 |
|----------------------------------|---------|
| `'general'` | `accountFilter` に `'general'` が含まれる |
| `'special'` | `accountFilter` に `'special'` が含まれる |
| `'both'` | `'general'` **または** `'special'` のいずれかが含まれる |
| `undefined`（完全空欄） | `accountFilter` に `'none'` が含まれる |

全チェック時は現状と同じ表示。セット内に何も含まれない場合は全ノードを非表示。

フィルタ後の `ministry` / `recipient` ノードは、残った `project-budget` から逆引きして
リンクが残るもののみを表示対象とする（既存の TopN フィルタ後処理と同様の方式）。

### 4. `app/sankey-svg/page.tsx` — UIコントロール追加

**UI仕様:**

フィルタパネル内に「会計区分」セクションを追加する。
3つの独立したチェックボックス：

```
☑ 一般会計  （4,769事業）
☑ 特別会計  （837事業）
☑ なし      （1事業）
```

デフォルトは全チェック（現状と同じ表示）。

**URL パラメータ:** `ac` を追加

文字列で有効な区分を列挙する。デフォルト（全チェック）は省略。

| `ac` の値 | 意味 |
|----------|------|
| （省略） | すべて表示（デフォルト） |
| `g` | 一般会計のみ |
| `s` | 特別会計のみ |
| `n` | なし（空欄）のみ |
| `gs` | 一般会計 + 特別会計 |
| `gn` | 一般会計 + なし |
| `sn` | 特別会計 + なし |

**state 追加:**
```
const [acGeneral, setAcGeneral] = useState(true)
const [acSpecial, setAcSpecial] = useState(true)
const [acNone,    setAcNone]    = useState(true)
```

URL パース例：`ac=gs` → `acGeneral=true, acSpecial=true, acNone=false`

`filterTopN` に渡すときに Set に変換：
```
new Set([
  ...(acGeneral ? ['general'] : []),
  ...(acSpecial ? ['special'] : []),
  ...(acNone    ? ['none']    : []),
] as ('general' | 'special' | 'none')[])
```

URL パース・シリアライズは既存の `filterMinistryNames` と同じパターンに従う。

## データ品質に関する注意事項

- 完全空欄（不明）は1事業のみのため、フィルタ精度への影響は無視できる
- both 57事業は一般会計・特別会計どちらのフィルタでも表示される仕様
- Sankey 図の合計金額はフィルタ後の表示ノードに基づくため、
  会計区分フィルタ適用時は合計値が変化することをユーザーに明示する

### `both` 分類の精度に関する既知の課題（別 Issue 対応予定）

**現象:** 「ポスト5G情報通信システム基盤強化研究開発事業(AI基盤モデル及び先端半導体関連技術開発事業)」
（PID=3522）はフィルタ上 `一般・特別` に分類されるが、RSシステム上では特別会計のみで執行されている。

**原因:** 2-1 CSV の行構造に起因する。

```
会計区分='' (集計行)  … 予算額・執行額あり（実際の数値はここ）
会計区分='特別会計'   … 予算額・執行額は空（会計内訳メタデータ行）
会計区分='一般会計'   … 予算額・執行額は空（会計内訳メタデータ行）
```

現実装は「会計区分列に値があるか」のみで判定するため、金額ゼロ（空）の `一般会計` メタデータ行の存在だけで `both` に分類される。実際の予算執行は特別会計のみ。

**方針:** RSシステムの元データをありのまま表示することが基本方針のため、現時点では修正不要。
元データ（RSシステム CSV）の記録精度の問題であり、システム側での補正は行わない。

**将来的な改善案（検討用）:**
- 判定ロジックを「金額がある行のみで会計区分を判定」に変更する
  → 予算額・執行額が空の会計区分行を除外してから一般/特別を判定
  → これにより `both` 57件のうち一部が `special` に正しく分類される可能性がある
- ただし変更によるデータ再生成が必要（`npm run generate-sankey-svg`）
