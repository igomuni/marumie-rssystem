# 支出先スライダー（オフセット方式）設計

## 目的

ユーザーが支出先ランキングの任意の区間を選択してSankeyに表示できるようにする。
Top100固定では網羅できない中位・下位支出先の分析が可能になる。

---

## 仕様

### スライダーの意味

| パラメータ | 既存 | 追加 |
|-----------|------|------|
| `topRecipient` | 表示する支出先の件数（ウィンドウ幅） | そのまま |
| `recipientOffset` | — | ウィンドウの開始位置（0始まり） |

ウィンドウ = `allSortedRecipients[offset : offset + topRecipient]`

**例: topRecipient=100, offset=0**
- 表示: 支出先 1〜100位
- その他の支出先: 101位以降

**例: topRecipient=100, offset=1**
- 表示: 支出先 2〜101位
- その他の支出先: 102位以降
- 1位（ウィンドウより前）は表示なし（集約もしない）

---

## 支出先の選定スコープ変更

### 現行

受取支出先ランキングを「TopN事業への直接支出」に限定して計算:
```
topSpendingIds → edges → recipientAmounts (Map)
→ sortedRecipients → slice(0, topRecipient)
```

### 変更後

**全事業を対象**に支出先ランキングを作成し、そこへウィンドウを適用する:
```
allEdges (全エッジ) → recipientTotalAmounts (Map: rid → 全事業合計額)
→ allSortedRecipients → slice(offset, offset + topRecipient)
```

理由: TopN事業が変わるとランキングが変化し、スライダーを動かしたときに支出先の順序が不安定になるため。全事業を対象にすることで支出先ランキングが安定し、スライダー操作が直感的になる。

---

## 上流ノード（事業・府省庁・総計）の連動変化

### 設計方針

- **事業ノードの表示値** = 当該ウィンドウ内の支出先への支出額合計（≦ 全支出額）
- **府省庁の表示値** = 配下の可視事業の合計
- ウィンドウ内への支出が**0の事業は表示しない**（集約ノードへ）
- 事業のTopNランキングは**ウィンドウ内支出額ベース（動的）**（スライドすると再ランキングされる）
  - 理由: スライダーを動かすとウィンドウ内に支出がなくなった事業は非表示になるため、総額ベース固定より動的再ランキングの方が直感的（ユーザー要求）
- 府省庁のTopNランキングは**総支出額ベース**（スライドしても順位は変わらない）

### 「その他の支出先」ノードの値

ウィンドウより後（(offset+topRecipient+1)位以降）への支出合計のみ。
ウィンドウより前の支出先への支出は `__agg-recipient` に含めず、単純にエッジとして表示しない。

### 「その他の支出先」ノードの表示高さ上限

オフセットを増やすとウィンドウより後の支出先数が増加し、`__agg-recipient` の合計値がウィンドウ内支出先の合計を大きく上回る。その結果、ノードが巨大になってウィンドウ内支出先のラベルが見えなくなる問題がある。

これを解消するため、**表示高さに上限を設ける**（実際の金額はツールチップに保持）。

```
minWindowRecipientValue = min(windowRecipients[i].value)
layoutCap = minWindowRecipientValue × topRecipient
__agg-recipient の表示高さ = min(aggRecipientValue, layoutCap)
```

意味: ウィンドウ内の最小支出先を `topRecipient` 個並べた高さを上限とする。これはウィンドウ列の総高さの上限に相当する。

- `RawNode.layoutCap` フィールドで上限を指定
- `RawNode.rawValue` フィールドに実際の金額を保持
- レイアウトエンジンはリンク合計値計算後に `layoutCap` を適用し、`rawValue` に実値を退避
- ツールチップ・ノードラベルは `rawValue ?? value` を表示
- 上限適用中は「※表示高さは上限値で制限」と表示

## filterTopN の変更箇所

### 引数追加

```
filterTopN(
  allNodes, allEdges,
  topMinistry, topProject,
  topRecipient,
  recipientOffset   ← 追加
)
```

### 処理変更: 支出先選定

```
// 変更前
const sortedRecipients = ... .sort(...);
const topRecipientIds = new Set(sortedRecipients.slice(0, topRecipient));

// 変更後
const allSortedRecipients = allEdges
  で r-* ターゲットの合計を計算
  → sort → allSortedRecipients

const windowRecipients = allSortedRecipients.slice(offset, offset + topRecipient);
const topRecipientIds = new Set(windowRecipients.map(([id]) => id));
```

### 処理変更: 事業のフィルタリング

TopN事業のうち、ウィンドウ内支出先への支出が 0 の事業は集約ノードへ移す:

```
// ウィンドウ内支出先への支出額を事業ごとに計算
const projectWindowValue: Map<projectId, amount>

// topProjectNodes を分割:
const visibleTopProjects = topProjectNodes.filter(p => projectWindowValue.get(p.id) > 0)
const hiddenTopProjects  = topProjectNodes.filter(p => !projectWindowValue.get(p.id))
// hiddenTopProjects は otherProjects に合流させる
```

### 事業ノードの表示値

```
// 変更前: p.value (全支出額)
// 変更後: projectWindowValue.get(p.id) を使用
nodes.push({ ...p, value: windowValue })
```

ただし事業のTopNランキング用ソートは p.value (総額) を使い続ける。

## UIコントロール

### スライダー

```
recipientOffset: [0 ← slider → maxOffset]
maxOffset = max(0, allSortedRecipients.length - topRecipient)
```

既存の支出先スライダー（`topRecipient`）の隣に配置。
表示例: `支出先: 37〜136位 / 全1,284件`

### リセット動作

Reset時は `recipientOffset = 0` に戻す。

## 既知の考慮事項

### 事業の表示値とエッジ値の整合

事業の表示値がウィンドウ内支出先への部分額になるため、**事業←府省庁のエッジ値も同様に部分額に変更する必要がある**。さもないと事業ノードの高さとエッジ幅が一致しない。

- `project-spending` ノードの `value` = ウィンドウ内支出先への合計
- `project-budget → project-spending` エッジの `value` = ウィンドウ内支出額（支出ノードと同値）
- `project-spending → recipient` エッジ = 変更なし（ウィンドウ内のみ）

予算と支出の表示が乖離するが、現状の実装でも同様の乖離（直接支出 vs 予算）があるため許容とする。

### 「その他」支出先（名前が「その他」の実データ）との混同

現行と同様。`その他` は通常の支出先ノードとして扱われ、ランキング内に自然に位置する。

### スライダー上限の動的変化

`topRecipient` スライダーを動かすと `maxOffset` が変わる。
`recipientOffset > maxOffset` になる場合は `Math.min(recipientOffset, maxOffset)` でクランプする。

## 実装スコープ

### filterTopN の変更（`app/sankey-svg/page.tsx`）

1. 引数 `recipientOffset` 追加
2. 全エッジから全支出先の合計額を計算し `allSortedRecipients` を生成
3. `windowRecipients = slice(offset, offset + topRecipient)`
4. 事業のウィンドウ内支出額を計算 (`projectWindowValue`)
5. ウィンドウ内支出 = 0 の事業を集約ノードに移動
6. 事業ノードの value をウィンドウ内支出額に変更
7. 府省庁ノードの value を再計算
8. `__agg-recipient` value = ウィンドウより後の支出先への支出合計（ウィンドウより前は含めない）
9. `__agg-recipient` に `layoutCap = minWindowRecipientValue × topRecipient` を設定（高さ上限）

### レイアウトエンジンの変更（`computeLayout`）

- リンク合計値でノード value を更新した直後に `layoutCap` を適用
- `rawValue` に実際の value を退避し、`value = layoutCap` に書き換える

### State追加（`RealDataSankeyPage` in `app/sankey-svg/page.tsx`）

```typescript
const [recipientOffset, setRecipientOffset] = useState(0);
```

### UI追加

- `recipientOffset` スライダー（またはナビゲーションボタン）
- 現在表示範囲の表示（例: `37〜136位 / 全1,284件`）
