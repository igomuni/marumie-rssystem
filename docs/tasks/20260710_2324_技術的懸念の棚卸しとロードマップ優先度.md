# 技術的懸念の棚卸しとロードマップ優先度 — WP1〜WP4 完了時点のふりかえり

日時: 2026-07-10 23:24 (JST)
ステータス: 検討（オーケストレーターによる懸念の棚卸しと優先度提案）
関連: [AIチャット機能進化ロードマップ](20260709_0759_AIチャット機能進化ロードマップと実装計画.md)（WP1〜WP4・WP5-0 実装完了時点）

## 1. 現在地

ロードマップの対話型機能はすべて実装済み（PR #253〜#260 の8本・2日間）。残る WP はいずれも**ユーザー判断ゲート付き**:

- **WP3-4 AIタグ付け**: LLM バッチのコスト投下判断が前提。需要の証拠（利用ログの解釈不能語・0件語）はまだ蓄積が薄い
- **WP5-1 公開ガード**: 「チャットを Vercel で公開するか」というプロダクト判断とセット

## 2. 技術的懸念（優先順）

### 懸念1: 自動テストがゼロのまま機能が積み上がった【最優先】

`app/lib/` の Pure 関数群（エージェントループ・highlights 6指標・spending-search・compareYearsSummary・クランプ群）は設計上テスト容易なのに、品質の守りが tsc / lint / 手動ハーネスのみ。境界値バグ（0除算・分離集計・excerpt 位置写像・分位点）は本来ユニットテストの領域で、CodeRabbit が拾った母集団計算の非対称も同類だった。

- **対処案**: Pure 関数限定の最小テストを1PRで導入（vitest 等・LLM 不要・CI 実行可能）。Sonnet 委譲向きの定型タスク

### 懸念2: hy3:free（単一無料プロバイダ）に体験が乗っている

成功率・レイテンシ（実測25〜140秒）・障害の癖はすべて Novita 経由の1モデル依存。一過性障害を複数回実測しており、無料枠の変更で体験が音もなく劣化しうる。エンドポイントは環境変数で差し替え可能だが、フォールバック連鎖はない。

- **対処案（運用）**: 評価ハーネス全シナリオのフル実行を定点観測として月次程度で回す（結果は docs/tasks に記録）。モデル乗り換え時は WP1-2 ハーネスで疎通・品質を比較

### 懸念3: Vercel サイズ問題は「封印」であって「解決」ではない

チャット関数の `outputFileTracingExcludes` は「Vercel では機能無効だから」成立する時限的回避。公開判断が下りた瞬間に R2 offload（既存検討 20260613）が前提条件として立ち上がる。関数ごとのデータ複製同梱の構造上、年度追加でデプロイ全体も線形に膨らむ。

- **対処案**: 公開の意思が固まる**前に** R2 offload の設計だけ先行させる（実装は公開判断とセットで）

### 懸念4: page.tsx の肥大継続

3,000行超にチャット状態・SSEパーサ・進行 state を追加してきた。レイヤー規約でロジック汚染は防げているが、状態管理の置き場が単一ファイルに集中。SSE パーサはページとハーネスの2実装。

- **対処案**: 次にチャット系 UI を触る PR で `useAiChat` カスタムフックへの抽出を同乗させる（専用PRを立てるほどではない）

### 軽微（気が向いたときの掃除）

- 利用ログ（data/usage-log.jsonl）にローテーションがない（dev 専用なので実害は先）
- ロードマップ文書が完了済み WP を反映していない（INDEX の状態ラベルで代替できている）

## 3. ロードマップとの優先度 — 提案

**推奨: 負債返済（懸念1→3）を先、ロードマップ残（WP3-4・WP5-1）は判断材料が揃うまで待つ。**

理由:

1. 対話型機能は完成しており、**急ぎの機能ギャップがない**。残 WP はどちらも「公開するか」「コストを投下するか」というプロダクト判断待ちで、技術側が先回りできるのは判断材料の整備だけ
2. 公開判断が下りたとき最初に必要なのは WP5-1 と R2 であり、その土台を安全に組むには**テストが先にあるほうが安い**（ガード・課金・レートリミットはバグの許容度が低い領域）
3. WP3-4（タグ付け）は利用ログの未充足需要が要求仕様の源泉という設計。ログの蓄積がまだ2日分しかなく、**今作ると需要ではなく想像でタグ体系を決めることになる**

### 提案する着手順

| 順 | 作業 | 規模 | 実施条件 |
|----|------|------|---------|
| ① | Pure 関数の最小テスト導入（懸念1） | 小・1PR | すぐ着手可（Sonnet 委譲） |
| ② | R2 offload の設計先行（懸念3） | 設計のみ | ①の後。実装は公開判断とセット |
| ③ | WP5-1 公開ガード + R2 実装 | 中〜大 | **公開判断が下りたら** |
| ④ | WP3-4 AIタグ付け | 中 + LLMコスト | 利用ログに未充足需要が溜まったら（月単位の蓄積後） |
| 随時 | useAiChat フック抽出（懸念4） | 小 | 次のチャットUI変更PRに同乗 |
| 運用 | ハーネスフル実行の定点観測（懸念2） | 実行のみ | 月次目安 |
