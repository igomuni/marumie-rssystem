## 5つの利用モード

### 1. 見る — 規模感の直感化（一般の納税者）

サンキー図の本質的な価値は金額の「比較の直感」。151兆円という数字は人間には処理できないが、帯の太さなら一目で分かる。ニュースで聞いた事業名を検索し、その事業が省庁の中でどれくらいの太さか、資金がどこへ流れているかを見る——最も裾野の広い使い方であり、フィルタパネル・品質スコア表示・法人番号表示はすべてこの「見る」を支える部品である。

### 2. 調べる — 仮説駆動の調査（記者・研究者・政策ウォッチャー）

実レポート群がそのままユースケースの証明になっている:

- ベンダー集中の定量化（e-Tax = NTTデータ親会社単独85%）
- 執行団体の壁（再エネ: SII が府省庁横断で1,747億円）
- 透明性の質の分類（「その他」67%の給付型 vs 企業名まで追える調達型）
- 計上のねじれの発見（国税システム群のデジタル庁一括計上・府省庁フィルタから漏れる）

従来は生CSVの自力集計が必要だった一次調査が、数十分で図つきで出る。品質スコアから「報告がずさんな事業」を逆引きして調査の入口にする使い方もある。

### 3. 頼む — 自然言語での調査委託（技術のない利用者 + AIエージェント）

クエリAPI・プレイブック・HATEOAS リンク群により、「〇〇について調べて」という自然言語の依頼から図つき調査レポートまでがエージェントで完結する。モード2が技術リテラシー不要になったと言い換えられる。サイトの性格が「可視化UI」から「データ基盤 + 調査エンジン」に変わった転換点。

### 4. 測る — 行政の報告品質へのフィードバック装置

品質スコア・法人番号誤記載の⚠表示・「その他」の可視化は、事業の中身ではなく**行政事業レビュー制度そのものの記載品質**を測っている。「688億円の交付金の支出の7割が『その他』」が誰にでも見える状態は、報告様式の改善圧力として機能しうる。行政側・制度設計側が自らの報告品質を確認する用途も含む、独立した利用モード。

### 5. 知る — 政府調達市場の構造分析（企業側）

どの企業がどの省庁からいくら受注し、誰に再委託しているか。競合分析・営業・入札戦略の材料としての使い方。法人番号キーの名寄せ（#243）が基盤で、gBizINFO 接続が入ればさらに強くなる。

## 独自価値 — 「どこで見えなくなるか」を描くこと

このサイトが普通の可視化サイトと違うのは、**「見えるもの」だけでなく「どこで追跡が途切れるか」を正確に描いている**点にある。「その他」ノード、執行団体の壁、FMS の終端化——透明性の議論では「何が見えないか」の地図こそが出発点であり、見えない部分の輪郭を示すこと自体が製品価値になっている。

## 出力の正しい受け取り方 — 「答え」ではなく「良い質問の生成」

対象は行政事業レビュー分（国の全予算の約27%）・単年度スナップショット・報告品質依存という限界がある以上、このサイトは会計監査の代わりにはならない。その代わり、

- 「この688億円はなぜ7割が『その他』なのか」
- 「なぜ KSK は1年で +221億円なのか」

という**一次資料に向かうべき質問を、誰でも・速く・根拠つきで作れる**。5つのモードすべてに共通する核はここにあり、機能追加の優先判断も「良い質問をより速く・より正確に生成できるか」を物差しにできる。
